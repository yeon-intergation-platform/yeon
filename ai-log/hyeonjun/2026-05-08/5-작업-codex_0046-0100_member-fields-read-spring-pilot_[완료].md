# member-fields read spring pilot

- 작업 목표: `member-fields` read lane를 다음 Spring 파일럿으로 열고 inventory/backlog부터 시작한다.
- 작업 범위: `GET /api/v1/spaces/{spaceId}/member-tabs/{tabId}/fields` 우선 검토
- 기준: Next는 outward auth/BFF 유지, Spring은 read source of truth 담당
- 비목표: write/reorder/auth migration 동시 진행

- `docs/architecture/spring-member-fields-read-pilot-inventory.md`로 현재 Next fields GET route의 overview lazy backfill + values 결합 구조를 고정했다. 1차 추천은 definition read만 Spring으로 이동하고 overview backfill과 member values 결합은 다음 lane으로 분리하는 것이다.
- `docs/architecture/spring-member-fields-read-package-plan.md`로 controller/service/repository/dto/mapper/model 경계와 1차에서 values/backfill을 제외하는 원칙을 고정했다.
- `docs/architecture/spring-member-fields-read-api-contract.md`로 outward/internal endpoint, header, `{ fields: [...] }` shape, `TAB_NOT_FOUND`/`TAB_SPACE_MISMATCH` 에러 초안을 고정했다.
- `docs/architecture/spring-member-fields-read-skeleton-file-plan.md`로 model/repository → dto/mapper/service → controller → Next cutover 순서를 고정했다.
- 차수 A 구현 완료: `MemberFieldDefinitionEntity`, `MemberFieldReadRepository`, `MemberFieldReadRepositoryTests`를 추가해 space lookup + tab lookup + deleted 제외 ordered field read를 Spring으로 올렸다.
- 차수 B/C 및 부분 cutover 완료: dto/mapper/service/controller와 테스트를 추가했고, Next `fields` GET은 `memberId`가 없을 때 Spring fetch로 전환했다. `memberId` values branch는 아직 legacy 경로를 유지한다.
- 검증 증거: `./gradlew test --tests *MemberFieldReadRepositoryTests`, `./gradlew test --tests *MemberFieldReadServiceTests`, `./gradlew test --tests *MemberFieldReadControllerTests`, `./gradlew test`, `pnpm --filter @yeon/web exec vitest run src/app/api/v1/spaces/[spaceId]/member-tabs/[tabId]/fields/__tests__/route.test.ts`, `pnpm --filter @yeon/web typecheck`, `pnpm --filter @yeon/web build`, runtime smoke `GET /spaces/space_member_fields_smoke/member-tabs/mtb_fields_smoke_tab/fields` -> 200 + 2 fields + deleted 제외 + options JSON 보존 확인.
- 이번 단계에서 제거/축소된 Next direct 로직: `memberId` 없는 GET 경로에서 `getOverviewTab(...)`, `createDefaultOverviewFields(...)`, `getFieldsForTab(...)` 직접 orchestration을 더 이상 실행하지 않고 Spring read 결과만 중계한다. 남은 legacy는 `memberId` values 결합 branch뿐이다.
- `docs/product/backlog/spring-member-field-values-read-pilot.md`로 `memberId` values branch를 다음 별도 파일럿으로 열었다.
- `docs/architecture/spring-member-field-values-read-pilot-inventory.md`로 현재 Next `memberId` branch의 overview backfill + definitions read + values join orchestration을 고정했다.
- `docs/architecture/spring-member-field-values-read-package-plan.md`와 `spring-member-field-values-read-api-contract.md`로 values lane을 `GET /spaces/{spaceId}/member-tabs/{tabId}/field-values?memberId=...` 내부 endpoint로 분리하고, Next가 `fields + values`를 조합하는 방향을 고정했다.
- values lane cutover 진행: backend `member_field_values.read` controller/service/repository/dto + tests를 추가했고, Next `fields` GET의 `memberId` branch는 이제 `fetchMemberFieldsFromSpring + fetchMemberFieldValuesFromSpring` 조합으로 동작한다.
- 이번 단계에서 제거/축소된 Next direct 로직: `memberId` branch에서 `getFieldValuesForDefinitions(...)`와 `getFieldsForTab(...)` 직접 호출을 제거했다. 현재 이 route에 남은 legacy read/write 결합은 overview 탭일 때 `getOverviewTab(...) + createDefaultOverviewFields(...)` lazy backfill뿐이다.
- verify 중 실제 버그 발견/수정: Spring `field-values` 응답의 `valueJson`이 Jackson 내부 속성 객체처럼 직렬화되던 문제를 확인했고, service에서 `JsonNode -> plain Object` 변환으로 수정했다. 재발 방지를 위해 service/controller test에 `valueJson` 배열 shape 검증을 추가했다.
- 검증 증거 추가: `./gradlew test --tests '*MemberFieldValueReadRepositoryTests'`, `./gradlew test --tests '*MemberFieldValueReadServiceTests'`, `./gradlew test --tests '*MemberFieldValueReadControllerTests'`, `./gradlew test`, `pnpm --filter @yeon/web exec vitest run src/app/api/v1/spaces/[spaceId]/member-tabs/[tabId]/fields/__tests__/route.test.ts`, `pnpm --filter @yeon/web typecheck`, `pnpm --filter @yeon/web build`, runtime smoke `GET /spaces/space_member_values_smoke/member-tabs/mtb_values_smoke_tab/field-values?memberId=mem_values_smoke` -> 200 + `valueJson[0] == in_progress` + deleted/other tab-space value 제외 확인.
- 남은 마지막 legacy 제거 lane으로 `docs/product/backlog/spring-member-fields-overview-bootstrap-pilot.md`를 추가했다.
- `docs/architecture/spring-member-fields-overview-bootstrap-pilot-inventory.md`로 현재 route의 `getOverviewTab(...) + createDefaultOverviewFields(...)` side effect를 read가 아닌 bootstrap/write 성격으로 고정했다.
- `docs/architecture/spring-member-fields-overview-bootstrap-package-plan.md`와 `spring-member-fields-overview-bootstrap-api-contract.md`로 `POST /spaces/{spaceId}/member-tabs/{tabId}/bootstrap-overview-fields` internal endpoint 초안을 고정했다.
- 다음 구현 목표: Spring bootstrap endpoint를 만든 뒤 Next `fields` GET route에서 direct overview backfill service 호출을 제거하고, overview 탭일 때도 Spring fetch만 하도록 축소한다.
- overview bootstrap cutover 진행: Spring `member_fields.bootstrap_overview` controller/service/repository/support/dto와 테스트를 추가했고, Next `fields` route는 이제 overview 기본 필드 초기화를 `POST /spaces/{spaceId}/member-tabs/{tabId}/bootstrap-overview-fields` internal endpoint로 위임한다.
- 이번 단계에서 제거/축소된 Next direct 로직: `apps/web/src/app/api/v1/spaces/[spaceId]/member-tabs/[tabId]/fields/route.ts`에서 `getOverviewTab(...)`, `createDefaultOverviewFields(...)`, `requireSpaceInternalIdByPublicId(...)` 직접 호출을 모두 제거했다. 이 route는 이제 auth/BFF + Spring fetch만 남는다.
