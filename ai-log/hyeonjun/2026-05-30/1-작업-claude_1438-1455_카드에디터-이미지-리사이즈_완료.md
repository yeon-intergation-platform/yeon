# 카드 에디터 이미지 리사이즈 — resizable image node + width/height serialize

- 실행 주체: claude
- 시작: 2026-05-30 14:38 / 종료: 2026-05-30 14:55
- 상태: 완료

## 요청

카드 에디터 본문에 삽입된 이미지를 별도 슬라이더/상단 컨트롤이 아니라 **본문 안 이미지 노드 자체**에서
드래그 리사이즈하게 한다.

- 이미지를 클릭해 선택 → 테두리 + 우하단 resize handle 노출
- 우하단 모서리 드래그로 width/height 실시간 변경
- 드래그 중 기본은 비율 유지, Shift 누를 때만 비율 해제
- 저장 시 width/height 함께 직렬화

## 진단

커밋된 코드에 슬라이더는 없었다. 이미 Tiptap `ResizableCardEditorImageExtension` node view가 있었으나
(1) 핸들이 `ew-resize`(가로 전용), (2) `width`만 조절·직렬화(height 항상 auto), (3) 핸들 상시 노출,
(4) "표시 크기: 480px × auto" 라벨 상시 표시. 스크린샷의 "슬라이더처럼 보이는 것"은 이미지 깨진 상태에서
collapsed wrapper + 핸들 + 라벨이 겹친 모습으로 판단(미리보기 깨진 이미지 아이콘이 단서).

## 변경 (apps/web/src/features/card-service/components/)

- `card-editor-image-utils.ts` (+ test): `CARD_EDITOR_IMAGE_MIN/MAX_HEIGHT`,
  `clampCardEditorImageHeight`, `parseOptionalCardEditorImageHeight`(없거나 비정상 → null=auto) 추가.
- `card-editor-extensions.ts`: node view를 코너 드래그 + 비율 유지(기본) + Shift 비율 해제 +
  width/height 직렬화로 재작성. `height` attribute 추가(default null, null이면 attribute 생략).
- `card-rich-markdown-editor-view.tsx`: 핸들 `nwse-resize`/`is-selected|is-resizing`에서만 노출,
  라벨 리사이즈 중에만, img `cursor:pointer`.
- `markdown-content.tsx`: 미리보기 sanitizeHtml에서 명시 height를 `aspect-ratio`+`height:auto`로 반영,
  width/height 보정을 `card-editor-image-utils`로 통일(local 상수/clamp 제거).

저장 경로는 `editor.getHTML()` → frontText/backText(HTML)라 `<img width height>` attribute로 직렬화된다.
일반(비율유지) 드래그는 height=null(auto)로 두어 width만 직렬화되고 원본 비율을 유지한다.
Shift 비율 해제로 W×H를 확정하면 height까지 직렬화된다.

## 적대적 코드 리뷰(워크플로) 반영

다차원 리뷰에서 확인된 8건 중 6건 반영, 1건 모델 단순화로 근본 제거, 1건 오탐 판정:

- (F1) `is-resizing` 클래스 누수 → `pointercancel` 처리 + cleanup에서 리스너/클래스 모두 해제, destroy 정리.
- (F3) `currentWidth/Height` stale(undo/redo) → 드래그 시작 시 문서에서 **live node attrs 직접 읽기**.
- (F5/F6) 명시 height 비율유지 드래그 clamp 비율깨짐 + 코너 축 불일치 → **일반 드래그는 항상 height=null로
  자연 비율 리셋** + dominant-axis(가로·세로 중 큰 쪽으로 폭 결정)로 단순화. 명시 height는 Shift로만 생성.
- (F7) 미리보기 명시 height + max-width 왜곡 → `aspect-ratio`로 반응형 비율 유지(에디터/미리보기 동일).
- (F4) 미로드 이미지 Shift 기준점 부정확 → fallback에 offsetHeight 추가(완전 해결은 아님, 2차 모드/희귀).
- (F8) 핸들 위치 오차 → wrapper/img 둘 다 `max-width:100%`라 함께 축소되어 어긋나지 않음(오탐).
