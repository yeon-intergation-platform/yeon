# Spring 전환 300단계 백로그 작성 로그

## 목표
- Spring 중심 전환을 여러 프롬프트로 나눠 안전하게 진행할 수 있도록 migration 브랜치와 300단계 백로그 초안을 만든다.

## 오늘 한 일
- 현재 브랜치를 `migration/spring-platform-core`로 분기했다.
- `study-platform-mvp`의 참고 context 파일 위치를 찾았다.
- Yeon용 `docs/product/backlog/spring-platform-migration.md` 초안을 만들었다.
- 백로그는 30차수 × 10단계 = 300단계로 쪼갰다.
- 이번 턴에서는 설계/문서 계획만 만들고 실제 Spring 코드 이관은 시작하지 않았다.

## 핵심 판단
- 전면 전환 목표는 유지하되, 작업 단위는 아주 작게 쪼개야 한다.
- 먼저 skills_context/SHARED 구조와 backend context/verify/local run 문서를 Yeon용으로 옮기는 쪽이 안전하다.
- auth보다 문서/컨텍스트/검증 체계를 먼저 고정하는 순서가 실수 위험을 낮춘다.

## 다음 추천 작업
1. 1차 중 1~3단계만 다루는 작은 프롬프트로 전환 원칙 SSOT 문서를 만든다.
2. 그 다음 `skills_context/SHARED/` 도입 여부와 폴더 구조만 결정한다.
3. 이후 `ai-surface-policy` 또는 `study-platform-backend-context` 중 하나만 Yeon용 초안으로 옮긴다.
