# 공개 콘텐츠 헤더 채널 링크 정상화

## 작업 시간

- 20:41 ~ 21:10 (KST)

## 범위

- `YEON 고객지원`·`YEON 뉴스`·`YEON 블로그` 제목 옆으로 Support·News·Blog 채널 탭을 이동한다.
- 공개 콘텐츠 채널을 향하는 화면 링크는 로컬에서 내부 `/support`·`/news`·`/blog` 경로를 유지하고, 운영 content subdomain 간 이동은 proxy의 공통 라우팅 규칙으로 canonical host에 전환한다.
- 기존 정규 URL은 SEO·RSS 용도로 보존한다.

## 제약

- 운영 데이터 덤프를 적용한 로컬 환경에서만 확인한다.
- PR 생성, push, `main` 병합, 배포는 하지 않는다.

## 결과

- Support·News·Blog 탭을 본문에서 제거하고 `YEON 고객지원`·`YEON 뉴스`·`YEON 블로그` 제목 바로 옆의 공통 헤더에 배치했다.
- 모바일에서는 같은 공통 헤더의 두 번째 줄에 탭을 배치해 제목·설정 버튼과 겹치지 않도록 했다.
- 채널 탭뿐 아니라 공개 콘텐츠 채널을 향하는 카드·필터·브레드크럼·CTA 링크도 canonical 운영 도메인이 아니라 내부 `/support`·`/news`·`/blog` 경로만 사용하도록 통일했다.
- content subdomain에서 다른 채널의 내부 경로로 들어오면 proxy가 대상 canonical host로 308 전환하도록 공통 라우팅을 보완했다.
- canonical 운영 URL은 SEO 메타데이터·RSS·구조화 데이터에만 유지했다.
