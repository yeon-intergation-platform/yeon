# 공개 콘텐츠 반복 정보 정리 작업 로그

## 작업 범위

- Support·News·Blog 공개 콘텐츠의 반복 패널과 중복 안내를 축소한다.
- News 공지의 적용 서비스·적용일은 제목 아래 압축 메타데이터로 유지한다.
- 공통 헤더, 운영 데이터 덤프, 공개 콘텐츠의 로컬 내부 링크 정책은 유지한다.

## 제약

- 로컬 브랜치에서만 수정하고 PR, push, `main` 병합, 배포를 하지 않는다.
- 사용자에게 보이는 UI 변경이므로 Playwright 기준의 before/after 데스크톱·모바일 스크린샷을 남긴다.

## 결정

- 본문을 자동으로 요약해 다시 노출하는 News·Support 패널은 제거한다.
- Blog의 운영/근거 패널은 사용자 독해 흐름을 끊으므로 제거하고 기존 하단 관련 글만 남긴다.
- News·Blog 홈의 중복 섹션명과 보조 문구는 필터 또는 제목 중 하나로 통합한다.

## 진행 상태

- [완료] News 공지의 적용 서비스·적용일을 제목 아래 한 줄 메타데이터로 압축하고, `핵심 확인 사항`·자동 안내 패널·반복 callout을 제거했다.
- [완료] Support 상세의 `먼저 확인할 것` 요약과 Blog 상세의 운영 주체·교차 채널·repo 근거 패널을 제거했다.
- [완료] News·Blog 홈의 중복 섹션명·보조 문구·분류 재나열 영역을 정리했고, Support 홈의 검색 중심 흐름은 유지했다.
