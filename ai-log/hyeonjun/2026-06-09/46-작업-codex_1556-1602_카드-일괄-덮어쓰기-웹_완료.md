# 카드 일괄 추가 덮어쓰기 버튼 웹 적용

## 목표

- 웹 카드 일괄 추가 화면에서 기존 카드 전체 삭제 후 붙여넣은 카드로 교체하는 덮어쓰기 액션을 명확히 제공한다.

## 진행

- card-service 규칙 확인 완료.
- 기존 web/mobile replaceCards 흐름 확인 완료.

## 변경

- 일괄 추가 폼 하단 액션 영역에 덮어쓰기 설명을 추가했다.
- 덮어쓰기 버튼을 `N장 덮어쓰기`로 표시하고, 추가 버튼 옆에 명확히 배치했다.
- 기존 `replaceCards` 포트는 게스트/서버 모두 기존 카드 삭제 후 bulk 생성 흐름을 사용함을 확인했다.

## 검증

- `CI=true pnpm --filter @yeon/web lint`
- `CI=true pnpm --filter @yeon/web typecheck`
- `CI=true pnpm verify:parity`
- `git diff --check`
- Playwright/Chrome 로컬 확인: `/card-service/decks/codex-overwrite-deck` 일괄 추가 모달에서 `1장 덮어쓰기`, `1장 추가`, 덮어쓰기 설명 노출 확인. 스크린샷: `/tmp/card-bulk-overwrite-actions.png`

## 참고

- 로컬 web 단독 실행 중 backend 미기동으로 community-chat/typing-character-frame 요청 503/500 로그가 있었으나, 카드 덱 상세 게스트 화면과 버튼 노출 검증에는 영향 없음.
