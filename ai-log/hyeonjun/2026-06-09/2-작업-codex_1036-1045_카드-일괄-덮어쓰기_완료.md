# 카드 일괄 추가 덮어쓰기

## 목표

- 카드 일괄 추가 화면에 `덮어쓰기` 버튼을 추가한다.
- 덮어쓰기는 기존 카드를 모두 삭제한 뒤 입력된 일괄 카드로 완전 대체한다.

## 제약

- 카드 서비스 web/mobile 패리티를 고려한다.
- 게스트/인증 저장소 모두에서 동작해야 한다.
- 작업 후 main PR/merge까지 진행한다.

## 진행

- `yeon-4`에서 `origin/main` 기반 브랜치 `codex/card-bulk-overwrite` 생성.
- web 일괄 추가 폼에 기존 `N장 추가` 옆 `덮어쓰기` 버튼과 되돌릴 수 없음 확인창을 추가했다.
- 공용 카드 item repository 포트에 `replaceCards`를 추가하고 web/mobile 어댑터를 구현했다.
- web/mobile 게스트 저장소에 기존 카드 삭제 후 새 카드 삽입 흐름을 추가했다.
- 인증 사용자는 기존 상세 조회 후 기존 item 삭제를 완료하고 일괄 생성으로 대체한다.
- 모바일 일괄 추가 시트에도 동일 의미의 `덮어쓰기` 액션/문구를 추가했다.
- Universal UI parity registry에 `replaceCards` 의미를 기록했다.

## 검증

- `CI=true pnpm install`
- `CI=true pnpm verify:parity`
- `CI=true pnpm --filter @yeon/web lint`
- `CI=true pnpm --filter @yeon/web typecheck`
- `CI=true pnpm --filter @yeon/mobile lint`
- `CI=true pnpm --filter @yeon/mobile typecheck`
- `CI=true pnpm --filter @yeon/ui typecheck`
- `CI=true pnpm --filter @yeon/web build`
- Playwright + 로컬 `@yeon/web start`: 덮어쓰기/추가 버튼 노출, 확인창 승인, 기존 2장 삭제 후 1장으로 대체, `전체 1` 확인.
- `git diff --check`

## 결과

- 카드 일괄 입력에서 새 카드 묶음으로 기존 덱 카드를 완전히 교체할 수 있다.
