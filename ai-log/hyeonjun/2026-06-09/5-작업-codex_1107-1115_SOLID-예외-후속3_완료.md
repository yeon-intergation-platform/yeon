# SOLID/예외 원칙 후속 3차

## 목표

- 300개 백로그 중 다음 P1 빈 catch 항목을 실제 코드 수정으로 해소한다.
- 예외를 숨기지 않고 원인 로그/보존/재던지기 기준을 적용한다.

## 범위

- 웹 타자 서비스 예외 처리 항목 24~36 우선.
- 상담 워크스페이스는 동결 범위이므로 제외한다.

## 진행

- `yeon-2`에서 `origin/main` 기반 `codex/solid-exception-followup-3` 브랜치 생성.
- 항목 24~36을 코드에서 확인하고 빈 catch를 원인 로그가 있는 catch로 교체했다.
- 룸 cleanup/퇴장 fallback은 기존 동작을 유지하면서 실패 원인을 `console.warn`에 남긴다.
- 백로그 항목 24~36을 완료 표시하고 3차 적용 완료 섹션을 추가했다.

## 검증

- `CI=true pnpm --filter @yeon/web lint` 통과
- `CI=true pnpm --filter @yeon/web typecheck` 통과
- `git diff --check` 통과
- 백로그 번호 300개 유지, 완료 항목 1~36 확인
- 대상 타자 서비스 파일 7개에서 `catch {` 제거 확인

## 결과

- 웹 타자 서비스 P1 빈 catch 항목 24~36 완료.
