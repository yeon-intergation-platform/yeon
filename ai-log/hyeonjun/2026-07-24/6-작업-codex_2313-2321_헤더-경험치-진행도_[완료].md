# 헤더 경험치 진행도 가시성 개선

- 시작: 23:13
- 종료: 23:21
- 상태: 완료
- 대상: `YeonProgressBar`, `YeonExperienceBadge` 웹·모바일 공용 UI
- 목표: 레벨 배지에서 채운 구간과 남은 구간을 즉시 구분할 수 있게 한다.
- 시각 검증: `/game-service` 공통 헤더의 before/after Playwright 스크린샷과 계산된 트랙·채움 너비

## 변경 결과

- 공용 진행바의 기본 트랙을 배경과 같은 `neutral 50`에서 구분되는 `neutral 100`으로 변경했다.
- 웹·모바일 헤더 경험치 막대를 `56x8px`로 맞춰 짧은 구간에서도 비율을 읽기 쉽게 했다.
- 퍼센트 문구나 장식은 추가하지 않고 트랙·채움의 대비만으로 정보 위계를 정리했다.

## 시각 검증

- 진입 화면: `/game-service` 공통 헤더
- 상태: 인증·경험치 API를 Playwright에서 `Lv.4`, `40 / 100 XP`로 고정
- before: 트랙 `48x8px`, 채움 `19px`(40%), 트랙 `rgb(250, 250, 250)`, 채움 `rgb(17, 17, 17)`
- after: 트랙 `56x8px`, 채움 `22px`(40%), 트랙 `rgb(229, 229, 229)`, 채움 `rgb(17, 17, 17)`
- 증거:
  - `header-experience-progress-screenshots/before-header-experience-40-percent-desktop.png`
  - `header-experience-progress-screenshots/after-header-experience-40-percent-desktop.png`
- 웹 모바일 viewport에서는 공통 헤더 배지가 의도적으로 숨겨지므로 별도 캡처 대상이 아니다. React Native 구현은 같은 크기·토큰으로 수정하고 모바일 typecheck와 패리티 검사로 확인했다.
