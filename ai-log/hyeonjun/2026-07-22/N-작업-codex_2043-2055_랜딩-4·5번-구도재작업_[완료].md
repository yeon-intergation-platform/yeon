## 확인한 문제

- 두 전경 모두 3:2 비율로 세로 면적이 커서 책상·모니터가 공통 배경을 덮는다.
- 전경이 이미지 영역 전체를 채워 돌출 경계가 약하고 완성 이미지처럼 보인다.
- 4번은 말풍선·모니터·인물의 시각 경쟁이 심하고, 5번은 모니터가 지나치게 커 인물과 의자 중심이 오른쪽으로 밀린다.

## 수정 방향

- 1·2·3번 전경과 유사한 약 2.3:1 와이드 비율로 다시 제작한다.
- 책상선은 전경 하단 20% 이내로 낮추고 공통 배경 노출 면적을 확보한다.
- 인물 머리와 전등만 헤더로 돌출하고 서비스 행동 소품은 이미지 영역 안에 둔다.
- 4번은 핵심 말풍선 3~4개만, 5번은 할 일 보드 중심으로 단순화한다.
- CSS 가짜 소품이나 배경 확장은 사용하지 않는다.

## 이미지 제작

- imagegen 편집 모드로 기존 커뮤니티·Today 원본 구도를 각각 재구성했다.
- 커뮤니티: 인물을 약 58% 지점에 두고 의자를 더 노출했으며, 모니터를 낮추고 핵심 UI 4개만 남겼다.
- Today: 인물과 할 일 보드를 중앙 주제로 두고 모니터를 좌측으로 축소했으며, 책상선을 낮췄다.
- 두 에셋 모두 단색 크로마 배경에서 생성한 뒤 공식 배경 제거 도구로 투명 WebP 전경을 만들었다.
