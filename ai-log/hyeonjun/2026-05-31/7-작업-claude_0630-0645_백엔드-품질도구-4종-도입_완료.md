## ArchUnit 규칙 (`LayeredArchitectureTest`)

하드(통과): 컨트롤러→리포지토리 직접참조 금지, 리포지토리→컨트롤러 의존 금지, 서비스→컨트롤러 의존 금지, @RestController는 ..controller.. 패키지.
동결(FreezingArchRule): 도메인 슬라이스 순환 — **기존 `credential_auth ↔ root_auth` 순환을 베이스라인으로 기록**(`archunit_store/`)하고 신규 순환만 실패. (ArchUnit이 발견한 실제 기존 부채; 즉시 수정은 범위 밖이라 동결로 보호.)

- `archunit.properties`: `freeze.store.default.allowStoreCreation=true`.
- 비요청 규칙(리포지토리→서비스)은 `FileKind` enum 위치로 인한 양성 위반이라 제외.

## 후속 권장 (이번 범위 밖, 사용자 확인 후)

- **requirements.yaml ↔ 시나리오 태그 추적성 검사기**(사용자가 "제일 중요"로 강조): Karate feature에 `@REQ-xxx` 태그 부여 + `requirements.yaml` 원장 + Gradle 태스크로 (a) feature의 REQ ID가 yaml에 존재, (b) P0 요구사항 테스트 누락 없음, (c) 삭제된 ID 미참조, (d) yaml의 테스트 경로 실존을 검사. Jira/Xray 없이 레포 내 추적성 확보.
- **리포트 노출**: Karate HTML 리포트(`target/karate-reports`)와 JaCoCo/REST Docs 산출물을 CI artifact로 업로드.
- **JaCoCo 목표**: 전체 % 강제 대신 핵심 도메인/P0 코드 테스트 누락 감지 위주(`jacocoTestCoverageVerification`에 패키지 단위 룰).
