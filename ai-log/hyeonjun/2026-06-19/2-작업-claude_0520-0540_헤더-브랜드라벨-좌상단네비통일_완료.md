# 헤더 브랜드 라벨 서비스별 분리 + 좌상단 "한 단계 위" 네비게이션 통일

## 배경

- 모든 서비스 헤더 좌상단 브랜드가 동일하게 "YEON"이고 클릭 시 무조건 `href="/"`.
- subdomain 운영에서 서비스 홈(card.yeon.world/)에서 좌상단을 눌러도 자기 자신에 머물러 플랫폼(yeon.world)으로 나갈 동선이 없었다.

## 변경

### 1. 브랜드 라벨 (서비스별)

- `apps/web/src/components/product-shell/product-header.tsx` `COMMON_HEADER_BRAND_LABELS`:
  타자="YEON 타자방", 카드="YEON 플래시카드", 커뮤니티="YEON 커뮤니티" (home="YEON" 유지)
- 콘텐츠: `public-content-data.ts` `PUBLIC_CONTENT_CHANNEL_CONFIG`에 `brandLabel` 필드 추가
  (뉴스="YEON 뉴스", 블로그="YEON 블로그", 고객지원="YEON 고객지원").

### 2. 좌상단 "한 단계 위" 원칙

- 신규 순수 함수 `resolveSectionBrandHref(basePath, pathname)` — `apps/web/src/lib/header-brand-nav.ts`.
  `usePathname()`이 subdomain에선 prefix 없는 경로(`/decks`), root/dev에선 prefix 포함(`/card-service/decks`)을 주는 점을 이용해 **host 없이 pathname만으로** 판별.
  - 하위 화면 → 서비스(채널) 홈
  - 서비스 홈 → 플랫폼(yeon.world, subdomain은 `PLATFORM_HOME_HREF` 절대 URL)
- `CommonProductHeader`: `useYeonPathname()` + `SERVICE_BASE_PATH` 매핑으로 `brandHref` 동적 계산.
- 콘텐츠: server 컴포넌트 `PublicContentShell`에서 pathname을 얻기 위해 브랜드 링크만 client 컴포넌트 `public-content-brand-link.tsx`로 분리.

## 남은 리스크

- subdomain 실제 동작(card.yeon.world/ → yeon.world 절대 이동)은 로컬 도메인 한계로 단위 테스트로만 보증. 운영 배포 후 1회 육안 확인 권장.
- 콘텐츠(뉴스/블로그/고객지원)는 유지보수 3종 밖이나 사용자 명시 지시로 포함.
- 3000 포트는 무관 프로젝트 점유 → yeon web은 3100으로 검증.
