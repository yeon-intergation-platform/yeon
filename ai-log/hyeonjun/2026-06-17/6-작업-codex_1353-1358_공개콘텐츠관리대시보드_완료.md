# 공개 콘텐츠 관리 대시보드

## 목표

- `/admin/content`에서 `support`, `news`, `blog` 공개 콘텐츠 현황을 읽기 전용으로 확인한다.
- 글 수정/삭제 없이 URL, sitemap, 서비스 분포, 출처 경로를 점검한다.

## 범위

- admin content 대시보드
- 채널별 content 목록 화면
- 공개 콘텐츠 registry 재사용

## 제외

- 본문 수정
- 글 삭제
- 예약 발행
- Spring CMS API
- 상담관리 콘텐츠

## 진행

- `/admin/content` 읽기 전용 대시보드 추가
- `/admin/content/[channel]` 채널별 목록 화면 추가
- 공개 콘텐츠 registry 기반 admin model 추가
- 회원/사용자 admin 상단에 공개 콘텐츠 링크 추가
- 상담관리 채널은 허용 채널에서 제외
