# 작업 로그 (2026-05-17)

## 주제

카드 서비스 모바일에서 웹과 동일하게 비회원 게스트 모드로도 사용 가능하게 전환하고
코드 유지보수성 정리

## 차수 1

- 작업내용: 카드 서비스 모바일 3개 화면(목록/상세/학습)에서 세션 검사 실패 시 블록처리 제거하고
  `resolveCardServiceSession`를 공통 세션 해석기로 분리해 `guest`/`server` 모드를 명시적으로 관리
- 논의 필요: 없음
- 선택지: (1) 화면별 개별 토큰 체크 유지, (2) 공통 세션 상태/모드 함수 도입
- 추천: 공통 세션 도입 + 모드 기반 쿼리키 재설계
- 사용자 방향: 유지보수 우선으로 공통 세션 도입

## 차수 2

- 작업내용: 게스트 전용 카드/덱 영속화를 위한 localStorage/SecureStore 기반 저장 유틸(`storage.ts`) 추가,
  카드/덱 조회·생성·수정·삭제·복습을 서버/게스트 모드로 분기
- 논의 필요: 로컬 데이터 마이그레이션 정책(회원 로그인 동기화 충돌 시 정책)
- 선택지: (1) 회원 전용만 유지(현재 동작), (2) 게스트 데이터 임시 저장 제공(요청 반영)
- 추천: 게스트 임시 저장 제공
- 사용자 방향: 2번 (비회원 사용 가능)

## 차수 3

- 작업내용: `pnpm --filter @yeon/mobile typecheck`, `pnpm --filter @yeon/mobile lint`로 검증
- 논의 필요: 없음
- 선택지: 타입체크 생략, 린트 생략
- 추천: 최소 체크 둘 다 실행
- 사용자 방향: 실행

## 검증

- `pnpm --filter @yeon/mobile typecheck` ✅
- `pnpm --filter @yeon/mobile lint` ✅
