- 날짜: 2026-05-12
- 작성자: codex
- 상태: 작업중
- 목적: main push 후 GitHub Actions가 성공으로 표시됐지만 실제 deploy job이 skipped되어 운영 서버에 타자방 로비 화면이 반영되지 않은 문제 수정

## 확인한 원인

- run `25690024972`는 success였지만 `build_*`, `publish_*`, `deploy_production`이 모두 skipped.
- `detect_changes`에서 `github.event.before` 커밋이 shallow checkout에 없어 `fatal: bad object`가 발생.
- 기존 `git rev-parse <sha>` 검사는 객체 존재를 보장하지 못해 fallback이 작동하지 않았고, 결과적으로 `web=false`, `race=false`, `backend=false`가 출력됨.

## 작업 계획

- `.github/workflows/docker-image.yml`의 `detect_changes` checkout에서 이전 커밋을 안정적으로 확보한다.
- 이전 커밋 존재 검사를 `git cat-file -e <sha>^{commit}` 기반으로 바꿔 객체가 없으면 전체 배포 대상으로 fallback한다.
- 검증 후 branch → PR(main) → merge로 반영한다.
