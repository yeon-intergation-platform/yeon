# 작업 로그: 배포 릴리즈 버전관리 구축

- 날짜: 2026-05-12
- 작성자: codex
- 상태: 작업중
- 목적: 운영 배포 산출물을 GitHub Release로 추적하고 에이전트가 배포 완료 전 release/운영 검증을 확인하도록 SSOT 구축

## 작업 계획

- GitHub Release 기반 최소 버전관리 규칙 문서화
- Docker 배포 workflow에 release 생성 job 추가
- 배포 이미지 참조를 `sha-<short-sha>`로 고정
- concurrency/always 조건으로 취소된 산출물이 배포로 이어지는 위험 보정

## 구현 내용

- root `package.json` version `0.0.0`을 제품 버전 SSOT로 명시.
- `pnpm version:bump -- <major|minor|patch>` 및 `pnpm release:verify -- vX.Y.Z` 스크립트 추가.
- `v*.*.*` tag push 시 GitHub Release를 생성하는 workflow 추가.
- 에이전트 하네스 규칙을 `AGENTS.md`와 `docs/agent-rules/deployment-versioning.md`에 추가.
- 운영 배포 workflow의 main concurrency cancel 방지, publish 성공 조건 확인, `sha-<short-sha>` 이미지 배포를 추가.

## 검증

- `pnpm release:verify -- v0.0.0` 통과.
- 임시 디렉터리에서 `node scripts/bump-version.mjs -- patch` 실행 후 `0.0.1` bump 확인.
- `git diff --check` 통과.
- `pnpm exec prettier --check ...` 통과.
- `bash bin/sync-skills.sh --check` 통과.
- `bash bin/verify-ssot.sh --project-only` 통과(도구 출력상 프로젝트 git 저장소 검사는 건너뜀).
