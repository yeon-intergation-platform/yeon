# 작업 로그: 커뮤니티 진입점 단일화 및 /community 채팅/피드 구성

## 작업내용
- `/community`를 채팅 진입의 단일 루트로 정리하고 홈 플랫폼 서비스 목록에 신규 서비스를 반영
- `/community` 페이지에서 실시간 채팅과 커뮤니티 글/댓글 인터페이스 배치
- 카드 타이핑 서비스 화면에 채팅 위젯을 동일 API로 임베드할 수 있도록 레이아웃 레벨에서 위젯을 추가할 수 있는 형태로 작업 준비

## 논의 필요
- `/community` 채팅에서 방 진입 정책을 `기본 방 자동 선택`으로 둘지, 사용자 선택형 목록을 기본으로 둘지
- 카드/타이핑 서비스 위젯의 기본 노출 위치(접기형/항상 노출)와 모바일 노출 방식

## 선택지
- 옵션 A: `/community` 및 서비스 내부 위젯 모두 OTP 기반 채팅 세션을 한 번 로그인하면 재사용하고, 방 선택은 드롭다운/목록으로 노출
- 옵션 B: 위젯은 최근 메시지 미리보기만 보여주고 별도 페이지로만 메시지 작성 가능
- 옵션 C: 각 화면별 전용 채팅 API 연동을 두어 역할 분리
- 추천: 옵션 A(기능 재사용성, 유지보수 측면에서 일관)

## 사용자 방향
- 추천 옵션 기준으로 진행

## 상태
- 완료

## 남은 항목
- 위젯/피드 컴포넌트 실제 구현 및 빌드/배포 전 검증 완료
- 머지 요청 반영 완료(현재 PR 진행 중)
