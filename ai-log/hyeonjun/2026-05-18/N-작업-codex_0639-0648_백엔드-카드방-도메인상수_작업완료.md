# 백엔드 카드방 도메인 상수화 작업완료

## 목표

- JPA Entity는 도입하지 않고 카드방 백엔드의 하드코딩 문자열을 도메인 관리 값으로 이동한다.

## 변경

- `card_rooms.domain`에 상태/역할/결과/가시성/메시지 타입/ID prefix/문구/오류/텍스트 규칙을 추가했다.
- `CardRoomService`의 상태 전이와 권한 분기를 도메인 enum 기반으로 변경했다.
- `CardRoomRepository`는 도메인 타입을 받아 DB value로만 변환하도록 일부 메서드 시그니처를 정리했다.
- DTO와 DB에 노출되는 문자열 값은 기존 계약을 유지했다.

## 검증

- pre-commit의 전체 `pnpm typecheck`가 모바일 Node 타입 로딩 누락으로 막히지 않도록 `apps/mobile/tsconfig.json`에 node type entrypoint를 명시한다.

- `cd apps/backend && ./gradlew test --tests '*CardRoom*'` 통과
- `git diff --check` 통과
