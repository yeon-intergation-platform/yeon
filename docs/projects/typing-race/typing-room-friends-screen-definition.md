# 타자방 친구/DM/초대 화면정의서

작성일: 2026-05-12
위치: `docs/projects/typing-race/typing-room-friends-screen-definition.md`
상태: 구현 전 화면 정의
관련 기능정의서: `docs/projects/typing-race/typing-room-friends-feature-definition.md`

## 0. 디자인 원칙

- 타자 서비스의 기존 원칙을 유지한다: 흰 배경, 검정 CTA, 기능이 먼저 보이는 미니멀 UI.
- 친구 기능은 레이스/준비 흐름을 방해하지 않는 보조 액션이어야 한다.
- 장식적 소셜 UI보다 `친구 추가`, `대화`, `초대`의 상태가 즉시 이해되어야 한다.
- 타자방 진행 중에는 DM/초대 UI가 레이스 입력 영역을 가리지 않는다.

## 2. `TR-FRIEND-01` 타자방 참여자 카드 친구 액션

### 2-1. 위치

타자방 로비의 참여자 카드 내부 하단 또는 우상단 보조 메뉴.

현재 참여자 카드의 핵심 정보:

- 닉네임
- 캐릭터
- 방장/참가자
- 준비상태

추가 정보:

- 친구 상태 chip
- 보조 액션 버튼

### 2-2. 카드 상태

| 상태 | 표시 | 주 액션 | 보조 액션 |
| --- | --- | --- | --- |
| 자기 자신 | `나` | 없음 | 없음 |
| profile 없음 | `프로필 없음` | 없음 | `프로필 만들기` 안내 가능 |
| 비친구 | `아직 친구 아님` | `친구 추가` | 없음 |
| 추가 중 | `추가 중...` | disabled | 없음 |
| 친구 | `친구` | `대화` | 현재 방이면 `초대됨` 없음 |
| 차단/불가 | `추가 불가` | disabled | 없음 |

### 2-3. 레이아웃

```txt
┌────────────────────────┐
│ 준비완료        방장   │
│                        │
│        [캐릭터]        │
│                        │
│ 닉네임                 │
│ 친구 상태 chip         │
│ [친구 추가] [대화]     │
└────────────────────────┘
```

- 카드 높이는 기존 참여자 카드와 동일하게 유지한다.
- 버튼은 2개 이상 늘어나면 `더보기` 메뉴로 접는다.
- 친구 추가 성공 후 버튼은 즉시 `대화`로 전환한다.

### 2-4. 상호작용

#### 친구 추가 클릭

- optimistic: 버튼을 `추가 중...`으로 바꾼다.
- 성공: chip `친구`, CTA `대화`.
- 실패: chip 원복, 카드 내부 또는 상단 toast로 오류 표시.

#### 대화 클릭

- DM 패널을 연다.
- 타자방 연결은 유지한다.
- 입력 focus는 DM 패널에 이동하되, 패널 닫기 후 기존 타자방 조작으로 돌아올 수 있어야 한다.

## 3. `TR-FRIEND-02` 친구 관리 drawer

### 3-1. 진입

- 타자방 헤더의 `친구` 버튼
- 타자 서비스 로비 헤더의 `친구` 버튼
- 모바일에서는 bottom sheet로 표시 가능

### 3-2. 구조

```txt
┌ 친구 ─────────────────────┐
│ 검색: [닉네임 검색]        │
│                            │
│ 내 친구                   │
│ ┌ 친구 row ─────────────┐ │
│ │ 아바타 닉네임          │ │
│ │ 최근 대화/상태         │ │
│ │ [대화] [초대]          │ │
│ └───────────────────────┘ │
│                            │
│ 받은 초대                 │
│ ┌ 초대 row ─────────────┐ │
│ │ 타자방명 · 초대한 친구 │ │
│ │ [입장] [닫기]          │ │
│ └───────────────────────┘ │
└────────────────────────────┘
```

### 3-3. 탭 구성

MVP 권장:

1. `친구`
2. `받은 초대`

후속 확장:

- `추천`
- `차단`
- `요청`은 즉시 친구 정책에서는 숨김 또는 deprecated.

### 3-4. Empty state

| 조건 | 문구 | CTA |
| --- | --- | --- |
| 친구 없음 | `아직 친구가 없어요. 타자방에서 함께 연습한 사람을 친구로 추가해보세요.` | 없음 |
| 초대 없음 | `받은 타자방 초대가 없어요.` | 없음 |
| profile 없음 | `친구 기능을 사용하려면 커뮤니티 프로필이 필요해요.` | `프로필 만들기` |

## 4. `TR-FRIEND-03` 친구 초대 모달

### 4-1. 진입

타자방 로비 헤더 `초대` 버튼 클릭.

기존 `초대 링크 복사`는 모달 하단 보조 액션으로 이동한다.

### 4-2. 구조

```txt
┌ 친구 초대 ─────────────────────────┐
│ 현재 방: 오늘의 타자방 · 2/4       │
│ 검색 [친구 이름 검색]              │
│                                    │
│ □ 민지        최근 대화 3분 전     │
│ □ 준호        온라인 상태 미확인    │
│ □ 수아        이미 초대됨           │
│                                    │
│ [초대 링크 복사]   [2명 초대하기]  │
└────────────────────────────────────┘
```

### 4-3. 선택 규칙

- 친구만 목록에 표시한다.
- 이미 현재 방에 있는 친구는 disabled + `참여중` 표시.
- 같은 방 pending 초대가 있는 친구는 disabled 또는 `다시 보내기` 정책 중 하나를 선택한다.
- MVP 추천: `이미 초대됨` disabled.

### 4-4. 방 상태별 노출

| 방 상태 | 친구 초대 | 링크 복사 | 문구 |
| --- | --- | --- | --- |
| 대기중 | 가능 | 가능 | 없음 |
| 시작중 | 불가 | 불가 | `이미 시작된 방에는 초대할 수 없어요.` |
| 진행중 | 불가 | 불가 | `진행 중인 방에는 초대할 수 없어요.` |
| 종료됨 | 불가 | 불가 | `종료된 방입니다.` |

### 4-5. 성공/실패 피드백

- 성공: `친구에게 초대를 보냈어요.`
- 부분 성공: `일부 친구에게만 초대를 보냈어요.` 실패 대상 row에 이유 표시.
- 실패: `초대를 보내지 못했어요. 잠시 후 다시 시도해주세요.`

## 5. `TR-FRIEND-04` 친구 DM 패널

### 5-1. 위치

- 데스크톱: 우측 drawer. 기존 로비 채팅 영역이 있으면 탭으로 분리 가능.
- 모바일: full-height bottom sheet.

### 5-2. 구조

```txt
┌ 민지와 대화 ────────────────┐
│ 친구 · 타자방에서 만남       │
│                              │
│ 메시지 리스트                │
│                              │
│ [메시지 입력................] │
└──────────────────────────────┘
```

### 5-3. 로비 채팅과 DM 구분

| 구분 | 로비 채팅 | 친구 DM |
| --- | --- | --- |
| 대상 | 현재 방 전체 | 친구 1명 |
| 저장 | race-server lobby message 또는 후속 정책 | Spring chat-service message |
| 지속성 | 방 생명주기 중심 | 대화방 지속 |
| 표시 위치 | 타자방 우측/하단 | DM drawer |

### 5-4. 상태

- loading: `대화방을 여는 중...`
- empty: `아직 대화가 없어요. 첫 메시지를 보내보세요.`
- error: Spring 오류 메시지 표시
- disconnected typing room: DM은 유지 가능하되 타자방 오류와 시각적으로 분리

## 6. `TR-FRIEND-05` 초대함

### 6-1. 위치

- 타자방 목록 상단 compact card
- 친구 drawer `받은 초대` 탭

### 6-2. 구조

```txt
┌ 받은 초대 ─────────────────┐
│ 민지가 초대한 타자방        │
│ 오늘의 타자방 · 2/4 · 08:12 만료 │
│ [입장]                     │
└────────────────────────────┘
```

### 6-3. 동작

- `입장` 클릭 → Spring 초대 accept → `/typing-service/rooms/{roomId}` 이동.
- 이동 후 race-server 입장 실패 시 기존 방 오류 화면을 보여준다.
- 만료된 초대는 목록에서 제거한다.

## 7. 반응형 기준

### 데스크톱

- 타자방 본문 레이아웃은 기존 설정/참여자/채팅 구조 유지.
- 친구 drawer/DM panel은 오른쪽 overlay로 열고 본문 width를 갑자기 밀지 않는다.
- 초대 모달은 최대 520px.

### 모바일

- 친구 관리/DM/초대는 bottom sheet.
- 참가자 카드 액션은 버튼 1개 + 더보기 메뉴로 축약한다.
- 레이스 중에는 DM 패널 진입을 숨기거나 결과/로비 상태에서만 허용한다.

## 8. 접근성

- 친구 추가/대화/초대 버튼은 `aria-label`에 대상 닉네임 포함.
- modal/drawer는 focus trap과 ESC 닫기를 지원한다.
- 성공/실패 피드백은 toast만 의존하지 말고 row 상태도 갱신한다.
- 색상 chip에는 텍스트 라벨을 함께 둔다.

- friends overview

### DM panel

- chat room summary
- chat messages

## 11. 화면 완료 기준

- 타자방 participant card에서 친구 상태가 자기 자신/비친구/친구/profile 없음으로 명확히 보인다.
- 비친구에게 `친구 추가`를 누르면 성공 후 같은 자리에서 `대화`로 바뀐다.
- 친구 drawer에서 친구 목록과 받은 초대를 확인할 수 있다.
- 초대 모달에서 친구를 선택해 현재 방에 초대할 수 있다.
- DM panel을 열어도 타자방 로비 연결이 끊기지 않는다.
