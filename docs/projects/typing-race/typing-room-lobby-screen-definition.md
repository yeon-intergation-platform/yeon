# TYPING_ROOM_LOBBY 화면정의서 / 기능정의서

작성일: 2026-05-12  
위치: `docs/projects/typing-race/typing-room-lobby-screen-definition.md`

이 문서는 타자방 로비(`TYPING_ROOM_LOBBY`)의 실행 기준을 담는다.

## 1. 핵심

- 화면명: YEON 타자방 로비
- 상위 화면: 타자방 생성/목록/초대 진입
- 다음 화면: 타자 레이스 진행 화면
- 핵심 기능: 설정 확인/수정, 참여자 준비 상태, 채팅, 초대, 시작
- 레이아웃 기준: PC 3단(설정/참여자/채팅), 모바일 수직 스택

### Q. 완주 모드/다른 모드

- 현재 운영 가능한 모드는 2개다.
  - `완주 모드` (`TYPING_ROOM_MODE.FINISH`)
  - `시간 제한` (`TYPING_ROOM_MODE.TIME_LIMIT`)
- `완주 모드`(FINISH)는 **문장을 끝까지 완주한 판 기준으로 진행**하는 기본 모드다.
- `시간 제한` 모드(time-limit)는 **시간을 다 쓰면 판이 종료**되는 확장 모드이다.
- 현재 PRD 기준은 기본 화면/권한/카운트다운 흐름을 우선 `완주` 기준으로 구현하고, `시간 제한`은 라벨 및 설정 값 동기화만 우선 노출한다.

## 2. 레이아웃

- 헤더: 화면명, 상태, 공개여부, 인원, 방이름, 방코드, 초대/나가기
- 가운데: 참여자 카드 + 빈자리
- 좌측: 방 설정
- 우측: 채팅
- 하단: 현재 설정 요약 + 시작/준비 액션
- 긴 안내문/학습용 문구는 상시 노출하지 않는다.

## 3. 화면 요소 정의

### 3-1. 설정 영역

수정 가능(방장) / 조회 전용(참가자)

- 공개 여부: 공개 / 비공개
- 최대 인원: 2 / 3 / 4
- 언어: 한글 / 영어 / 코드
- 문장 길이: 짧은 문장 / 긴 문장
- 난이도: 쉬움 / 보통 / 어려움
- 판 수: 1 / 3 / 5
- 진행 방식: 완주 모드 / 시간 제한
- 덱: 기본 / 선택

### 3-2. 참여자 영역

- 방장/참여자 역할 라벨
- 준비 상태 표시: 준비완료/대기중
- 현재 인원 + 빈자리 슬롯(최대 인원 기준)
- 참가자 카드에서 친구 추가 보조 액션

### 3-3. 채팅 영역

- 최근 메시지 표시 + 전송 입력
- 텍스트 전송 실패/제한 처리
- 시스템 메시지:
  - 입장/퇴장
  - 준비 완료
  - 설정 변경

### 3-4. 하단 액션

- 방장: 시작하기
- 참가자: 준비하기 / 준비취소
- 방 설정 요약 텍스트: 언어/길이/난이도/판수/진행방식/덱

## 4. 권한 매트릭스

### 방장

- 설정 수정 가능
- 초대/채팅/친구추가/시작 가능
- 자동 준비완료(버튼 없음)

### 참가자

- 설정 보기만 가능
- 준비하기/준비취소 가능
- 시작하기 불가
- 나가기 가능

## 5. 주요 기능 정의 (F-01 ~ F-10)

### F-01. 로비 진입

- 입장 경로: 방 생성 완료, 초대 링크, 공개방 목록
- 입장 실패:
  - 시작중/진행중/종료됨: `이미 시작된 방입니다.`
  - 정원 초과: `방이 가득 찼습니다.`
  - 미존재: `존재하지 않는 방입니다.`

### F-02. 방 설정 수정

- 방장만 수정 가능
- 상태가 대기중일 때만 수정
- 진행중 수정 불가
- 참가자는 읽기 전용
- 최대 인원은 현재 참가자 수보다 작게 변경 불가

### F-03. 참여자 목록 표시

- 닉네임 + 역할 + 준비 상태 표시
- 입/퇴장 이벤트 실시간 반영
- 빈자리 슬롯은 최대 인원 기준으로 렌더링

### F-04. 준비하기 / 준비 취소

- 참가자 준비: 준비하기/준비취소 토글
- 방장은 자동 준비완료
- 설정 변경 시 정책 결정에 따라 참가자 준비 상태 초기화 적용(권장: 초기화)

### F-05. 시작하기

- 시작자격:
  - 방장 + 대기중 + 모든 참가자 준비완료 + 유효 설정
- 버튼 상태: 미충족 비활성 / 충족 활성 / 요청중 로딩
- 시작 시 3→2→1→START 카운트다운 후 진행 화면 동시 진입

### F-06. 초대 링크 복사

- 대상: 모든 사용자
- 성공 토스트: `초대 링크가 복사되었습니다.`
- 실패 토스트: `링크를 복사할 수 없습니다.`

### F-07. 채팅

- 실시간 대화 (방 참가자만)
- 빈 메시지 전송 불가
- 메시지 길이 제한(권장 500자)
- 전송 실패 상태 표시

### F-08. 친구 추가

- 상세 정책은 `docs/projects/typing-race/typing-room-friends-feature-definition.md`와 `docs/projects/typing-race/typing-room-friends-screen-definition.md`를 따른다.
- 참여자 카드의 보조 액션으로 노출한다.
- 자기 자신은 비노출/비활성 처리한다.
- 신규 정책은 요청 대기/수락이 아니라 `친구 추가` 액션 1회로 즉시 친구 관계를 만든다.
- 친구 관계/DM/초대의 백엔드 source of truth는 Spring(`apps/backend`)이다. Next.js는 신규 백엔드 소유자가 아니다.

### F-09. 나가기

- 참가자: 로비에서 제거
- 방장: MVP에서는 방을 종료하고 남은 참가자 연결을 정리한다.
- 마지막 사용자: 방 종료 상태로 처리한다.

### F-10. 진행중 방 입장 제한

- 상태 `시작중`, `진행중`, `종료됨`은 입장 차단
- 안내 문구: `이미 시작된 방입니다.`

## 6. 상태 정의

### 방 상태

- 대기중: 입장/설정/준비/채팅 허용
- 시작중: 카운트다운 진행
- 진행중: 타자 진행
- 종료됨: 방 종료

### 사용자 상태

- 방장 / 참가자 / 준비전 / 준비완료 / 빈자리

## 7. 버튼 정의

| 버튼      | 대상   | 위치        | 조건           | 동작            |
| --------- | ------ | ----------- | -------------- | --------------- |
| 시작하기  | 방장   | 하단        | 모든 준비 완료 | 카운트다운 시작 |
| 준비하기  | 참가자 | 하단        | 대기중         | 준비완료 전환   |
| 준비 취소 | 참가자 | 하단        | 준비완료       | 준비전 전환     |
| 초대      | 전체   | 헤더        | 대기중         | 링크 복사       |
| 나가기    | 전체   | 헤더        | 항상           | 로비 퇴장       |
| 전송      | 전체   | 채팅        | 메시지 입력됨  | 메시지 전송     |
| 친구 추가 | 전체   | 참여자 메뉴 | 친구 아님      | 즉시 친구 추가 |

## 8. 데이터 정의

- Room
  - roomId, roomCode, roomName, status, visibility, currentCount, maxCount, language, sentenceLength, difficulty, roundCount, playMode, deckId, deckName, hostUserId
- Participant
  - userId, nickname, role, readyStatus, isFriend
- ChatMessage
  - messageId, roomId, senderId, senderNickname, messageType, content, createdAt

## 9. 운영 노출 금지 문구

- 시작 준비 / 시작 준비 문구 일괄 제거
- 방장 자동 준비 완료 안내 문구 상시 노출 금지
- MVP 규칙, 입장 제한 규칙은 정책문구로 별도 안내하지 않고 로직 및 토스트로 처리

## 10. 문서 고정 규칙

- 진행 방식 기본값/추가 모드(`시간 제한`)는 코드 상수와 동기화
- 정책/요구사항 변경 시 본 문서 우선 수정 후 `typing-room-lobby-backlog.md`에 반영할 것
- 방장 퇴장 정책은 현재 MVP 기준 `방 종료`가 source of truth다.
