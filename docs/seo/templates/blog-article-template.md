# Blog 문서 템플릿

## 사용 범위

`blog.yeon.world`의 engineering, product, devlog, essay 글에 사용한다. 공식 공지와 문제 해결 문서는 각각 news와 support에 둔다.

## Engineering 글 구조

```md
# 구체적인 기술 결정 제목

## 문제

어떤 기술 문제나 운영 문제가 있었는지 적는다.

## 선택지

- 선택지 A:
- 선택지 B:
- 선택지 C:

## 결정

왜 이 방식을 선택했는지 적는다.

## 구현 근거

- repo 경로
- 주요 함수 또는 모듈
- 테스트 또는 검증 방법

## 남은 제약

아직 해결하지 않은 부분과 다음 개선 방향을 적는다.
```

## Product/devlog/essay 글 구조

```md
# 사용자가 이해할 수 있는 제품 맥락 제목

## 배경

왜 이 주제를 다루는지 적는다.

## 만든 것 또는 배운 것

실제 기능, 운영 결정, 실패, 배운 점을 적는다.

## 사용자에게 주는 의미

서비스 사용자나 운영자에게 어떤 의미가 있는지 적는다.

## 관련 링크

- support 문서
- news 공지
- 제품 URL
```

## 발행 전 체크

- [ ] engineering/product/devlog/essay 중 하나로 분류된다.
- [ ] 실제 기능, 코드, 운영 정책 중 하나와 연결된다.
- [ ] 관련 support 또는 news 링크가 있다.
- [ ] 홍보 문장만 있고 근거가 없는 글이 아니다.
