# 공개 콘텐츠 제목 작성 원칙

이 문서는 `support.yeon.world`, `news.yeon.world`, `blog.yeon.world`에 발행할 글의 제목 기준이다. 상담 워크스페이스는 유지보수 동결 대상이므로 제목 후보에도 넣지 않는다.

## 공통 원칙

- 제목은 검색자가 실제로 입력할 만한 문장에 가깝게 쓴다.
- 서비스명, 공개 host, 제품명, 문제 상황 중 하나를 반드시 드러낸다.
- `가이드`, `오류 해결`, `권한 안내`, `업데이트 소식`처럼 맥락이 빠진 제목은 쓰지 않는다.
- 한 제목은 하나의 검색 의도나 운영 사실만 다룬다.
- 과장된 마케팅 문장보다 사용자가 얻을 결과를 먼저 보이게 한다.

## Support 제목

`support.yeon.world` 제목은 `서비스 단서 + 행동/문제 + 해결 방식` 구조를 기본으로 한다.

좋은 제목:

- 디스코드 서버에 NEXA AI 봇 추가하는 방법
- NEXA 봇이 응답하지 않을 때 확인할 5가지
- typing.yeon.world에서 타자연습을 시작하는 방법
- card.yeon.world에서 플래시카드 덱을 만드는 방법
- community.yeon.world에서 커뮤니티 글을 쓰는 방법

피할 제목:

- NEXA 가이드
- 오류 해결
- 권한 안내
- 사용법

## News 제목

`news.yeon.world` 제목은 공식 사실, 대상 서비스, 변경 성격을 함께 보여준다.

좋은 제목:

- YEON support.yeon.world 오픈 안내
- NEXA 디스코드 권한 안내 문서를 공개했습니다
- 타자연습 기본 사용 가이드를 추가했습니다

피할 제목:

- 업데이트 소식
- 새 소식
- 공지

## Blog 제목

`blog.yeon.world` 제목은 기술 선택, 제품 판단, 운영 이유가 드러나야 한다.

좋은 제목:

- 왜 support, news, blog를 분리했는가
- Search Console과 sitemap을 운영 절차에 넣는 이유
- 타자 서비스에서 실시간 서버가 필요한 이유

피할 제목:

- 개발 이야기
- 회고
- 기술 글

## 발행 전 확인

- 제목만 봐도 어떤 서비스 또는 주제를 다루는지 알 수 있다.
- 제목만 봐도 사용자가 얻을 답이나 운영 사실을 예상할 수 있다.
- 같은 channel 안에 너무 비슷한 제목이 없다.
- 제목과 본문 H1, description, slug가 서로 다른 주제를 말하지 않는다.
