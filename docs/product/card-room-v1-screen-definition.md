# 카드방 v1 화면 정의서

## 목적

카드방은 타자방처럼 캐릭터를 고른 뒤, 한 명이 앞면을 보고 답을 말하고 다른 한 명이 뒷면을 확인해 `OK` 또는 `포기` 결과를 확정하는 채팅 기반 암기 검증 화면이다. 이번 v1은 실시간 서버 연동 전 화면/권한/상태 규칙을 고정하는 정의서다.

## 역할

| 역할                     | 설명                                         | 볼 수 있는 카드 정보                    | 주요 권한                                   |
| ------------------------ | -------------------------------------------- | --------------------------------------- | ------------------------------------------- |
| 외우는 사람(`MEMORIZER`) | 앞면을 보고 채팅/음성으로 답을 말하는 사용자 | 기본은 앞면만. `GIVE_UP` 이후 뒷면 공개 | `포기`, 결과 확정 후 `다음 카드`            |
| 봐주는 사람(`CHECKER`)   | 뒷면을 확인해 정답 여부를 판정하는 사용자    | 앞면과 로컬로 뒤집은 뒷면               | 카드 뒤집기, `OK`, 결과 확정 후 `다음 카드` |

## 캐릭터 선택

- 카드방은 타자방의 캐릭터 레지스트리와 프로필 선택 UI를 재사용한다.
- `/card-service` 홈에서 닉네임과 캐릭터를 선택하고, 로비/생성/학습 화면에 현재 선택 캐릭터를 표시한다.
- v1 화면 골격에서는 캐릭터 선택을 로컬 프로필 상태로만 유지한다. 방 참가자 캐릭터 서버 저장/실시간 동기화는 후속 차수에서 정의한다.
- 학습 화면은 내 캐릭터와 상대 역할 캐릭터를 함께 보여줘 “누가 외우고/봐주는지”를 즉시 인지하게 한다.

## 상태 모델

```ts
type ParticipantRole = "MEMORIZER" | "CHECKER";
type CardRoomPhase =
  | "ANSWERING"
  | "PASSED"
  | "GIVEN_UP"
  | "REVEALED"
  | "FINISHED";
type CardResult = "OK" | "GIVE_UP";
```

- MVP 저장 대상 결과는 `OK`, `GIVE_UP`만 둔다.
- `HINTED_OK`는 힌트 기능과 함께 후속 확장으로 둔다.
- 봐주는 사람의 카드 뒤집기 여부는 로컬 UI 상태다. 외우는 사람 화면에는 동기화하지 않는다.
- `PASSED`, `GIVEN_UP` 이후에만 `다음 카드`를 노출한다.
- 마지막 카드에서 다음을 누르면 `FINISHED`로 전환한다.

## 버튼/권한 규칙

| phase       | 외우는 사람                            | 봐주는 사람                             |
| ----------- | -------------------------------------- | --------------------------------------- |
| `ANSWERING` | 앞면 확인, 채팅 입력, `포기` 가능      | 카드 클릭으로 뒷면 로컬 확인, `OK` 가능 |
| `PASSED`    | 뒷면은 공개하지 않음, `다음 카드` 가능 | `OK` 완료 상태, `다음 카드` 가능        |
| `GIVEN_UP`  | 뒷면 공개, `다음 카드` 가능            | 포기 결과 확인, `다음 카드` 가능        |
| `REVEALED`  | v1에서는 직접 진입 없음                | 후속 정답 보기 확장 상태                |
| `FINISHED`  | 결과 요약 확인                         | 결과 요약 확인                          |

## 화면 구조

### 데스크톱

- 전체는 흰 배경, 검정 CTA, 회색 보더를 사용한다.
- 학습 진행 화면은 왼쪽 카드 35%, 오른쪽 채팅 65% 비율을 기본으로 한다.
- 상단에는 카드 번호, 역할, 방 상태, 로비 복귀 CTA를 둔다.
- 왼쪽 카드 영역은 앞면/뒷면/결과 버튼을 담당한다.
- 오른쪽 채팅 영역은 답변 흐름과 시스템 메시지를 담당한다.

### 모바일

- 상단에는 현재 카드 번호와 역할을 항상 표시한다.
- 본문은 `카드`, `채팅` 탭으로 나눈다.
- 탭 전환 중에도 phase와 결과 버튼 규칙은 동일하게 유지한다.

## 라우트

| 라우트                         | 역할                                                          |
| ------------------------------ | ------------------------------------------------------------- |
| `/card-service`                | 캐릭터 선택 + 카드방 입장 / 내 덱 보기 / 새 덱 만들기 중심 홈 |
| `/card-service/decks`          | 기존 내 덱 목록, 게스트 병합, 새 덱 생성 화면                 |
| `/card-service/rooms`          | 카드방 로비. 검색/필터/방 만들기 CTA/빈 상태                  |
| `/card-service/rooms/new`      | 카드방 생성 진입 화면 골격                                    |
| `/card-service/rooms/[roomId]` | 학습 진행 화면 골격                                           |

## 후속 구현 경계

- 실시간 방 목록, 실시간 채팅, 참가자 동기화는 이번 차수에서 제외한다.
- OK/포기 저장 API는 Spring(`apps/backend`)에서 계약을 먼저 정의한다.
- Next.js route handler는 신규 도메인 쓰기 원천이 되지 않는다.
