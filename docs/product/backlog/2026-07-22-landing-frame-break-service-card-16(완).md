# 랜딩 YEON Today 카드 크기 정합성 수정

> 상태: 완료

## 1차 — 로그인 게이트 카드 래퍼 정합성 수정

- 작업내용: 프레임브레이크 카드가 링크 또는 로그인 게이트 버튼으로 렌더링돼도 동일한 패딩·정렬·본문 굵기를 사용하게 한다.
- 논의 필요: Today 이미지나 카드 내부 구조가 아니라 `YeonButton`의 기본 크기·정렬 스타일이 카드 wrapper에 남는 것이 원인이므로 공통 프레임브레이크 카드 경계에서 초기화해야 한다.
- 선택지:
  - A. Today 카드에만 별도 크기 CSS를 추가한다.
  - B. 프레임브레이크 카드 공통 wrapper가 요소 종류와 무관하게 동일한 레이아웃 계약을 갖게 한다.
- 추천: B. 인증 상태에 따라 `a`와 `button`이 전환돼도 시각적 크기가 달라지지 않게 한다.
- 사용자 방향: YEON Today 카드 크기를 다른 카드와 동일하게 수정한다.

## 2차 — 인증 상태 및 반응형 회귀 검증

- 작업내용: 게스트 상태의 Today와 공개 Discord 카드의 외곽·이미지 폭·높이·본문 굵기를 비교하고 375px·1040px·1228px·1440px에서 확인한다.
- 논의 필요: 버튼 기본 스타일을 제거하면서 focus ring, 클릭 동작, 로그인 모달 진입은 유지해야 한다.
- 선택지:
  - A. 육안 스크린샷만 비교한다.
  - B. Playwright 계산 스타일과 스크린샷을 함께 비교한다.
- 추천: B. padding·align-items·이미지 폭을 수치로 확인하고 로그인 모달도 직접 연다.
- 사용자 방향: localhost에서 즉시 확인 가능하게 기존 개발 서버를 유지한다.

## 완료 조건

- Today와 인접 카드의 외곽 폭 및 이미지 폭이 동일해진다.
- Today 설명문이 버튼 기본 굵기를 상속하지 않는다.
- 로그인 모달과 focus 상태가 유지된다.
- 모바일·데스크톱에서 가로 스크롤이 없다.

## 완료 결과

- 프레임브레이크 카드 공통 wrapper에 패딩 0, stretch 정렬, 상단 정렬, 본문 기본 굵기를 명시했다.
- 1228px에서 Today 카드 높이가 458.59px에서 442.59px로 정리되고 이미지 폭이 274.17px에서 364.67px로 확대돼 Discord 카드와 일치했다.
- 375px·1040px·1228px에서 Today와 인접 카드의 외곽 크기 및 이미지 크기가 동일하다.
- 1440px에서도 외곽 폭 424px, 이미지 폭 422px가 동일하며 설명 줄 수에 따른 자연스러운 세로 높이 차이만 남는다.
- Today 클릭 시 공통 로그인 모달이 정상적으로 열리고 모든 검증 뷰포트에서 가로 스크롤이 없다.
