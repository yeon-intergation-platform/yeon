# 공개 콘텐츠 CI 품질 실패 수정

## 1차

작업내용:

- PR #769 merge 후 main `Frontend Quality` 실패 원인을 확인한다.
- 공개 콘텐츠 coverage 테스트가 현재 registry article 수에서 파생되게 수정한다.
- blog essay 품질 테스트에서 지적된 실제 기능 연결 부족 글을 보강한다.
- 수정 후 실패한 테스트와 관련 검증을 다시 실행한다.

논의 필요:

- coverage 테스트에서 article 수를 고정값으로 둘지 registry 파생값으로 둘지.

선택지:

- 고정값: 콘텐츠 수가 바뀔 때마다 테스트를 갱신해야 한다.
- registry 파생값: 현재 registry와 리포트 계산의 정합성을 검증한다.
- 최소값: 성장 방향만 보지만 정확한 회귀를 덜 잡는다.

추천:

사용자 방향:
