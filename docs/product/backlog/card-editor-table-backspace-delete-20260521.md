# 카드 에디터 표 Backspace 삭제 UX 백로그 (2026-05-21)

## 배경

직접 작성 에디터에서 표는 우클릭 메뉴로 삭제할 수 있게 되었지만, 사용자는 키보드 중심 흐름에서도 표를 삭제할 수 있어야 한다. 단, 셀 내용을 편집하는 중 Backspace 한 번으로 표 전체가 사라지면 위험하므로 표/셀 범위 선택 상태에 한해 안전하게 처리한다.

## 1차

### 작업내용

- 표 전체 또는 복수 셀이 선택된 상태에서 Backspace/Delete 입력 시 표 전체를 삭제한다.
- 일반 셀 텍스트 편집 중 Backspace는 기존처럼 셀 내용만 편집되게 유지한다.
- 삭제 후 toolbar/overlay 상태를 갱신한다.

### 논의 필요

- 없음. 사용자가 Backspace 삭제를 요구했고, 실수 삭제 방지를 위해 선택 상태 기반으로 구현한다.

### 선택지

1. 표 안에서 Backspace를 누르면 항상 표 삭제
   - 실수 삭제 위험이 크다.
2. 표 전체/복수 셀 선택 상태에서만 Backspace/Delete로 표 삭제
   - 사용자 편의와 안전성을 같이 만족한다.
3. 우클릭 삭제만 유지
   - 키보드 사용 흐름을 충족하지 못한다.

### 추천

2번. 사용자는 키보드 삭제를 기대하지만, 일반 텍스트 편집과 표 삭제를 구분해야 한다.

### 사용자 방향

추천 기준으로 진행한다.

## 완료 기준

- 선택된 표/복수 셀에서 Backspace/Delete로 표가 삭제된다.
- 일반 셀 내부 Backspace는 표 삭제로 오작동하지 않는다.
- web lint/typecheck/build를 통과한다.
