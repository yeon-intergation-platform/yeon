# dev:all Spring 백엔드 환경 자동 연결

## 배경

로컬 `pnpm dev:all` 실행 시 Spring 백엔드가 8080 포트로 떠도, 웹 카드 서비스 Spring client는 기본값 8081을 바라본다. 8081에는 모바일/Expo dev 서버가 배정될 수 있어 카드 덱 목록/생성 요청이 엉뚱한 서버로 전달된다.

## 1차

### 작업내용

- `scripts/dev-all.mjs`의 서비스 포트 산정 결과를 web 프로세스 환경에 전달한다.
- web 프로세스에 `SPRING_BACKEND_BASE_URL=http://127.0.0.1:<backendPort>`를 자동 주입한다.
- Spring 내부 인증용 `SPRING_INTERNAL_TOKEN`을 web/backend 양쪽에 동일하게 주입한다.
- 사용자가 이미 `SPRING_INTERNAL_TOKEN`을 지정했다면 그 값을 우선한다.

### 논의 필요

- 로컬 기본 토큰을 고정 문자열로 둘지, 매 실행 랜덤으로 둘지 결정이 필요하다.

### 선택지

1. 로컬 기본 토큰 고정: 재시작/디버깅이 단순하다.
2. 매 실행 랜덤 토큰: 더 격리되지만 로그/재현이 번거롭다.

### 추천

- `pnpm dev:all`은 로컬 개발 편의성이 우선이므로 고정 기본 토큰을 사용하고, 운영/배포는 기존 환경변수 값을 우선한다.

### 사용자 방향

- 추천 기준으로 진행한다.
