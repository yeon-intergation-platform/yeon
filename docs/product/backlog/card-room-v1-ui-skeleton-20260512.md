# 카드방 v1 화면 정의서 + UI 골격

## 1차

### 작업내용

- 카드방 학습 진행 화면 정의서를 `docs/product/`에 추가한다.
- `/card-service`를 캐릭터 선택, 카드방 입장, 내 덱 보기, 새 덱 만들기 중심 홈으로 전환한다.
- 기존 덱 목록 홈은 `/card-service/decks`로 분리한다.
- `/card-service/rooms`, `/card-service/rooms/new`, `/card-service/rooms/[roomId]` 화면 골격을 추가한다.
- 실시간 서버 없이 클라이언트 상태 모델로 캐릭터 표시, 역할, 카드 phase, OK/포기/다음 카드 버튼 규칙을 확인 가능하게 만든다.

### 논의 필요

- 카드방 실시간 소유권과 참가자 캐릭터 동기화를 Spring REST/SSE로 시작할지 race-server 방 프로토콜로 시작할지 후속 차수에서 결정해야 한다.
- OK/GIVE_UP 저장 API 계약과 카드방 생성 API 계약은 별도 백로그가 필요하다.

### 선택지

1. 문서 + 화면 골격만 먼저 구현하고 서버 연동은 후속 차수로 분리한다.
2. 화면과 동시에 임시 API route를 추가한다.
3. 기존 덱 학습 화면에 카드방 UI를 섞어 넣는다.

### 추천

- 1번. Next.js가 신규 백엔드 역할을 맡지 않도록 하고, 화면 정의와 사용성 검증을 먼저 고정한다.

### 사용자 방향

- 문서 + 화면 골격 범위로 진행한다.
