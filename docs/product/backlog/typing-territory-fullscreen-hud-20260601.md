# 타자 점령전 전체 화면 HUD 개편 백로그

기준일: 2026-06-01

## 1차: 레이아웃 구조 전환

- 작업내용: `typing-service/territory` 화면을 `100vw`, `100vh` 기준 전체 화면 대전 게임 HUD로 재구성한다. 기존 `max-width` 카드형 레이아웃과 큰 좌우 여백을 제거한다.
- 논의 필요: 기존 타자 서비스 흰 배경 규칙과 이번 점령전 전용 다크 게임 UI 요구가 충돌한다.
- 선택지:
  - A. 기존 흰 배경 규칙 유지
  - B. 점령전 route에 한해 사용자 지시의 다크 네이비/모던 게임 HUD 적용
- 추천: B. 이번 요청은 명시적으로 기존 카드형 흰 화면을 금지하고 전체 화면 모던 게임 UI를 요구한다.
- 사용자 방향: 점령전은 전체 화면 모던 대전 HUD로 간다.

## 2차: 보드/팀/HUD 정보 구조

- 작업내용: 중앙 보드를 화면 주인공으로 배치하고, 상단 타이머/점수, 좌우 팀 패널, 하단 입력 컨트롤 바를 구성한다.
- 논의 필요: Phaser scene과 React fallback board 중 어느 쪽이 주 보드가 되어야 하는가.
- 선택지:
  - A. Phaser scene을 주 보드로 유지하고 React tile grid는 조작/접근성 보조로 유지
  - B. React tile grid를 주 보드로 만들고 Phaser를 제거
- 추천: A. 이전 후속 단계에서 Phaser 연출을 추가했으므로 주 시각 보드는 Phaser로 유지한다.
- 사용자 방향: Phaser 보드를 대형 중앙 보드로 유지한다.

## 3차: 모던 팔레트와 검증

- 작업내용: 원색/픽셀아트를 제거하고 muted red/blue, dark navy 배경, refined HUD 패널을 적용한다. typecheck/lint/build와 Playwright 시각 검증을 수행한다.
- 논의 필요: 운영 배포까지 진행할지 여부.
- 선택지:
  - A. 로컬 검증 후 PR만 생성
  - B. Yeon 정책대로 PR merge 후 main 배포 흐름까지 진행
- 추천: B. 저장소 파일 변경 시 main 배포가 프로젝트 정책이다.
- 사용자 방향: 구현 후 main 배포까지 진행한다.
