# 쥔장의 포트폴리오 사이트 구축

## 목표

- YEON 랜딩에 9번 서비스 `쥔장의 포트폴리오`를 추가한다.
- `portforlio.yeon.world`에서 최현준의 공개 포트폴리오, 이력서, GitHub, 블로그와 프로젝트 갤러리를 제공한다.
- 갤러리 이미지는 후속 작업에서 파일 경로만 연결할 수 있도록 콘텐츠와 이미지 소스를 분리한다.

## 범위 밖

- 갤러리 게시물 작성용 관리자 화면과 서버 저장소
- 방문자 인증, 댓글, 좋아요, 업로드 기능
- PDF 원본 내용 수정

## 1차수 — 공개 계약과 콘텐츠 원천 확정

### 작업내용

- 요청된 PDF 2종에서 공개 프로필 문구, GitHub, 블로그, 프로젝트 근거를 확인한다.
- `portforlio.yeon.world`와 내부 `/portfolio` 경로의 rewrite·canonical 계약을 정의한다.
- PDF 원본을 변경하지 않고 웹 정적 다운로드 자산으로 복사한다.

### 논의 필요

- 요청 호스트의 `portforlio` 철자를 그대로 서비스 계약으로 사용할지 여부

### 선택지

- A. 요청값 `portforlio.yeon.world`를 그대로 사용한다.
- B. 영문 표준 철자인 `portfolio.yeon.world`로 임의 변경한다.

### 추천

- A. 사용자가 지정한 공개 호스트를 정확히 보존하고 코드·SEO·문서에서 단일 값으로 관리한다.

### 사용자 방향

- 추천 기준으로 진행한다.

## 2차수 — 랜딩 카드와 포트폴리오 화면 구현

### 작업내용

- 플랫폼 서비스 목록에 9번 `쥔장의 포트폴리오` 카드를 추가한다.
- 공개 링크 4종(포트폴리오 PDF, 이력서 PDF, GitHub, 블로그)을 명확한 CTA로 제공한다.
- PDF의 실제 프로젝트를 바탕으로 제목 한 줄과 짧은 설명이 있는 반응형 갤러리 그리드를 구성한다.
- 갤러리 데이터의 선택적 `imageSrc`를 단일 콘텐츠 원천으로 두고, 이미지가 없을 때 의도된 대체 화면을 표시한다.

### 논의 필요

- 이미지가 준비되지 않은 초기 갤러리의 표현 방식

### 선택지

- A. 게시물을 숨긴다.
- B. 프로젝트 번호와 이미지 준비 상태를 표시하고 제목·설명은 바로 공개한다.

### 추천

- B. 콘텐츠와 그리드 구조를 먼저 검증할 수 있고, 이후 `imageSrc`만 채워도 레이아웃이 유지된다.

### 사용자 방향

- 추천 기준으로 진행한다.

## 3차수 — 라우팅·접근성·시각 검증

### 작업내용

- subdomain root/하위 경로/legacy path, canonical, sitemap, robots를 테스트한다.
- 다운로드 링크의 파일명·HTTP 응답, 외부 링크의 새 창 안전 속성, 갤러리 콘텐츠를 검증한다.
- 375px 모바일과 데스크톱에서 Playwright로 랜딩 및 포트폴리오 화면을 확인하고 스크린샷을 남긴다.
- lint, typecheck, 관련 단위 테스트, build, 저장소 SSOT 검증을 실행한다.

### 논의 필요

- 없음. 공개 정적 페이지의 최소 검증 범위로 진행한다.

### 선택지

- A. 정적 코드 검증만 수행한다.
- B. 단위 테스트와 실제 브라우저·다운로드 검증을 함께 수행한다.

### 추천

- B. 새 도메인 라우팅과 반응형 UI는 실제 요청 및 브라우저 증거가 필요하다.

### 사용자 방향

- 추천 기준으로 진행한다.

## 완료 조건

- YEON 랜딩에 9번째 포트폴리오 카드가 노출되고 올바른 경로로 진입한다.
- `portforlio.yeon.world`가 `/portfolio` 화면으로 연결되는 코드·SEO 계약이 테스트로 고정된다.
- 포트폴리오/이력서 PDF가 다운로드되고 GitHub/블로그 링크가 실제 공개 주소를 가리킨다.
- 이미지가 없는 프로젝트도 갤러리 카드로 읽을 수 있고 이후 이미지 경로만 추가할 수 있다.
- 데스크톱·모바일 시각 검증 및 저장소 필수 검증이 통과한다.
