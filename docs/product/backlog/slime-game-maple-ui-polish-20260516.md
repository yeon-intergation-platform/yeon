# 슬라임 게임 메이플풍 화면 개선 백로그 (2026-05-16)

## 1차

### 작업내용

- `/slime-game`의 현재 에셋 검수장형 흰 배경 레이아웃을 폐기하고, 사용자가 제시한 레퍼런스처럼 풀스크린 2D 횡스크롤 RPG 장면으로 재구성한다.
- 숲 배경, 다층 발판, 하단 스킬 슬롯, 좌측 캐릭터 HUD, 우측 미니맵/퀘스트, 코인/메뉴 UI를 한 화면 안에 배치한다.
- 슬라임 주인공이 전투 중인 장면이 즉시 보이도록 초기 구도와 캐릭터 스케일을 조정한다.
- 기존 이동/점프/사다리/공격/코인/포탈 상태 루프는 유지하되, 화면 표현을 게임 UI 중심으로 강화한다.

### 논의 필요

- 이번 차수에서 실제 신규 일러스트 생성까지 할지, 기존 에셋과 CSS/DOM 레이어링으로 먼저 레퍼런스에 가까운 게임 화면을 만들지 결정이 필요하다.

### 선택지

- A. 기존 에셋 + CSS 아트 + DOM 레이어링으로 즉시 playable 화면을 개선한다.
- B. 신규 배경/캐릭터 일러스트를 다시 생성해 에셋 파이프라인부터 재구성한다.

### 추천

- A. 지금 문제는 구조와 화면 밀도가 에셋 검수장처럼 보이는 것이므로, 먼저 한 화면짜리 실제 게임 HUD/스테이지로 고친 뒤 필요하면 B를 후속 차수로 진행한다.

### 사용자 방향

- 추천안 A 기준으로 진행.

### 완료 기록

- `/slime-game`을 흰 배경 에셋 검수장 레이아웃에서 풀스크린 2D RPG 스테이지/HUD 화면으로 교체했다.
- 좌측 캐릭터 HUD, 우측 미니맵/퀘스트, 하단 스킬/메뉴/EXP 바, 중앙 전투 구도, 숲 발판/장식/몬스터를 한 화면에 배치했다.
- `pnpm --filter @yeon/web lint`, `pnpm --filter @yeon/web typecheck`, `pnpm --filter @yeon/web build` 통과.
- `next start -p 3104` + system Chrome Playwright headless screenshot으로 `/slime-game` 화면 구도 확인.
