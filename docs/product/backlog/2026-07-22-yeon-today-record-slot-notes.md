# YEON Today 시간 블록 설명 백로그

## 목표

하루 기록의 각 시간 블록에 실제로 한 일을 구체적으로 적고, 기록된 설명을 타임라인 블록 안에서 짧게 확인할 수 있게 한다. 기존 Spring 소유 `note` 필드와 200자 검증을 그대로 사용한다.

## 1차 — 시간 블록 설명 입력·수정

- 작업내용: 활동이 기록된 시간 블록에 `설명 추가` 진입점을 제공하고, 타임라인 아래의 인라인 편집 영역에서 설명을 입력·수정·삭제할 수 있게 한다.
- 논의 필요: 설명 편집을 모달, 블록 내부 입력, 타임라인 아래 인라인 편집 중 어떤 방식으로 제공할지 결정한다.
- 선택지: 블록 내부 입력, 별도 모달, 선택 블록 아래 공용 인라인 편집 영역.
- 추천: 좁은 블록을 입력 UI로 과밀하게 만들지 않고 모바일 키보드에도 대응하기 쉬운 타임라인 아래 인라인 편집 영역을 사용한다.
- 사용자 방향: 시간 블록에 구체적인 설명을 적을 수 있게 한다.

## 2차 — 블록 내 설명 표시

- 작업내용: 저장된 설명을 활동명 아래에 최대 두 줄로 표시하고, 전체 설명은 편집 영역과 마우스 제목 속성에서 확인할 수 있게 한다.
- 논의 필요: 블록에서 노출할 줄 수와 글자 크기를 정한다.
- 선택지: 한 줄 말줄임, 두 줄 말줄임, 아이콘만 표시.
- 추천: 의미를 파악할 수 있으면서 블록 높이를 과도하게 늘리지 않는 두 줄 말줄임을 사용한다.
- 사용자 방향: 공간이 좁으므로 일부 글자만 보이더라도 블록 아래쪽에 표시한다.

## 3차 — 상태·반응형·접근성 검증

- 작업내용: 활동 변경 시 기존 설명 보존, 빈 설명 저장 시 제거, 삭제 시 편집 상태 정리, 키보드 포커스, 모바일 가로 넘침을 검증한다.
- 논의 필요: 없음.
- 선택지: 정적 검사만 수행, 실제 브라우저에서 입력·저장·재표시까지 검증.
- 추천: 실제 브라우저에서 서버 요청 본문과 화면 재표시까지 검증한다.
- 사용자 방향: 추천 기준으로 진행한다.

## 완료 조건

- 기록된 시간 블록마다 최대 200자의 설명을 저장·수정·제거할 수 있다.
- 활동을 바꿔도 기존 설명이 의도치 않게 사라지지 않는다.
- 저장된 설명이 활동명 아래 최대 두 줄로 표시된다.
- 설명 편집과 활동 재지정·기록 삭제 동작이 서로 충돌하지 않는다.
- 데스크톱과 모바일에서 가로 넘침 없이 사용할 수 있다.
