# 방 생명주기/생성 전이 안정화 백로그

## 1차

### 작업내용
- 타자방과 카드방의 자동삭제/정리 로직 위치를 비교한다.
- 카드방이 참가자 0명 이후에도 남는 원인을 확인하고 타자방과 같은 race-server 생명주기 경계로 맞춘다.
- 타자방 생성 후 간헐적으로 방으로 이동하지 않고 생성 모달이 남는 전이 버그를 수정한다.
- 관련 race-server/web 회귀 테스트 또는 타입 검증을 추가/수행한다.

### 논의 필요
- 없음. 사용자가 실제 버그로 지목했고, 타자방/카드방 생명주기 일관성이 필요하다.

### 선택지
- A. race-server room dispose/leave 경계에서 카드방 Spring 상태를 정리한다.
- B. Spring 배치/TTL cleanup을 추가한다.
- C. 클라이언트에서 임시로 숨긴다.

### 추천
- A를 우선 적용한다. 현재 타자방과 카드방 모두 실시간 방 생명주기는 race-server가 소유하므로, 빈 방 정리는 room lifecycle 경계에서 처리하는 것이 가장 직접적이다. 필요하면 후속으로 Spring TTL 보강을 둔다.

### 사용자 방향
- 추천 기준으로 진행한다.

## 완료 기록
- 카드방 퇴장/빈 방 종료를 Spring 상태와 race-server 연결 생명주기에 연결했다.
- 타자방 생성 성공 시 모달 닫힘과 생성 참가자 식별자 고정을 적용했다.
