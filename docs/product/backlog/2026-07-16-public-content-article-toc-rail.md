# 공개 콘텐츠 상세 목차 왼쪽 레일 정리

## 목표

공개 콘텐츠 상세의 본문 목차가 본문 그리드 열을 차지해 읽기 시작점이 오른쪽으로 밀리는 문제를 해소한다. 넓은 데스크톱에서는 목차를 본문 왼쪽 여백의 sticky 레일로 분리하고, 중간 폭·모바일에서는 접이식 목차를 유지한다. 로컬에서만 확인하며 PR, push, `main` 병합, 배포는 하지 않는다.

## 1차 — 목차 레이아웃 분리

### 작업내용

- 데스크톱 목차를 본문 그리드에서 제거하고 본문 왼쪽의 절대 위치 레일로 옮긴다.
- 목차 레일은 스크롤 중 상단 여백을 유지하도록 sticky로 둔다.
- 본문은 목차 유무와 관계없이 같은 시작선·읽기 폭을 사용한다.
- 왼쪽 여백이 부족한 중간 폭과 모바일에서는 기존 접이식 목차를 보여 준다.

### 논의 필요

- 긴 목차가 화면 높이를 넘을 때의 처리 방식.

### 선택지

1. 긴 목차도 화면 밖으로 계속 늘어뜨린다.
2. 목차 레일 내부만 최대 높이와 세로 스크롤을 둔다.

### 추천

2. 본문 스크롤을 방해하지 않으면서 긴 글의 모든 항목을 유지할 수 있다.

### 사용자 방향

목차가 본문 공간을 차지하지 않고 왼쪽에 떠서 사용자의 스크롤을 따라오게 한다.

## 진행 결과

- 완료: 데스크톱 목차를 본문 그리드 열에서 분리해 본문 왼쪽 여백의 독립 레일로 옮겼다.
- 완료: 목차 레일은 본문 높이를 기준으로 `sticky` 동작하고, 화면 높이를 넘는 경우 내부에서만 스크롤한다.
- 완료: `xl` 미만에서는 접이식 목차를 유지해 좁은 화면에서 본문 폭을 침범하지 않는다.
- 검증: 공개 콘텐츠 단위 테스트 98건, Chromium E2E 28건, lint, typecheck, 데스크톱·모바일·스크롤 상태 스크린샷을 통과했다.
- 범위: 로컬 커밋까지만 진행하며 PR, push, `main` 병합, 배포는 하지 않는다.
