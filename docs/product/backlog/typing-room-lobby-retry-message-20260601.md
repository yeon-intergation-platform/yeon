# 타자방 로비 재시도 문구 안정화 백로그

## 1차: 연결 실패 후 로비 상태 문구 고정

- 작업내용
  - 타자방 목록 재시도 중 로딩 문구와 실패 문구가 계속 번갈아 보이지 않게 한다.
  - 최초 로딩은 기존처럼 진행 상태를 보여주되, 연결 실패가 감지된 뒤에는 재시도 중에도 실패 문구를 유지한다.
  - 무한 재시도 동작 자체는 유지한다.
- 논의 필요
  - 실패 문구 유지 전환 기준을 훅의 에러 상태로 둘지, 화면 컴포넌트의 실패 이력으로 둘지.
- 선택지
  - A. 화면 컴포넌트에서 한 번이라도 에러가 발생했는지 기억해 표시 문구만 안정화한다.
  - B. 훅의 상태 모델을 바꿔 재시도 중에도 별도 `retrying-after-error` 상태를 노출한다.
- 추천
  - A. 네트워크 재시도 로직을 건드리지 않고 사용자 문구 흔들림만 최소 수정한다.
- 사용자 방향
  - 실패 후에는 `타자방 서버에 연결할 수 없어요.` 문구를 유지한다.
