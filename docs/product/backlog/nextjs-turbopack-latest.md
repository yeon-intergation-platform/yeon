# Next.js 최신 패치, Turbopack 명시, 자동 SemVer 릴리즈

## 1차

### 작업내용

- 현재 웹 앱의 Next.js 버전과 Turbopack 사용 여부를 확인한다.
- npm 기준 최신 Next.js 패치 버전으로 업데이트한다.
- 개발/프로덕션 빌드 스크립트에서 Turbopack 사용을 명시한다.
- `main` 배포 workflow가 성공하면 다음 `vX.Y.Z` 태그와 GitHub Release가 자동 생성되도록 한다.

### 논의 필요

- Next.js major 업그레이드는 별도 호환성 검증이 필요하다.
- 현재 요청 범위에서는 같은 16.x 패치 최신화만 수행한다.
- 자동 릴리즈 버전은 배포 성공 시점의 기존 SemVer 태그 중 가장 큰 값에서 patch를 1 올린다.
- 자동 태그는 배포 완료 커밋을 가리키며, `package.json` version과 반드시 같아야 하는 제약은 제거한다.

### 선택지

- A. 현재 버전을 유지하고 상태만 보고한다.
- B. 최신 패치로 업데이트하고 Turbopack 플래그를 명시한다.
- C. Canary/RC까지 올린다.
- D. 배포 성공 후 SemVer 태그/릴리즈를 자동 생성한다.

### 추천

- B + D. 운영 안정성을 유지하면서 최신 패치, 명시적 Turbopack 사용, 배포 단위 자동 버전 생성을 확보한다.

### 사용자 방향

- 사용자가 "사용하고 있지 않다면 도입"을 요청했으므로, 이미 사용 중인 사실은 보고하되 최신 패치 및 명시 설정을 적용한다.
- 사용자가 workflow 완료 후 자동 `숫자.숫자.숫자` 버전 부여를 요청했으므로 자동 SemVer 릴리즈 workflow를 추가한다.
