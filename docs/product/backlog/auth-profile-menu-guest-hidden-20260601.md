# 비로그인 공통 헤더 프로필 메뉴 숨김 백로그

## 1차: 세션이 없으면 내정보 메뉴 버튼 자체를 숨긴다

- 작업내용
  - 공통 서비스 헤더의 기본 프로필 메뉴 버튼이 현재 인증 세션을 확인하도록 한다.
  - 세션 확인 전/비로그인/세션 조회 실패 상태에서는 버튼과 드롭다운을 렌더링하지 않는다.
  - 인증된 사용자에게만 내정보보기/로그아웃 메뉴를 노출한다.
- 논의 필요
  - 비로그인 상태에서 로그인 CTA를 헤더에 보여줄지 여부.
- 선택지
  - A. 비로그인 상태에서는 프로필 메뉴 영역을 숨긴다.
  - B. 비로그인 상태에서는 로그인 버튼으로 대체한다.
- 추천
  - A. 사용자가 문제 삼은 것은 로그인하지 않았는데 내정보/로그아웃이 보이는 상태이므로, 우선 잘못된 계정 액션 노출을 제거한다.
