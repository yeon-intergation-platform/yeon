# 카드덱 인증 토큰 검증 경로 정합화

## 1차

### 작업내용

- 카드덱 PATCH 요청에서 브라우저 쿠키가 존재해도 401이 발생하는 원인을 Next BFF 인증 검증 경로와 Spring 세션 검증 경로 기준으로 분리한다.
- Web은 HttpOnly 쿠키, Mobile은 Authorization Bearer를 사용하되 둘 다 동일한 Spring 세션 토큰 검증 모델을 쓰도록 유지한다.
- 카드덱 API가 인증 실패를 추측하지 않고 세션 토큰 검증 결과에 따라 일관되게 동작하도록 수정한다.

### 논의 필요

- 없음. 사용자가 웹/모바일 세션 모델 통일과 401 해결을 지시했다.

### 선택지

1. Web도 Bearer로 통일한다.
2. Web은 쿠키, Mobile은 Bearer를 유지하고 Spring 세션 토큰 검증 모델만 통일한다.
3. 카드덱 API만 임시 우회한다.

### 추천

- 2번. 쿠키/Bearer는 플랫폼별 운송수단이고, 인증 상태의 진실은 Spring 세션으로 통일한다.

### 사용자 방향

- 추천 기준으로 진행한다.
