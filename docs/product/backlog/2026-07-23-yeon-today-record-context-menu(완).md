# YEON Today 기록 우클릭 메뉴와 간결한 설명 편집

## 목표

시간 블록의 좌클릭 기록 추가와 설명 편집 동작이 겹치지 않도록 보조 작업을 OS식 컨텍스트 메뉴로 분리하고, 설명 편집 UI를 짧은 인라인 바 형태로 단순화한다.

## 1차 — 시간 블록 동작 분리

- 작업내용
  - 기록이 있는 영역을 우클릭하면 해당 기록의 설명 편집·삭제 메뉴를 연다.
  - 좌클릭은 기존 기록 추가 흐름을 유지한다.
  - 키보드와 터치 사용자는 명시적인 더보기 버튼으로 같은 메뉴에 접근한다.
- 논의 필요
  - 우클릭이 없는 모바일에서도 보조 작업을 발견하고 실행할 수 있어야 한다.
- 선택지
  - 우클릭만 제공한다.
  - 우클릭과 더보기 버튼을 함께 제공한다.
- 추천
  - 우클릭과 더보기 버튼을 같은 메뉴에 연결한다.
- 사용자 방향
  - OS처럼 오른쪽 클릭 시 드롭다운이 열리도록 한다.

## 2차 — 설명 편집 UI 단순화

- 작업내용
  - 큰 카드형 편집 패널을 활동 선택, 한 줄 설명, 취소·저장 버튼만 있는 압축형 인라인 바로 바꾼다.
  - 삭제는 컨텍스트 메뉴로 옮겨 편집 UI의 책임을 줄인다.
- 논의 필요
  - 설명은 시간 블록에서 한 줄로 표시되므로 편집 입력도 한 줄이 적절한지 확인한다.
- 선택지
  - 기존 두 줄 textarea를 유지하고 주변 요소만 줄인다.
  - 한 줄 입력으로 바꾸고 글자 수 안내는 입력 우측에 작게 표시한다.
- 추천
  - 한 줄 입력으로 바꾸고 200자 제한은 유지한다.
- 사용자 방향
  - 현재 편집 화면을 더 간단하게 고친다.

## 3차 — 검증과 배포

- 작업내용
  - Playwright로 좌클릭 기록 추가, 우클릭 메뉴, 더보기 메뉴, 간결한 편집 UI를 데스크톱·모바일에서 확인한다.
  - lint, typecheck, 관련 테스트, 패리티 검증을 실행한다.
  - 변경을 커밋하고 PR을 `main`에 병합한다.
- 논의 필요
  - 없음.
- 선택지
  - 정적 검증만 수행한다.
  - 실제 브라우저 동작과 시각 증거까지 남긴다.
- 추천
  - 실제 브라우저 동작과 before/after 스크린샷을 남긴다.
- 사용자 방향
  - 바로 사용할 수 있는 완성 상태로 반영한다.

## 완료 결과

- 좌클릭은 두 번째 기록 추가, 우클릭은 설명 편집·기록 삭제 메뉴로 분리했다.
- 터치·키보드 환경은 시간 블록의 더보기 버튼으로 같은 메뉴에 접근한다.
- 설명 편집은 활동 선택, 한 줄 설명, 취소·저장만 남긴 압축형 인라인 바로 교체했다.
- 데스크톱·모바일 Playwright 검증과 정적 검사를 통과했다.
