# 방 음성통화 런타임 플래그 활성화 복구 (2026-05-18)

## 1차

### 작업내용

- 운영 컨테이너에 `NEXT_PUBLIC_ENABLE_ROOM_VOICE_CALL=true`가 있어도 클라이언트에서 음성통화가 꺼진 것으로 표시되는 문제를 해결한다.
- Next public env의 빌드타임 인라인 한계 때문에 런타임 컨테이너 env를 읽는 설정 API를 추가하고, 클라이언트 음성통화 훅이 해당 값을 사용하도록 바꾼다.
- compose/dev-all 환경 전달도 같은 플래그 이름으로 정리한다.

### 논의 필요

- 없음. 사용자는 실제 env가 true인데 UI가 비활성화되는 버그를 제보했다.

### 선택지

1. Docker build arg에만 플래그를 추가한다.
2. 런타임 설정 API를 추가해 컨테이너 env를 클라이언트가 조회한다.
3. 음성통화를 항상 켠다.

### 추천

- 2번. 운영 컨테이너 env를 바꿨을 때 재빌드 없이 의도대로 동작하고, feature flag도 유지된다.

### 사용자 방향

- 추천 기준으로 진행한다.
