# 랜딩 백지 학습 카드 프레임 브레이크 적용

> 상태: 완료

## 1차 — 백지 학습 원본 에셋 분리

- 작업내용: 사용자 제공 이미지를 백지 학습용 배경 WebP와 투명 전경 WebP로 분리한다.
- 논의 필요: 전경의 흰색 노트·전등 빛·전구 아이콘은 보존하면서 외부 흰 배경만 제거해야 한다.
- 선택지:
  - A. 이미지 외곽과 연결된 흰 배경만 투명화해 내부 흰색 요소를 보존한다.
  - B. 밝은 픽셀 전체를 일괄 투명화한다.
- 추천: A. 연결 영역 기반 제거로 노트와 조명 디테일의 손실을 방지한다.
- 사용자 방향: 기존 타자 카드와 동일한 두 장 구조로 백지 학습 카드에 적용하고 main에 병합한다.

## 2차 — 랜딩 카드 공용 프레임 브레이크 구성

- 작업내용: 타자·백지 학습 카드가 각 서비스별 배경/전경/대체 텍스트를 선택해 동일한 카드 구조를 재사용하게 한다.
- 논의 필요: 카드별 조건문 복제를 피하면서 서비스 번호, 제목 안전영역, 상태 배지, CTA 위계를 유지해야 한다.
- 선택지:
  - A. 서비스 slug를 키로 하는 프레임 브레이크 에셋 설정 객체를 두고 공용 렌더링을 사용한다.
  - B. 백지 학습 전용 카드 JSX를 하나 더 복제한다.
- 추천: A. 두 서비스의 차이는 데이터로 두고 레이아웃은 한 곳에서 유지한다.
- 사용자 방향: 타자 카드와 같은 프레임 브레이크 표현을 백지 학습에 적용한다.

## 완료 조건

- 백지 학습 카드가 배경 1장과 투명 전경 1장으로 렌더링된다.
- 제목·번호·상태 배지가 전경에 가려지지 않는다.
- 타자 카드의 기존 표현과 동작이 유지된다.
- 375px, 480px, 1040px에서 이미지 로드와 가로 스크롤 부재를 확인한다.

## 완료 결과

- 원본에서 보라색 방 배경을 1414×426 WebP로 분리했다.
- 이미지 외곽과 연결된 흰 영역만 제거해 전등·전구·노트·인물을 보존한 1454×630 투명 WebP 전경을 만들었다.
- 서비스 slug 기반 설정 객체에서 타자·백지 학습 카드의 배경, 전경, 순서, 대체 텍스트를 관리하도록 공용 렌더링을 확장했다.
- 375px, 480px, 1040px에서 제목·상태 배지 비겹침, 이미지 로드, 가로 스크롤 부재를 확인했다.
- web lint, typecheck, build와 Universal UI parity, 프로젝트 SSOT 검증을 통과했다.
