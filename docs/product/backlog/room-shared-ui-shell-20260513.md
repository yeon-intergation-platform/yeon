# 방 공통 UI 셸 백로그 (2026-05-13)

## 1차: 카드방/타자방 중복 UI 셸 최소 공통화

### 작업내용

- 카드방/타자방 로비와 대기 화면에서 반복되는 캐릭터 요약 카드, 방 생성 모달 오버레이, 참가자 카드 UI를 공통 컴포넌트로 분리한다.
- 카드방/타자방의 방 생성·입장·게임 진행 도메인 로직은 그대로 유지하고, 표현 계층만 공유한다.
- `typing-service` 내부 캐릭터 렌더링을 카드방이 직접 복제하지 않도록 공통 방 UI 레이어를 둔다.

### 논의 필요

- 이번 차수는 큰 구조 변경 없이 UI 셸 중복 제거만 진행한다.
- 프로필 저장소 통합, Colyseus 연결 훅 통합, 방 상태 모델 통합은 도메인 영향이 커서 후속 차수로 분리한다.

### 선택지

1. 로비/참가자/모달 UI 셸만 공통화한다.
2. 방 생성/입장 훅까지 한 번에 공통화한다.
3. 카드방/타자방 전체 RoomScreen을 하나의 추상화로 합친다.

### 추천

- 1번. 카드방과 타자방은 실시간 방이라는 UX는 같지만 상태 전이와 서버 소유권이 다르므로, UI 셸만 먼저 공유하는 것이 안전하다.

### 사용자 방향

- 비어 있음. 추천 기준으로 진행한다.
