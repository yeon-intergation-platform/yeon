# Admin content ops UX 개선

## 배경

`/admin`이 운영 허브가 아니라 회원 관리로 바로 이동하고, 공개 콘텐츠/타자 덱/타자 캐릭터 관리 화면의 진입점이 흩어져 있다. 공개 콘텐츠 운영 화면은 33개 글 전체에 `meta description 누락`과 source traceability 경고를 표시하고 있어 실제 운영 판단이 어렵다.

## 1차

### 작업내용

- `/admin`을 운영 허브로 전환한다.
- 공개 콘텐츠, 회원 관리, 사용자 경험치, 타자 덱, 타자 캐릭터, 타자방 운영으로 이동할 수 있는 공통 admin 내비게이션을 만든다.
- `/admin/typing-characters`를 내비게이션에서 직접 접근 가능하게 한다.
- 공개 콘텐츠 seed와 DB 마이그레이션에 `meta_description`, `source_paths` 기본값을 보강한다.
- 타자 덱 관리자 화면에서 생성 폼과 상세 패널이 한 화면에 어색하게 고정되는 흐름을 줄인다.

### 논의 필요

- 지금 단계의 admin은 직접 수정/삭제보다 운영 상태 확인과 관리 화면 진입을 우선한다.
- 공개 콘텐츠 source 경로는 정적 seed 파일 기준으로 먼저 기록하고, 이후 CMS/import workflow가 생기면 더 세분화한다.

### 선택지

- A. `/admin`을 단순 링크 목록으로만 둔다.
- B. `/admin`을 운영 허브로 만들고 콘텐츠/타자/사용자 주요 상태를 함께 노출한다.
- C. 각 admin 화면마다 별도 헤더를 유지한다.

### 추천

B를 선택한다. 관리자가 `/admin`에 들어왔을 때 현재 운영 상태와 다음 관리 화면을 바로 선택할 수 있어야 하고, 공통 nav를 두면 `/admin/typing-characters` 같은 숨은 페이지가 줄어든다.

### 사용자 방향

추천안 기준으로 진행한다.
