### 차수 1 — 커뮤니티 진입점을 `/community`로 통합 및 페이지 내 실시간 채팅/피드 구성

- 작업내용: `/community`를 커뮤니티 단일 진입점으로 확정하고, 페이지 상단에 실시간 채팅 섹션을 배치한 뒤 기존 타이핑/카드 서비스 화면에도 서비스 내부형 채팅 위젯을 추가한다. 커뮤니티 서비스 카드를 홈에 반영해 사용자가 `/community`로 이동하도록 한다.
- 논의 필요: `/community`에서 표시할 채팅 상대/방 선택 방식(최신 방 우선, 전체 목록 필터, 고정 공개방 등)과 카드/타이핑 서비스 위젯 노출 전략(항상 표시/접기 가능/반응형 크기) 간 우선순위 결정.
- 선택지:
  - 옵션 A (권장): `/community`는 로그인/OTP 인증 후 1:1 채팅방 목록+메시지 스트리밍(폴링) + 커뮤니티 피드/댓글 기능을 보여주는 단일 페이지로 구성하고, 카드/타이핑 서비스에는 동일 API를 이용한 미니 채팅 위젯을 고정 배치한다.
  - 옵션 B: `/community`를 피드 중심 페이지로 두고 채팅은 모달 오버레이로 별도 진입.
  - 옵션 C: 기존 `/typing-service/rooms` 내부 채팅과 `/card-service` 헤더 버튼을 유지한 채 보조 진입점으로 계속 노출.
- 추천: 옵션 A.
- 사용자 방향: 옵션 A 기준으로 진행.

### 차수 2 — 커뮤니티 글/댓글 익명 CRUD 허용

- 작업내용: `/community`에서 로그인이 없어도 닉네임+비밀번호를 입력해 글 작성, 수정, 삭제, 댓글 작성/삭제를 수행할 수 있도록 익명 식별 흐름을 추가한다.
- 논의 필요: 익명 작성자의 권한 검증을 프로필 기반 임시 동기화 방식으로 둘지, 별도 익명 토큰 기반으로 둘지.
- 선택지:
  - 옵션 A (권장): 익명 입력값(닉네임/비밀번호)으로 내부 임시 작성자 프로필을 생성/조회해 세션 없이도 `postId` 기반 소유권 검증을 수행한다.
  - 옵션 B: 쿠키 기반 임시 토큰을 발급해 토큰과 비밀번호만으로 삭제/수정 검증을 수행한다.
  - 옵션 C: 익명 글은 작성/삭제만 허용하고 수정은 로그인 사용자만 허용한다.
- 추천: 옵션 A(기존 DB 스키마를 유지하면서 익명 글/댓글 소유권을 재현하기 쉬움).
- 사용자 방향: 추천 기준으로 진행.

### 차수 1 — 카드 작성 모달 전환과 편집 UX 일관성 복구

- 작업내용: `card-service` 덱 상세 화면에서 카드 추가를 사이드 패널이 아닌 중앙 모달(모바일은 바텀시트/풀스크린 대응)로 전환하고, 카드 생성/수정/내보내기 흐름을 같은 편집 경험으로 통일한다. 수정 화면에도 마크다운 에디터와 이미지 입력, 자동 임시저장, 모달 닫기 경고를 적용하고 불필요한 연필 버튼·우측 치우친 액션·드래그 충돌·작은 타이포를 함께 정리한다.
- 논의 필요: 카드 이미지 저장을 별도 필드로 분리할지, Markdown 본문과 연결된 보조 메타데이터로 둘지 결정이 필요하다.
- 선택지:
  - 옵션 A. 카드 아이템에 질문/답변별 이미지 URL 필드를 추가하고 생성/수정 공통 폼에서 관리한다.
  - 옵션 B. 이미지 기능은 Markdown 텍스트에만 의존하고 별도 저장 필드는 두지 않는다.
- 추천: 옵션 A. 생성/수정/미리보기/교체/삭제/임시저장을 안정적으로 구현하기 쉽고 export/복원 UX도 명확하다.
- 사용자 방향: 추천 기준으로 진행.
