# 공개 콘텐츠 에디토리얼 정리 2차

## 목표

Support는 실제 문제 해결, News는 실제 변경 공지, Blog는 근거가 있는 제작 기록으로 읽히게 정리한다. 공통 헤더의 구조와 채널 이동은 유지하되, 공개 콘텐츠에서만 BGM·설정 제어를 숨긴다.

## 1차 — 피드의 실제성 회복

### 작업내용

- 채널 분리·운영 원칙·내부 사이트맵처럼 사용자 행동을 돕지 않는 News·Blog 글을 공개 피드에서 제거한다.
- News에는 이번 공개 콘텐츠 정리라는 실제 변경 공지 한 건만 남기고, 적용 범위와 사용자 조치를 짧게 기록한다.
- Blog에는 NEXA, 타자방, 플래시카드, 커뮤니티의 실제 구현 근거가 있는 기록만 남긴다.
- Dailyting처럼 YEON 공개 블로그의 서비스 맥락과 맞지 않는 글은 노출하지 않는다.

### 논의 필요

- 없음. 공개 피드의 정보 신뢰가 우선이다.

### 선택지

1. 기존 글을 문장만 다듬어 모두 유지한다.
2. 실제 서비스 변경·구현 근거가 없는 글은 피드에서 제거한다.

### 추천

2. 문체보다 콘텐츠 인벤토리가 먼저 정리되어야 실제 서비스처럼 읽힌다.

### 사용자 방향

AI 티가 나거나 실제 서비스 같지 않은 요소를 전부 정리하고, 사람이 여러 서비스를 비교하며 사용할 때 자연스럽게 고친다.

## 2차 — 화면 밀도와 검증

### 작업내용

- Support 홈의 채널 분리 공지 카드를 제거하고 문제 해결 진입만 남긴다.
- News 한 건 상태에서 중복되는 최신 목록을 숨긴다.
- 공개 콘텐츠 헤더에서만 BGM·설정 제어를 숨기고 나머지 서비스 헤더 동작은 유지한다.
- 데스크톱·모바일의 Support·News·Blog 홈과 대표 글을 Playwright로 확인하고 스크린샷을 남긴다.

### 논의 필요

- 없음.

### 선택지

1. 시각 요소를 더 추가해 빈 영역을 채운다.
2. 필요한 정보만 남기고 빈 공간은 읽기 여백으로 사용한다.

### 추천

2. 현재의 무채색 편집물 분위기는 유지하면서도 화면을 덜 산만하게 만든다.

### 사용자 방향

실제 사용자에게 필요한 정보와 행동만 남기고, 검증 후 `main`에 병합한다.

## 결과

- News는 실제 공개 콘텐츠 정리 공지 한 건만, Blog는 서비스 구현 근거가 있는 다섯 건만 공개 피드에 남겼다. 제거한 기존 URL은 404가 되고 사이트맵·정적 경로에서 빠진다.
- Support 홈의 최근 공지 카드를 제거하고 검색·서비스 진입·자주 찾는 문제에 집중시켰다.
- News의 중복 최신 목록과 한 항목짜리 필터·목차를 숨겼고, Blog는 필터 결과 전체를 보여 준다.
- 공개 콘텐츠 공통 헤더에서는 BGM·설정 제어만 숨기고 Support·News·Blog 탭과 내부 로컬 경로는 유지했다.
- Support 긴 글의 목차는 본문 칼럼 밖 왼쪽 고정 레일에 배치했다.
