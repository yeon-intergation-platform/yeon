# 랜딩 서비스 카드 프레임 브레이크 테스트

> 상태: 완료

## 1차 — 타자연습 카드 시각 테스트

- 작업내용: 사용자가 제공한 일러스트를 배경과 투명 전경으로 분리하고, 랜딩의 `키보드 타자연습` 카드에만 프레임 브레이크 레이어 구조를 적용한다.
- 논의 필요: 작은 모바일 카드에서도 인물 돌출이 제목·상태·CTA를 가리지 않는지 실제 화면 확인이 필요하다.
- 선택지:
  - A. 배경 1장과 투명 전경 1장을 별도 레이어로 배치한다.
  - B. 완성된 합성 이미지를 카드 안에 단일 이미지로 배치한다.
- 추천: A. 레이어를 분리해야 돌출 높이와 모바일 크롭을 CSS에서 조절할 수 있다.
- 사용자 방향: 추천안 A로 빠르게 로컬 테스트한다.

## 완료 조건

- 카드 배경과 전경 에셋이 별도 파일로 존재한다.
- 데스크톱과 375px 모바일에서 인물, 상태 배지, 제목, 설명, CTA가 충돌하지 않는다.
- Playwright 스크린샷과 lint/typecheck 결과를 작업 로그에 남긴다.

## 결과

- 배경과 투명 전경을 WebP 2개로 분리했다.
- 동일 전경을 CSS에서 한 번 더 잘라 사용해 인물 머리만 카드 이미지와 헤더의 경계를 넘게 했다.
- 데스크톱과 375px 모바일에서 제목·운영 상태·CTA가 가려지지 않는 것을 확인했다.
