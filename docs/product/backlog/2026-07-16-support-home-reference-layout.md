# Support 홈 레퍼런스 기반 탐색 구조 개선

## 목표

공통 헤더는 그대로 유지하면서 Support 첫 화면을 `검색 → 서비스 선택 → 자주 찾는 문제 → 서비스별 문서 → 문의` 순서로 재구성한다. 운영 사본을 복원한 로컬 환경에서만 확인하며 PR, push, `main` 병합, 배포는 하지 않는다.

## 1차 — 첫 화면 정보 위계

### 작업내용

- Support 히어로에 서비스별 문제 해결 목적을 명확히 드러낸다.
- 실제 News 공지 데이터를 재사용한 최신 공지 패널을 배치한다.
- 기존 검색 URL 상태와 결과 동작을 유지한 채 검은 검색 영역으로 강화한다.

### 논의 필요

- 채널 전환 탭을 공통 헤더로 되돌릴지 여부.

### 선택지

1. 공통 헤더 안으로 이동한다.
2. Support 히어로 오른쪽 보조 영역에 유지한다.

### 추천

2. 공통 헤더를 유지하라는 기존 방향에 맞춰, 히어로 오른쪽의 보조 탐색으로 유지한다.

### 사용자 방향

공통 헤더는 그대로 두고, 첨부 레퍼런스처럼 Support 메인 화면을 개선한다.

## 2차 — 문서 진입 구조와 검증

### 작업내용

- 서비스 카드와 자주 찾는 문제의 위계를 레퍼런스에 맞게 다듬는다.
- 기존 운영 데이터 기반 카드 링크, 검색, 반응형 화면을 확인한다.
- 로컬 커밋만 생성한다.

### 논의 필요

- 없음.

### 선택지

1. 카드와 문서를 모두 같은 무게로 나열한다.
2. 서비스 선택과 자주 찾는 문제를 먼저 보여 주고 서비스별 문서는 뒤에 둔다.

### 추천

2. 처음 방문한 사용자가 자신의 문제 영역을 빠르게 고를 수 있다.

### 사용자 방향

모든 제안은 적용하지 않고, 실제 탐색에 도움이 되는 항목만 적용한다.
