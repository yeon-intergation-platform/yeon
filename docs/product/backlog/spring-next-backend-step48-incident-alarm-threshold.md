# 차수48 장애 감지 임계치 설정
- 생성일시: 2026-05-09T21:49:00+09:00
- 작업내용: Spring/Next 이관 구간의 장애 알람 임계치와 완화 규칙을 정의해 운영 대응 속도를 높인다.
- 논의 필요: 알림 민감도(과잉 알림) vs 탐지 지연 간 균형.
- 선택지:
  1. 민감도 높은 실시간 알림
  2. 누적 오류율 중심의 완화 알림
- 추천: 누적 임계치 + 급등 알림 병행
- 사용자 방향: 추천 기준 진행
- 확장 산출:
  - 기본 임계치: p95 지연 급등, 500 에러율, DB 커넥션 timeout
  - 완화 규칙: 5분 이동평균 기반 재경보 억제
  - 연결 채널: 알림 채널(슬랙/메일) + 상태 페이지 노트
