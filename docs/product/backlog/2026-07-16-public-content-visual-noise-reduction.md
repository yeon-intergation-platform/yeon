# 공개 콘텐츠 반복 정보 제거 및 시각적 정돈

## 목표

Support·News·Blog의 공통 헤더는 유지한다. 대신 모든 글과 홈에 반복되어 사용자의 읽기 흐름을 끊는 안내 패널·중복 제목·보조 문구를 제거하고, News 공지의 `적용 서비스`와 `적용일`만 제목 아래의 압축 메타데이터로 남긴다. 운영 데이터 사본을 사용한 로컬 환경에서만 확인하며 PR, push, `main` 병합, 배포는 하지 않는다.

## 1차 — 상세 글 반복 정보 축소

### 작업내용

- News 상세의 공통 `핵심 확인 사항` 패널과 카테고리별 자동 설명 카드를 제거한다.
- News 공지는 `적용 서비스 · 적용일`을 제목 아래의 작은 메타데이터로만 보여 준다.
- Support 상세의 본문 체크리스트를 복제하는 `먼저 확인할 것` 요약을 제거한다.
- Blog 상세의 운영 주체, 교차 채널 링크, repo 근거 패널을 제거하고 기존 하단 관련 글만 유지한다.
- `support 글을 반복하지 않습니다`처럼 채널 운영 원칙을 매 글에서 재설명하는 callout을 데이터에서 삭제한다.

### 논의 필요

- 공지의 적용 범위 정보는 유지할지 여부.

### 선택지

1. 공지 정보 카드를 상세마다 유지한다.
2. 적용 서비스와 적용일만 제목 아래 메타데이터로 압축한다.

### 추천

2. 공지에서 실제 판단에 필요한 사실은 남기면서 본문 시작 전의 카드 벽을 없앨 수 있다.

### 사용자 방향

적용일과 적용 서비스는 유용하지만 큰 공간을 차지하지 않게 하고, `핵심 확인 사항`과 `support 글을 반복하지 않습니다` 같은 반복 요소는 없앤다.

## 2차 — 홈 탐색 위계 정리와 검증

### 작업내용

- News 홈에서 필터와 `중요 공지`·`최신 소식` 사이의 중복 레이블과 안내 문구를 줄인다.
- Blog 홈에서 이미 필터로 제공되는 분류 목록을 다시 나열하는 `읽는 경로` 영역을 제거한다.
- Support 홈의 검색 → 서비스 선택 → 자주 찾는 문제 → 서비스별 문서 → 문의 흐름은 유지하고, 상세 글의 중복 보조 정보만 축소한다.
- 데스크톱·모바일 스크린샷과 Playwright로 세 채널의 홈 및 대표 상세 글을 확인한다.

### 논의 필요

- 없음.

### 선택지

1. 카드와 보조 정보를 같은 밀도로 계속 노출한다.
2. 사용자가 바로 행동하는 데 필요한 정보만 남기고, 설명은 글 본문과 관련 글로 위임한다.

### 추천

2. 무채색 편집물 분위기를 유지하면서도 정보의 우선순위가 명확해진다.

### 사용자 방향

Support·News·Blog에서 산만하게 만드는 요소를 없애고 전체적으로 다듬는다.

## 진행 결과

- 완료: News 공지의 적용 서비스·적용일은 제목 아래의 작은 메타데이터로 유지하고, `핵심 확인 사항`·카테고리별 자동 안내·`support 글을 반복하지 않습니다` callout을 제거했다.
- 완료: Support 상세의 본문 중복 요약과 Blog 상세의 운영·근거 패널을 제거했다.
- 완료: News·Blog 홈의 중복 섹션명·분류 목록·보조 문구를 정리하고, Support 홈의 문제 해결 흐름은 유지했다.
- 검증: 공개 콘텐츠 단위 테스트 98건, Chromium E2E 27건, lint, typecheck, production build와 데스크톱·모바일 스크린샷 확인을 통과했다.
- 범위: 로컬 커밋까지만 진행하며 PR, push, `main` 병합, 배포는 하지 않는다.
