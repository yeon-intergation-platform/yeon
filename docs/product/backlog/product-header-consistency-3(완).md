# 제품 헤더 시각 규격 통일 3차 백로그

- 상태: 완료
- 결과: 백지·공개 콘텐츠(Support·News·Blog)의 상단 제품 헤더를 공통 `CommonProductHeader`로 통일하고, 뉴스 채널 전환은 44px 보조 탐색으로 유지했다.

## 1차 — 공통 헤더 계약과 서비스 정체성 확정

### 작업내용

- 공통 제품 헤더에 백지, 고객지원, 뉴스, 블로그의 서비스 정체성과 상위 이동 경로를 추가한다.
- `blurt.yeon.world`의 백지 홈·세션이 타자 서비스의 브랜드를 재사용하지 않게 분리한다.
- 공개 콘텐츠 채널은 기존 Support·News·Blog 탐색을 유지하되, 상단 제품 헤더 프레임은 플래시카드 기준을 재사용한다.

### 논의 필요

- 뉴스의 채널 전환은 서비스 공통 헤더의 유틸리티가 아니라 공개 콘텐츠 전용 보조 탐색으로 유지할지 판단이 필요하다.

### 선택지

1. 뉴스 화면마다 헤더 클래스를 수동으로 맞춘다.
2. 공통 제품 헤더는 하나로 재사용하고, 공개 콘텐츠 채널 이동은 별도 보조 탐색으로 분리한다.

### 추천

- 2번. 브랜드·높이·글자·터치 타깃의 source of truth를 하나로 유지하면서 뉴스의 채널 탐색 목적도 보존한다.

### 사용자 방향

- `blurt.yeon.world`는 `YEON 타자방`이 아닌 `YEON 백지`로 표기한다.
- YEON 뉴스도 플래시카드 기준 공통 헤더를 일관되게 적용한다.

## 2차 — 백지와 뉴스 화면 적용

### 작업내용

- 백지 홈과 세션의 타자 전용 헤더를 백지 전용 공통 헤더로 교체한다.
- 공개 콘텐츠 셸의 독자 헤더를 공통 제품 헤더로 교체하고, Support·News·Blog 이동은 접근 가능한 보조 탐색으로 유지한다.
- 뉴스 홈의 기존 콘텐츠, 채널 전환, SEO/추적 링크, 공개 서브도메인 리라이트는 변경하지 않는다.

### 논의 필요

- 백지·뉴스의 BGM/설정은 공통 제품 헤더 기본 동작을 따르되, 콘텐츠 본문 기능으로 확장하지 않는다.

### 선택지

1. 백지와 뉴스에서 각각 다른 헤더 컴포넌트를 유지한다.
2. 공통 제품 헤더를 재사용하고 서비스별 브랜드·보조 탐색만 주입한다.

### 추천

- 2번. 다음 서비스 추가 시에도 헤더 드리프트가 재발하지 않는다.

### 사용자 방향

- 추천 기준으로 진행한다.

## 3차 — 브라우저 증거와 main 반영

### 작업내용

- `blurt.yeon.world`와 `news.yeon.world`의 desktop(1440px)·mobile(375px) 전후 화면을 Playwright로 캡처한다.
- `@yeon/web` lint/typecheck/build, 관련 단위 테스트, `git diff --check`를 통과시킨다.
- 변경 파일만 한국어 커밋으로 커밋하고 PR(main)을 생성·병합한다.

### 논의 필요

- GitHub 필수 검사가 남아 있으면 보호 규칙을 우회하지 않고 통과 후 병합한다.

### 선택지

1. 정적 검사만 수행한다.
2. 정적 검사와 실제 공개·로컬 화면 비교를 모두 남긴다.

### 추천

- 2번. 사용자가 지적한 문제가 시각·브랜드 정체성이므로 실제 브라우저 증거가 필요하다.

### 사용자 방향

- 작업 후 PR을 main에 병합한다.
