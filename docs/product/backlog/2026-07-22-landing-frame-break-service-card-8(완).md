# 랜딩 키보드 타자연습 카드 전경 구도 재교체

> 상태: 완료

## 1차 — 새 원본 에셋 분리

- 작업내용: 사용자 제공 이미지를 키보드 타자연습 카드용 방 배경 WebP와 투명 전경 WebP로 분리한다.
- 논의 필요: 전등 내부 빛을 포함한 원본 픽셀을 보존하면서 외부 흰 배경만 투명화해야 한다.
- 선택지:
  - A. 외곽과 연결된 흰 배경만 제거하고 전경 내부의 밝은 영역은 원본에서 복원한다.
  - B. 밝은 픽셀 전체를 일괄 제거한다.
- 추천: A. 전등·모니터·인물 외곽의 손상을 최소화한다.
- 사용자 방향: 새로 제공한 이미지를 이용해 타자연습 카드를 다시 만들고 로컬호스트에서 즉시 확인한다.

## 2차 — 카드 교체 및 로컬 검증

- 작업내용: 기존 프레임 브레이크 레이아웃은 유지하고 새 에셋 경로와 실제 크기만 반영한다.
- 논의 필요: 제목·번호·운영 상태 안전영역과 반응형 크롭을 유지해야 한다.
- 선택지:
  - A. 새 버전 파일명으로 교체해 브라우저 캐시를 피한다.
  - B. 기존 v2 파일을 덮어쓴다.
- 추천: A. 새 v3 경로로 즉시 변경 여부를 식별한다.
- 사용자 방향: localhost에서 새 이미지를 바로 확인할 수 있게 한다.

## 완료 조건

- 제공 이미지의 방 영역과 투명 전경이 각각 실제 이미지 에셋으로 렌더링된다.
- CSS로 전등이나 잘린 요소를 생성·연장하지 않는다.
- 제목·번호·운영 상태가 전경과 겹치지 않는다.
- 모바일과 데스크톱에서 가로 스크롤 없이 표시된다.

## 완료 결과

- 새 원본의 방 영역을 1412×480 WebP 배경으로 분리했다.
- 외곽과 연결된 흰 배경과 손·키보드 사이의 닫힌 흰 영역을 제거한 1448×580 투명 WebP 전경을 만들었다.
- 전등 내부 빛, 모니터, 키보드, 머그컵, 인물은 원본 픽셀 그대로 보존했다.
- 기존 카드 구조와 CSS는 변경하지 않고 `typing-service`의 에셋 경로와 실제 높이만 v3로 교체했다.
- 375px와 1040px에서 제목 비겹침, 이미지 로드, 가로 스크롤 부재를 확인했다.
- web lint, typecheck, build, Universal UI parity, 프로젝트 SSOT 검증을 통과했다.
