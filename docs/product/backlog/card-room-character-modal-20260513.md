# 카드방 캐릭터 상태 연결과 만들기 모달 전환

## 1차

### 작업내용

- `/card-service`에서 선택한 캐릭터 상태가 `/card-service/rooms` 로비의 오른쪽 상단 표시까지 이어지는 실제 상태 저장 경로를 확인한다.
- 기존 상태 관리/저장 방식을 우선 사용해 카드방 로비가 임의 기본 캐릭터로 돌아가지 않게 수정한다.
- `/card-service/rooms`의 카드방 만들기 CTA가 `/card-service/rooms/new`로 이동하지 않고 로비 위 overlay modal을 열도록 변경한다.
- 기존 카드방 생성 흐름을 유지하되, 불필요한 페이지 이동과 중간 URL 노출을 줄인다.
- 타자방과 카드방의 방 목록/빈 상태/모달/생성 흐름 중 실제 중복만 확인하고, 안전한 범위에서만 작은 공통화 또는 제안을 남긴다.

### 논의 필요

- 카드방 생성이 현재 페이지 컴포넌트에 강하게 결합되어 있으면, 대규모 추출 없이 로비 내부 모달로 옮길 수 있는 범위를 먼저 확인해야 한다.

### 선택지

1. 기존 카드방 생성 컴포넌트/훅을 로비 모달에서 재사용하거나 최소 추출한다.
2. `/rooms/new` 페이지를 유지하고 로비에서만 iframe/redirect처럼 감싼다.
3. 카드방/타자방 방 생성 UI를 대규모 공통 컴포넌트로 재작성한다.

### 추천

- 1번. 현재 구조를 먼저 따르고, 타자방 UX 패턴은 overlay 동작만 참고한다. 공통화는 실제 중복이 확인된 작은 유틸/컴포넌트 수준으로 제한한다.

### 사용자 방향

- 추천 기준으로 진행한다.
