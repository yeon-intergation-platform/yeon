# community random guest nickname 2026-05-13

## 배경

커뮤니티 미로그인 사용자의 기본 이름이 모두 `익명이`로 고정되어 있어 실시간 채팅에서 서로 다른 게스트를 구분하기 어렵다. 기본 표시명은 계정 정보가 아니라 익명성을 유지하는 랜덤 닉네임이어야 한다.

## 1차: 게스트 기본 닉네임 생성 SSOT 추가

### 작업내용

- 커뮤니티 게스트 기본 닉네임을 `익명` + 4자리 랜덤 숫자 형식으로 생성한다.
- 같은 브라우저에서는 새로고침 후에도 기본 닉네임이 유지되도록 저장소에 보관한다.
- 실시간 채팅과 커뮤니티 피드의 기본 닉네임 초기값이 같은 유틸을 사용하게 한다.

### 논의 필요

- 없음. 사용자가 `익명` + 랜덤숫자 4자리 형식을 지정했다.

### 선택지

1. sessionStorage 기준으로 탭 세션마다 새 닉네임을 만든다.
2. localStorage 기준으로 브라우저별 기본 닉네임을 유지한다.

### 추천

- 2번. 같은 미로그인 사용자가 새로고침하거나 다시 방문해도 채팅 구분명이 유지되어 UX가 안정적이다.

### 사용자 방향

- 추천 기준으로 진행한다.

## 2차: 적용 및 검증

### 작업내용

- 채팅 메시지 전송 payload의 `guestNickname` 기본값을 새 닉네임 유틸로 교체한다.
- 게시글/댓글 작성 UI의 기본 닉네임도 같은 유틸로 초기화한다.
- 타입검사, lint, 빌드, SSOT 검증을 수행한다.

### 논의 필요

- 없음.

### 선택지

1. 기본값만 교체한다.
2. 닉네임 입력 UI 동기화까지 확장한다.

### 추천

- 1번. 이번 요청의 핵심은 모든 게스트가 같은 기본 이름으로 보이는 문제 해결이며, 입력 UI 확장은 별도 UX 변경으로 분리한다.

### 사용자 방향

- 추천 기준으로 진행한다.
