# YEON Today 이중 기록 카드 단일 메뉴 3차 (완료)

## 목표

한 시간에 기록이 두 개 있을 때 각 삼각형의 무게중심에 기록 정보 묶음을 안정적으로 배치하고, 중복 더보기 버튼을 하나로 통합한다.

## 1차 — 삼각형 무게중심 정보 배치

- 작업내용
  - 카드를 대각선 기준의 두 삼각형으로 분할한다.
  - 첫 번째 삼각형의 무게중심 `(1/3, 1/3)`과 두 번째 삼각형의 무게중심 `(2/3, 2/3)`에 기록 정보를 배치한다.
  - 각 무게중심에 아이콘, 활동명, 한 줄 설명을 하나의 묶음으로 표시한다.
- 논의 필요
  - 정보 묶음은 삼각형 배경의 클리핑 영향을 받지 않으면서 시각적으로 해당 삼각형 안에 있어야 한다.
- 선택지
  - 대각선을 완만한 상하 분할로 바꾼다.
  - 완전한 삼각형 분할을 유지하고 각 무게중심에 정보 묶음을 놓는다.
- 추천
  - 기하학적 중심이 분명한 두 번째 선택지를 적용한다.
- 사용자 방향
  - 삼각형의 무게중심에 아이콘·문자·설명 세트가 들어가야 한다.

## 2차 — 메뉴 진입점 통합

- 작업내용
  - 이중 기록 카드에 표시되는 더보기 버튼을 하나로 통합한다.
  - 단일 메뉴에서 첫 번째·두 번째 기록의 설명 편집과 삭제를 모두 제공한다.
  - 각 기록 행의 우클릭은 해당 기록 메뉴를 여는 기존 동작을 유지한다.
- 논의 필요
  - 터치 환경에서는 우클릭할 수 없으므로 단일 더보기 버튼에서 두 기록을 모두 조작할 수 있어야 한다.
- 선택지
  - 더보기 버튼을 모두 제거하고 우클릭만 유지한다.
  - 더보기 버튼 하나에 두 기록의 작업을 모은다.
- 추천
  - 접근성을 보존하는 단일 더보기 메뉴를 적용한다.
- 사용자 방향
  - 이중 기록 카드에 더보기 버튼이 두 개 보이지 않게 한다.

## 3차 — 실제 화면 검증과 배포

- 작업내용
  - Playwright로 모바일·데스크톱 이중 기록 카드와 단일 메뉴를 확인한다.
  - lint, typecheck, 단위 테스트, 패리티 검증을 실행한다.
  - 커밋·push·PR(main)·병합을 완료한다.
- 논의 필요
  - 없음.
- 선택지
  - 정적 검증만 수행한다.
  - 실제 브라우저 동작과 스크린샷 증거까지 남긴다.
- 추천
  - 실제 브라우저 검증과 전후 스크린샷을 남긴다.
- 사용자 방향
  - 보기 이상한 상태가 남지 않도록 실제 카드 기준으로 확인한다.

## 완료 결과

- 각 삼각형의 무게중심에 `아이콘 + 활동명 + 설명`을 하나의 정보 묶음으로 배치했다.
- 이중 기록 카드의 더보기 버튼을 하나로 통합하고, 두 기록의 편집·삭제 작업을 단일 메뉴에 모았다.
- 각 삼각형을 오른쪽 클릭하면 해당 기록의 메뉴가 열리는 동작은 유지했다.
- Playwright에서 데스크톱·모바일 화면, 단일 버튼, 메뉴 4개 항목, 가로 오버플로 0px를 확인했다.
- 웹 lint, typecheck, 1,146개 테스트, 패리티·스킬·SSOT·diff 검증을 통과했다.
