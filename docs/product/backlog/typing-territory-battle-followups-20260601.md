# 타자 점령전 후속 구현 백로그

기준일: 2026-06-01
대상: Phaser 연출, 재접속 복구, 2~6인/지연 테스트

## 1차: Phaser 점령전 연출

### 작업내용

- `packages/typing-race-engine`에 점령전 보드 전용 Phaser mount API를 추가한다.
- 웹 `/typing-service/territory`에서 서버 snapshot을 엔진에 전달해 보드 색상, 점수, 타이머, 최근 점령 효과를 표시한다.
- 기존 흰 배경 UI 원칙은 유지하고, 픽셀/게임 연출은 플레이 영역 안으로 제한한다.

### 논의 필요

- 캐릭터 스프라이트를 팀별로 즉시 붙일지, 우선 보드 중심 연출로 시작할지.

### 선택지

- A. 보드 중심 Phaser 연출 먼저 구현한다.
- B. 캐릭터/관중/스킬 애니메이션까지 한 번에 구현한다.

### 추천

- A. 서버 규칙과 snapshot 안정성을 먼저 고정한다.

### 사용자 방향

- 추천 기준으로 진행한다.

## 2차: 재접속 복구

### 작업내용

- race-server에서 비정상 이탈 시 일정 시간 재접속을 허용한다.
- 재접속 중 플레이어 상태를 삭제하지 않고 snapshot에 상태를 남긴다.
- 웹 hook에서 `reconnectionToken`을 저장하고 reconnect 우선 시도 후 실패 시 새 방 입장으로 fallback한다.

### 논의 필요

- 재접속 유예 시간 기본값.

### 선택지

- A. 30초 유예.
- B. 60초 유예.

### 추천

- A. 60초 게임에서 30초면 충분하고 stale room 누적을 줄인다.

### 사용자 방향

- 추천 기준으로 진행한다.

## 3차: 2~6인/지연 테스트

### 작업내용

- race-server smoke script를 추가한다.
- 2명, 6명 join/start/submit/capture를 검증한다.
- 제출 간격 지연과 too-fast 거절을 함께 검증한다.
- 재접속 smoke를 포함한다.

### 논의 필요

- CI에 온라인 WebSocket smoke를 기본 포함할지.

### 선택지

- A. 로컬/운영 수동 smoke script부터 둔다.
- B. CI에서 서버를 띄워 자동 실행한다.

### 추천

- A 후 B. 이번 차수는 script와 로컬 검증을 먼저 완료한다.

### 사용자 방향

- 추천 기준으로 진행한다.
