# YEON Today 시간 블록 2중 기록 백로그

## 목표

하루 기록의 한 시간 블록에 최대 두 개의 독립 활동을 기록한다. 첫 기록은 기존처럼 선택 활동을 클릭 즉시 저장하고, 같은 블록을 다시 클릭하면 같은 활동을 포함한 두 번째 기록을 추가한다. 두 기록은 `/` 방향 사선으로 분할해 각각의 활동과 설명을 보여준다.

## 제품 불변조건

- 한 시간 블록의 기록 수는 `0~2개`다.
- 같은 활동을 두 번 기록할 수 있다.
- 두 기록의 설명은 서로 독립적이며 각각 최대 200자다.
- 한 기록일 때는 60분, 두 기록일 때는 각 30분으로 활동별 시간을 계산한다.
- 두 기록이어도 기록률과 `기록 시간`은 한 시간으로 계산한다.
- 기록 순서는 `entryIndex` 0, 1로 서버가 관리하며, 하나를 삭제하면 남은 기록을 0으로 압축한다.

## 1차 — 서버 저장 모델과 마이그레이션

- 작업내용: `today_activity_slots`에 `entry_index`를 추가하고 시간당 최대 두 행을 저장하도록 고유 제약을 확장한다. 기존 데이터는 모두 첫 번째 기록인 0으로 유지한다.
- 논의 필요: 두 기록의 활동 시간을 동시 60분씩 볼지, 시간 블록을 균등 분할할지 결정한다.
- 선택지: 각 60분 중복 집계, 각 30분 균등 집계, 사용자별 분량 입력.
- 추천: 별도 시간 입력 없이 하루 총시간 정합성을 지키는 각 30분 균등 집계를 사용한다.
- 사용자 방향: 한 시간 블록에 최대 두 활동을 넣으며, 같은 활동 두 번도 허용한다. 시간 계산은 추천 기준으로 진행한다.

## 2차 — API 계약과 상태 전이

- 작업내용: 시간 슬롯 응답에 최대 두 개의 `entries`를 추가한다. `entryIndex` 없는 PUT은 빈 순서에 새 기록을 추가하고, `entryIndex`가 있으면 해당 기록의 활동·설명을 수정한다. DELETE는 `entryIndex`가 있으면 한 기록만 삭제한다.
- 논의 필요: 기존 단일 기록 클라이언트와 배포 순서 호환을 어떻게 유지할지 결정한다.
- 선택지: 응답 계약을 즉시 교체, 기존 `activityType/note`를 첫 기록 호환 필드로 유지하며 `entries` 추가.
- 추천: 기존 필드를 첫 기록 기준으로 유지하고 신규 `entries`를 추가해 구버전·신버전 양방향 배포 호환성을 확보한다.
- 사용자 방향: 추천 기준으로 진행한다.

## 3차 — 첫 클릭·두 번째 클릭 UX

- 작업내용: 빈 블록 클릭은 기존처럼 즉시 첫 기록을 추가하고, 한 기록 블록을 다시 클릭하면 선택 활동을 두 번째 기록으로 즉시 추가한다. 두 기록 블록은 추가 동작을 막고 각 분할 영역을 편집 진입점으로 전환한다.
- 논의 필요: 두 기록이 가득 찬 블록에서 세 번째 클릭의 동작을 정한다.
- 선택지: 마지막 기록 교체, 오류 토스트, 추가 동작을 없애고 각 기록 편집만 허용.
- 추천: 예기치 않은 덮어쓰기를 막기 위해 두 영역을 개별 편집 버튼으로 전환하고 `2/2` 상태를 전달한다.
- 사용자 방향: 첫 클릭 즉시 삽입 흐름을 유지하고 두 번째 클릭으로 두 번째 기록을 추가한다.

## 4차 — 사선 분할과 독립 설명

- 작업내용: 두 기록 블록을 좌하단에서 우상단으로 올라가는 `/` 사선으로 분할한다. 위·왼쪽과 아래·오른쪽 영역에 각 활동 아이콘·이름·한 줄 설명을 표시하고, 영역 클릭 시 해당 설명 편집기와 개별 삭제 행동을 연다.
- 논의 필요: 좁은 모바일 블록에서 설명을 몇 줄 노출할지 결정한다.
- 선택지: 설명 숨김, 각 한 줄 말줄임, 각 두 줄 말줄임.
- 추천: 활동명 우선순위를 유지하고 설명은 각 한 줄 말줄임으로 표시한다. 전체 설명은 편집기와 title에서 확인한다.
- 사용자 방향: 두 기록은 사선 경계로 구분하고 같은 활동이어도 설명을 다르게 적을 수 있어야 한다.

## 5차 — 검증과 배포

- 작업내용: Flyway 마이그레이션, Spring 서비스·컨트롤러, Zod 계약·API 클라이언트, 웹 상호작용을 테스트한다. 데스크톱·375px 모바일에서 사선 경계와 터치 영역을 Playwright로 검증한다.
- 논의 필요: 없음.
- 선택지: 정적 검사만 수행, 서버·브라우저 상태 전이까지 통합 검증.
- 추천: 같은 활동 두 번, 서로 다른 설명, 개별 삭제 후 압축, 세 번째 추가 거부까지 실제 요청과 화면으로 검증한다.
- 사용자 방향: 추천 기준으로 진행한다.

## 완료 조건

- 기존 기록은 마이그레이션 후 첫 번째 기록으로 보존된다.
- 빈 블록 첫 클릭과 한 기록 블록 두 번째 클릭이 즉시 서버에 저장된다.
- 같은 활동 두 개와 서로 다른 설명을 한 블록에 저장할 수 있다.
- 두 기록은 `/` 사선과 활동명으로 색상에 의존하지 않고 구분된다.
- 두 기록의 설명을 각각 수정·삭제할 수 있다.
- 두 기록 활동 시간은 각 30분이며 동일 활동 두 개는 합계 60분이다.
- 시간당 세 번째 기록은 서버와 UI 모두 허용하지 않는다.
- 375px 모바일부터 데스크톱까지 가로 넘침과 터치 충돌이 없다.
