# Today 서비스 디자인 시스템 보드 개편

## 목표

`todo.yeon.world`의 Today 화면을 스크린샷처럼 실전형 작업 보드로 개편한다. 새 시각 언어를 임의로 만들지 않고, Yeon 흰 배경 서비스 디자인 시스템의 정규 색상과 공용 UI 패턴만 사용한다.

## 1차

### 작업내용

- 상단 입력 영역을 검색/추가/필터가 한 줄로 스캔되는 작업 바 형태로 정리한다.
- 선택 날짜 기준 통계, 지금 할 일, 추천 작업, Today/Inbox/Done 컬럼을 한 화면에서 비교할 수 있게 재배치한다.
- 우측에 월간 전체보기와 우선 정리 추천 패널을 배치한다.
- 추천 작업은 새 저장 필드 없이 기존 우선순위, 예상 시간, 메모 여부, 상태로 파생 계산한다.
- 새 UI 색상은 `bg-white`, `#111`, `#666`, `#aaa`, `#e5e5e5`, `#fafafa`, 검정 CTA 범위로 제한한다.

### 논의 필요

- 추천 점수 산식을 실제 사용자 취향에 맞춰 조정할지 여부.
- 장기적으로 추천 설정을 저장할지 여부.
- 2시간 이상 예상 시간 옵션을 모델에 추가할지 여부.

### 선택지

- A. 기존 localStorage 모델을 유지하고, 파생 추천/필터/레이아웃만 추가한다.
- B. task 모델에 추천 점수와 설정 필드를 저장한다.
- C. Spring API/DB로 Today 서비스를 이전하면서 추천 로직까지 서버화한다.

### 추천

선택지 A를 추천한다. 지금 목표는 실용적인 개인용 보드이고, 디자인 개편을 위해 영구 모델을 키우면 KISS/YAGNI를 해친다. 기존 필드로도 충분히 추천과 필터를 만들 수 있다.

### 사용자 방향

스크린샷처럼 만들되, 디자인은 임의 색/패턴을 쓰지 말고 Yeon 디자인 시스템 안에서 구현한다.

## 완료 조건

- 스크린샷의 핵심 구조인 입력/필터, 지금 할 일, 추천 작업, 3컬럼 보드, 월간 달력, 정리 추천이 보인다.
- 디자인 시스템 정규 색상 외 신규 hex를 추가하지 않는다.
- 기존 날짜 이동/달력/localStorage 데이터와 호환된다.
- 테스트와 로컬 화면 검증을 통과한다.
