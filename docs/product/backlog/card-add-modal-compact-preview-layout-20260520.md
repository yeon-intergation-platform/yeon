# 카드 추가 모달 스크롤 최소화 레이아웃 (2026-05-20)

## 배경

카드 추가 모달을 브라우저 확대율 100%에서 열었을 때 질문 작성 영역과 답변 작성 영역이 동시에 보이지 않아, 사용자가 카드 한 장을 만들 때마다 모달 내부 스크롤을 반복해야 한다. 현재 직접 작성 화면은 질문 에디터와 답변 에디터가 각각 자체 미리보기를 포함하는 세로 반복 구조라 전체 높이가 길다.

## 1차

### 작업내용

- 직접 작성 화면의 목표를 "스크롤을 덜 해도 질문/답변 작성과 카드 미리보기를 한 번에 확인할 수 있는 카드 제작 화면"으로 둔다.
- 질문/답변 에디터를 왼쪽 작성 컬럼에 함께 배치한다.
- 기존 질문별/답변별 미리보기 중복 구조를 제거하고, 오른쪽에 단일 실시간 미리보기 패널을 둔다.
- 오른쪽 미리보기는 실제 카드처럼 앞면과 뒷면을 동시에 보여주며, 마크다운/이미지 렌더링 결과를 즉시 반영한다.
- 질문/답변 에디터는 카드 추가 전용 compact 높이를 사용하고, 긴 내용은 에디터 내부에서 스크롤되게 한다.
- 취소/저장 액션은 모달 하단에서 항상 접근 가능하게 유지한다.
- 기존 임시저장, 이미지 업로드, 붙여넣기, 드롭, 저장 동작은 유지한다.

### 논의 필요

- 미리보기 카드가 긴 내용을 모두 펼쳐야 하는지, 카드 내부 스크롤로 제한해야 하는지 확인이 필요하다.
- 모바일에서는 데스크톱과 같은 좌우 분할을 적용할 수 없으므로 작성 우선 구조와 미리보기 위치의 균형을 확인해야 한다.

### 선택지

1. 데스크톱에서 왼쪽 작성/오른쪽 앞뒤 카드 미리보기로 재구성하고, 모바일은 세로 스택으로 유지한다.
2. 하나의 플립 카드 미리보기만 두고 앞면/뒷면을 탭으로 전환한다.
3. 현재 구조를 유지하되 에디터 높이와 여백만 줄인다.

### 추천

1번. 사용자의 불편은 질문/답변과 미리보기가 동시에 보이지 않는 구조에서 발생하므로, 미리보기를 단일 오른쪽 패널로 합치고 앞면/뒷면을 동시에 보여주는 편이 스크롤 최소화 목표에 가장 직접적이다.

### 사용자 방향

추천 기준으로 진행한다. 100% 확대율에서 질문/답변 작성 영역과 실시간 카드 미리보기의 핵심이 첫 화면에 함께 보이도록 최적화한다.
