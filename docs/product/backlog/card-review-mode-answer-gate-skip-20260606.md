# 카드 복습 모드 정답 확인/스킵 흐름

## 상태

- 완료

## 배경

- 복습 모드가 처음부터 문제와 정답, 난이도 선택지를 함께 보여 주고 있다.
- 진행 표기가 항상 첫 카드처럼 보이며, 리뷰 후 다음 학습 카드로 진행되지 않는다.
- 난이도 기록 없이 현재 문제를 건너뛰는 스킵 동작과 웹 키보드 단축키가 필요하다.

## 1차

### 작업내용

- 복습 모드는 최초에 문제만 보여 주고, 정답 확인 후 정답과 난이도 선택지를 보여 준다.
- 진행 표기는 `현재 학습 카드수 / 전체 카드수`로 표시하고 `|` 문자를 제거한다.
- 리뷰 저장 성공 후 다음 카드로 이동한다.
- 오른쪽 상단 스킵 버튼을 추가하고, 스킵은 리뷰 난이도를 기록하지 않는다.
- 웹에서 `s`는 스킵, `Space`는 복습 모드 정답 확인에 연결한다.
- 카드 서비스 web/mobile 패리티를 고려해 모바일 복습 화면도 동일한 정답 확인 흐름으로 맞춘다.

### 논의 필요

- 마지막 카드에서 스킵/리뷰 후 순환할지 종료 화면을 둘지 정해야 한다.

### 선택지

- A. 마지막 카드에서 스킵/리뷰 시 첫 카드로 순환한다.
- B. 마지막 카드에서 그대로 머물고 정답 표시 상태만 초기화한다.
- C. 별도 완료 화면을 추가한다.

### 추천

- A. 기존 리뷰 저장 성공이 첫 카드로 돌아가던 동작과 가장 가깝고, 이번 수정 범위를 작게 유지한다.

### 사용자 방향

- 사용자는 복습 모드 정답 확인 흐름, 진행 카운터 수정, `|` 제거, 스킵 버튼, `s`/`Space` 단축키, main 머지를 요구했다.

## 완료 결과

- 웹 복습 모드는 최초에 문제와 `정답 확인하기`만 보여 주고, 정답 확인 후 정답과 난이도 선택지를 보여 준다.
- 웹 진행 표기는 `카드 N/전체`로 표시하며 `|` 문자를 제거했다.
- 웹 리뷰 저장 성공과 스킵 모두 다음 카드로 이동한다. 마지막 카드에서는 첫 카드로 순환한다.
- 웹 스킵은 리뷰 난이도를 저장하지 않는다.
- 웹 단축키는 `Space` = 정답 확인, `s` = 스킵으로 동작한다.
- 모바일 복습 패널도 정답 확인 전에는 답과 난이도 선택지를 숨기고, 복습 중 오른쪽 상단 액션을 `스킵`으로 맞췄다.
