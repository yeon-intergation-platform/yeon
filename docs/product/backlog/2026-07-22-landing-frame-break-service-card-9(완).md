# 랜딩 키보드 타자연습 카드 전경 누끼 교체

> 상태: 완료

## 1차 — 제공 이미지 누끼 에셋 제작

- 작업내용: 사용자 제공 1574×792 이미지에서 외곽 흰 배경을 제거한 투명 WebP 전경을 만든다.
- 논의 필요: 전등 내부 빛, 캐릭터·모니터·키보드·머그컵 픽셀은 보존하고 닫힌 흰 배경 영역만 추가 제거해야 한다.
- 선택지:
  - A. 외곽 연결 영역과 명확한 내부 흰 배경 영역만 투명화한다.
  - B. 모든 밝은 픽셀을 일괄 투명화한다.
- 추천: A. 원본의 밝은 디테일을 훼손하지 않는다.
- 사용자 방향: 1번 카드 전경에 제공 이미지를 누끼 처리해 사용한다.

## 2차 — 로컬 반영 및 검증

- 작업내용: 방 배경과 카드 CSS는 유지하고 새 전경 경로와 실제 크기만 교체한다.
- 논의 필요: 새 가로 구도에서 제목·운영 배지와 전경이 겹치지 않아야 한다.
- 선택지:
  - A. 캐시가 분리되는 v4 파일명을 사용한다.
  - B. 기존 v3 파일을 덮어쓴다.
- 추천: A. 브라우저에서 교체 결과를 즉시 식별한다.
- 사용자 방향: localhost에서 바로 확인할 수 있게 한다.

## 완료 조건

- 새 투명 전경이 원본 픽셀로 렌더링된다.
- 전등 내부 빛과 주요 전경 요소가 보존된다.
- CSS로 누끼나 요소 연장을 만들지 않는다.
- 모바일·데스크톱에서 제목 비겹침과 가로 스크롤 부재를 확인한다.

## 완료 결과

- 제공한 1574×792 이미지에서 외곽 흰 배경과 팔·키보드 사이의 닫힌 흰 영역만 제거했다.
- 전등 내부 빛, 인물, 모니터, 키보드, 머그컵을 원본 픽셀 그대로 보존한 투명 WebP 전경을 만들었다.
- v3 방 배경과 기존 카드 JSX/CSS는 유지하고 `typing-service` 전경 경로와 실제 크기만 v4로 교체했다.
- 375px와 1040px에서 새 전경 로드와 가로 스크롤 부재를 확인했다.
- 기존 제목·운영 배지는 전경보다 위 레이어에서 가독성을 유지한다.
