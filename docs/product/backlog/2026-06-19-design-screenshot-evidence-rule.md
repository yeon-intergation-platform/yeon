# UI 디자인 변경 스크린샷 증거 규칙

## 배경

UI/디자인 변경은 말로만 "고쳤다"고 보고하면 실제 화면에서 어디가 어떻게 바뀌었는지 추적하기 어렵다. 앞으로 서비스 완성도 루프를 돌릴 때 Playwright 기반 스크린샷을 작업 산출물로 남기는 규칙이 필요하다.

## 1차

### 작업내용

- UI/디자인 변경 시 스크린샷을 남기는 공식 가이드를 추가한다.
- AGENTS.md에 해당 가이드를 필수 참조로 연결한다.
- 회고 로그에 스크린샷 없이 완료 보고하는 패턴을 재발 방지 규칙으로 남긴다.

### 논의 필요

- 모든 UI 작업에 desktop/mobile before/after를 강제하면 작은 문구 수정에도 비용이 생긴다.

### 선택지

- 선택지 A: 모든 UI 변경에 무조건 desktop/mobile before/after 4장을 요구한다.
- 선택지 B: 기존 화면 변경은 가능한 한 before/after를 남기고, 신규 화면은 after를 남기며, 불가 시 이유와 대체 증거를 요구한다.
- 선택지 C: 큰 디자인 변경에만 스크린샷을 요구한다.

### 추천

- 선택지 B. 증거를 남기는 기준은 강제하되, 신규 화면/캡처 불가 상황까지 현실적으로 처리한다.

### 사용자 방향

- "개발할때 수정 디자인 화면 어디서 어떻게 됬는지 스크린샷으로 다 남겨놔" 요청에 따라 선택지 B 기준으로 즉시 적용한다.
