# 로컬 모바일 변경 묶음 main 병합

## 배경

- 로컬 `yeon` 워크트리에 모바일 2앱 실행, 카드서비스 모바일, 익명 세션, 접근성/입력 파서 보강 변경이 누적되어 있다.
- 사용자는 `.tmp/dev-all/*.log`는 추적하지 않게 하고, 나머지 변경은 전부 main에 병합하라고 지시했다.

## 1차

### 작업내용

- `.tmp/dev-all/*.log`를 Git 추적에서 제거하고 `.gitignore`로 재추적을 막는다.
- 로컬 모바일/계약/API client/dev-all 변경과 관련 backlog/ai-log를 origin/main 기준 브랜치에 반영한다.
- 모바일 중심 lint/typecheck와 diff check를 수행한 뒤 PR(main)로 병합한다.

### 논의 필요

- 모바일 변경 묶음이 크므로 배포 후 실제 Expo 실행 확인은 사용자가 로컬에서 이어서 확인한다.

### 선택지

1. 로그 제외 후 로컬 변경 전체를 한 PR로 병합한다.
2. 모바일 앱 구성, 카드서비스, dev-all을 여러 PR로 분리한다.
3. 일부 변경만 선별해 병합한다.

### 추천

- 사용자 지시에 따라 1번으로 진행한다. 단, 로그 파일은 추적 제외하고 검증 실패 시 즉시 멈춘다.

### 사용자 방향

- `.tmp/dev-all/*.log`는 아예 추적하지 않게 하고, 나머지 전부 머지한다.
