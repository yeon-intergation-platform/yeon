# 타자 점령전 레퍼런스 기능 지원 백로그

## 1차: 레퍼런스 이미지 자산 고정

- 작업내용
  - 로비/플레이/결과 레퍼런스 이미지를 프로젝트 내부 `screenshot/typing-territory/`에 보관한다.
  - 향후 점령전 UI 구현 시 아래 자산을 기준으로 비교한다.
    - `screenshot/typing-territory/lobby-reference-2026-06-01-201256.png`
    - `screenshot/typing-territory/gameplay-reference-2026-06-01-201426.png`
    - `screenshot/typing-territory/result-reference-2026-06-01-195518.png`
- 논의 필요
  - 레퍼런스의 캐릭터/배경/이모티콘을 직접 asset으로 만들지, 우선 기능 구조만 따라갈지 결정 필요.
- 선택지
  - A. 이미지 자산은 레퍼런스로만 두고, 현재 Yeon 캐릭터/팀 색상 체계로 기능을 구현한다.
  - B. 레퍼런스와 같은 캐릭터/배경/아이콘 리소스를 별도 제작한다.
- 추천
  - A. 현재 저장소에 원본 게임 asset이 없으므로 기능 구조를 먼저 맞추고, 시각 리소스는 별도 제작한다.

## 2차: 로비 기능 지원

- 작업내용
  - 점령전 대기방에 방 정보 패널(방 이름, 게임 타입, 게임 시간, 보너스 게임, 현재 인원)을 제공한다.
  - 팀별 슬롯 UI를 제공하고 참가자 준비 상태를 표시한다.
  - 방장 표시, 프로필 팝오버, 팀 이동, 준비/준비 해제, 방 나가기 버튼을 지원한다.
  - 이모티콘 표시, 채팅/상세보기 로그 패널을 지원한다.
- 논의 필요
  - 기존 타자방 대기방을 확장할지, 점령전 전용 대기방을 별도 route로 만들지 결정 필요.
- 선택지
  - A. 기존 타자방 대기방 안에 점령전 모드 패널을 추가한다.
  - B. `/typing-service/territory/lobby` 같은 전용 대기방을 만든다.
- 추천
  - A. 사용자가 “로비에 참가한다음에 점령전”이라고 지시했으므로 기존 타자방 참가 상태를 source of truth로 유지한다.
- ## 사용자 방향

## 3차: 플레이 기능 지원

- 작업내용
  - 팀 점수판, 좌우 팀 랭킹, 중앙 보드, 현재 차례/지시 말풍선, 카운트다운을 표시한다.
  - “뒤집은 판” 수, 입력창, 입력하기 버튼을 하단 조작부로 고정한다.
  - 팀별 색상은 레퍼런스의 분홍/노랑 또는 주황/보라 계열을 UI 라벨과 일관되게 매핑한다.
- 논의 필요
  - 현재 단어 점령 방식과 레퍼런스의 카드 뒤집기 방식 사이의 게임 규칙 차이를 어디까지 맞출지 결정 필요.
- 선택지
  - A. 현재 단어 점령 규칙은 유지하고 카드 뒤집기 형태로 시각화한다.
  - B. 실제 카드 뒤집기 턴제 규칙으로 서버 프로토콜을 변경한다.
- 추천
  - A. 기존 seed/protocol 무결성을 지키며 화면 경험부터 맞춘다.
- ## 사용자 방향

## 4차: 결과 기능 보강

- 작업내용
  - 승리 배지, 팀별 판 점수/보너스/팀 점수/내 순위/획득 포인트를 표시한다.
  - 좌우 팀 랭킹을 표시하고, 대기방 자동 복귀 카운트다운을 제공한다.
  - 하단 방 나가기/대기방 액션을 지원한다.
- 논의 필요
  - 보너스 게임 점수와 획득 포인트를 서버 원천 필드로 분리할지 결정 필요.
- 선택지
  - A. 현재 점령 수/점수에서 파생해 표시한다.
  - B. 서버 result breakdown 필드를 추가한다.
- 추천
  - B. 결과 화면은 사용자 신뢰가 중요하므로 장기적으로 서버 result breakdown을 SSOT로 둔다.
