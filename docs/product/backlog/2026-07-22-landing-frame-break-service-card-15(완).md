# 랜딩 커뮤니티 카드 전용 비주얼 적용

> 상태: 완료

## 1차 — 커뮤니티 완성형 비주얼 적용

- 작업내용: 사용자가 제공한 커뮤니티 이미지를 WebP 에셋으로 최적화하고 4번 커뮤니티 카드의 이미지 영역에 적용한다.
- 논의 필요: 이미지는 인물과 채팅 말풍선·공지 패널이 합쳐진 완성형이므로 프레임브레이크 전경으로 분리하지 않고 한 장의 카드 비주얼로 사용해야 한다.
- 선택지:
  - A. 커뮤니티 카드만 전용 완성형 비주얼을 사용하고 다른 카드의 공통 배경·전경 구조를 유지한다.
  - B. 인물 누끼를 새로 분리하고 말풍선 UI를 제거한다.
- 추천: A. 제공된 커뮤니티 맥락과 정보 밀도를 그대로 보존하면서 변경 범위를 최소화한다.
- 사용자 방향: 제공 이미지를 4번 커뮤니티 카드에 넣는다.

## 2차 — 반응형 크롭 및 회귀 검증

- 작업내용: 375px·1040px·1440px에서 커뮤니티 이미지의 인물, 말풍선, 공지 패널 노출과 카드 레이아웃을 확인한다.
- 논의 필요: 카드의 가로 비율 때문에 원본 상하단 일부가 잘릴 수 있으므로 핵심 콘텐츠가 남는 `object-position`을 선택해야 한다.
- 선택지:
  - A. 카드 영역을 채우는 `object-cover`와 시각 중심 위치를 조정한다.
  - B. `object-contain`으로 원본 전체를 보이되 넓은 여백을 허용한다.
- 추천: A. 기존 카드의 꽉 찬 이미지 리듬을 유지하면서 인물과 주요 커뮤니티 UI가 보이게 맞춘다.
- 사용자 방향: localhost에서 즉시 확인할 수 있도록 기존 개발 서버를 유지한다.

## 완료 조건

- 4번 커뮤니티 카드에 제공 이미지가 표시된다.
- 1~3번 프레임브레이크와 5~8번 공통 배경은 바뀌지 않는다.
- 모바일·데스크톱에서 인물과 핵심 말풍선이 과도하게 잘리지 않고 가로 스크롤이 없다.
- 관련 검증과 Playwright 전후 스크린샷을 남긴다.

## 완료 결과

- 제공된 1536×1024 PNG를 172KB WebP로 최적화해 커뮤니티 전용 비주얼로 적용했다.
- 커뮤니티에만 배경 경로와 대체 텍스트를 선언하고 기존 카드 렌더링 구조를 재사용했다.
- 375px·1040px에서는 원본 비율에 가깝게 노출되고, 1440px 3열에서는 인물·주요 말풍선·공지 패널을 남긴 채 상하단만 제한적으로 잘린다.
- 공통 배경 7개, 커뮤니티 전용 비주얼 1개, 타자·백지·플래시카드 전경 각 1개가 유지됨을 Playwright DOM으로 확인했다.
- 세 뷰포트 모두 문서 너비와 viewport 너비가 일치해 가로 스크롤이 없다.
