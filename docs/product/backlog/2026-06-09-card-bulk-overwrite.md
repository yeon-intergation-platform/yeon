# 카드 일괄 추가 덮어쓰기 백로그

## 배경

카드 덱에서 AI가 만든 카드 묶음을 일괄 추가할 수 있지만, 기존 카드 전체를 지우고 새 묶음으로 다시 맞추는 흐름이 없다. 사용자가 실제 사용 중 기존 카드를 하나씩 삭제하는 과정이 번거롭다고 확인했으므로, 일괄 추가 옆에 덮어쓰기 액션을 추가한다.

## 1차

### 작업내용

- 일괄 카드 입력 화면의 기존 `N장 추가` 버튼 옆에 `덮어쓰기` 버튼을 추가한다.
- 덮어쓰기는 현재 덱의 기존 카드를 모두 삭제한 뒤, 파싱된 일괄 카드 목록으로 덱을 완전히 대체한다.
- 카드 item repository 포트에 덮어쓰기 동사를 추가해 web/mobile 의미를 동일하게 맞춘다.
- 게스트/인증 분기 모두에서 동작하게 한다.

### 논의 필요

- 덮어쓰기 전 확인창 문구와 모바일 확인 UX 수준.

### 선택지

1. 웹에만 버튼을 추가한다.
2. 웹 버튼과 공용 repository 포트/모바일 액션까지 함께 추가한다.
3. 백엔드에 전용 replace API를 새로 추가한다.

### 추천

- 2번. Universal UI 패리티상 카드 item 저장 동사는 shared-contract이므로 공용 포트에 의미를 추가하고 양 플랫폼 어댑터를 맞춘다. 백엔드 전용 API는 이번 요구보다 크므로 기존 delete/add 조합으로 처리한다.

### 사용자 방향

- 추천 기준으로 진행한다.
