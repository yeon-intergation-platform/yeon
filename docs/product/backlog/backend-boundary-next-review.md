# Next 백엔드 역할 경계 및 모바일 Expo 경고 정리 백로그

## 1차: Expo SDK 패치 버전 정합화

- 작업내용
  - `apps/mobile`에서 Expo가 기대하는 `expo`, `expo-linking` 패치 버전으로 의존성을 올린다.
  - 모바일 타입체크로 앱 런타임 경고 해소 범위를 검증한다.
- 논의 필요
  - 없음. Expo CLI가 명시한 패치 버전 정합화 작업이다.
- 선택지
  - A. Expo CLI 권장 패치 버전으로 바로 업데이트한다.
  - B. 경고만 문서화하고 버전은 유지한다.
- 추천
  - A. 패치 버전 차이로 발생한 호환성 경고이므로 의존성 정합화를 우선한다.
- 사용자 방향
  - 추천 기준으로 진행.

## 2차: origin/main 기준 Next 백엔드 역할 리뷰

- 작업내용
  - `origin/main` 기준으로 Next route handler, `apps/web/src/server`, Spring client 사용 현황을 분류한다.
  - 현재 구현된 커뮤니티/채팅/카드/타이핑 관련 도메인 구조가 확장 가능한지, 임시 땜빵인지 판단하고 필요한 최소 수정을 한다.
- 논의 필요
  - 이미 Next가 직접 DB/서비스 레이어를 보유한 구간이 남아 있어 즉시 전면 이관은 위험하다.
- 선택지
  - A. 신규 정책은 SSOT에 고정하고, 현존 Next 백엔드 역할은 리뷰 문서와 이관 후보로 분류한다.
  - B. 이번 작업에서 모든 Next 백엔드 역할을 Spring으로 즉시 이동한다.
- 추천
  - A. 전면 이관은 대규모 마이그레이션이므로, 이번 차수는 경계 명시와 실태 감사, 신규 개발 차단을 완료한다.
- 사용자 방향
  - 추천 기준으로 진행.

## 3차: 에이전트 SSOT 경계 명시

- 작업내용
  - 전역/프로젝트 에이전트가 참조하는 SSOT에 “Next는 백엔드 역할을 추가로 맡지 않고, 백엔드 역할은 Spring만 담당한다”는 정책을 명시한다.
  - 기존 Next route handler는 Spring 이전 전까지 호환/프록시/BFF 역할만 허용한다고 적는다.
- 논의 필요
  - 없음. 사용자가 명시 요청했다.
- 선택지
  - A. 프로젝트 `AGENTS.md`와 서버 규칙 SSOT에 모두 반영한다.
  - B. `AGENTS.md`에만 반영한다.
- 추천
  - A. 에이전트 온보딩과 세부 서버 규칙 모두에서 같은 정책을 보게 한다.
- 사용자 방향
  - 추천 기준으로 진행.
