# Search Console 제출 대상 누락 방지 검증

## 1차

작업내용: Search Console sitemap 제출 스크립트와 운영 문서가 같은 host 목록을 말하는지 검증하는 스크립트를 추가하고, 게임 호스트 반영 뒤 남은 문서 불일치를 정리한다.

논의 필요: 검증을 어느 게이트에 묶을지 결정해야 한다. 처음에는 수동 실행 가능한 repo 검증으로 추가하고, 반복 누락이 생기면 `verify-ssot.sh` 또는 CI에 연결한다.

선택지:

- A. `bin/verify-search-console-targets.mjs`를 추가해 제출 스크립트 dry-run 출력과 운영 문서를 대조한다.
- B. 제출 대상 목록을 별도 JSON으로 분리해 스크립트와 문서 생성까지 자동화한다.

추천: A. 기능 변경 없이 운영 누락을 줄이는 목적에는 검증 스크립트가 충분하고, JSON 분리는 문서 생성 방식까지 바꾸는 구조 변경이다.

사용자 방향: 추천 기준으로 진행한다.
