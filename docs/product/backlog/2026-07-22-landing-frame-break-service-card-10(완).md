# 랜딩 타자연습 카드 전등 잘림 보정

> 상태: 완료

## 1차 — 전경 캔버스 배치 보정

- 작업내용: v4 누끼 전경의 실제 픽셀은 유지하고 투명 캔버스 여백으로 전등이 카드 왼쪽 경계에 잘리지 않게 조절한다.
- 논의 필요: 인물 크기와 제목 안전영역을 유지하면서 전등 전체를 보여야 한다.
- 선택지:
  - A. 전경 왼쪽에 투명 여백을 추가해 전체 구도를 카드 안쪽으로 이동한다.
  - B. CSS 도형으로 잘린 전등을 연장한다.
- 추천: A. 원본 픽셀만 사용하고 CSS 가짜 요소를 만들지 않는다.
- 사용자 방향: 전등이 잘리지 않게 하되 CSS로 전등 윗부분을 만들지 않는다.

## 2차 — 반응형 시각 검증

- 작업내용: 375px과 1040px에서 전등 경계, 제목·상태 가독성, 가로 스크롤을 확인한다.
- 논의 필요: 투명 캔버스 확대로 전경이 지나치게 작아지지 않아야 한다.
- 선택지:
  - A. 최소 여백만 추가하고 기존 공용 레이아웃을 유지한다.
  - B. 서비스별 CSS 위치 예외를 추가한다.
- 추천: A. 구조 분기 없이 에셋 자체의 안전 여백으로 해결한다.
- 사용자 방향: localhost에서 바로 확인할 수 있게 한다.

## 완료 조건

- 모바일 카드에서 전등 외곽이 왼쪽 경계에 잘리지 않는다.
- 전등은 원본 이미지 픽셀만 사용한다.
- 카드 JSX/CSS에 전등 생성·연장 코드를 추가하지 않는다.
- 제목·상태 가독성과 가로 스크롤 부재를 유지한다.

## 완료 결과

- v4 누끼 전경의 원본 픽셀을 180px 오른쪽으로 이동한 1574×792 v8 전경을 만들었다.
- 전등 갓, 상단 관절, 세로 암이 모바일 카드 왼쪽 경계 안에 들어오도록 조절했다.
- 이미지 경로만 v8로 변경했으며 카드 JSX/CSS와 전등 픽셀은 수정하지 않았다.
- 375px와 1040px에서 전등 전체 노출, 제목·상태 가독성, 가로 스크롤 부재를 확인했다.
