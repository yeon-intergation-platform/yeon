# 카드방/타자방 음성통화 백로그 (lane-yeon-3)

## 목적

카드방·타자방에서 텍스트 채팅과 독립적으로 음성통화 기능을 추가해 실시간 협업 체감도를 높인다.

## 작업 방식

- 하나의 구현 lane(yeon-3)에서 **순차적으로 전부 진행**
- 병렬 분할 없이 설계→구현→검증을 한 흐름으로 완결

## 적용 범위

- 룸 기반 음성통화 UI: 시작/종료/음소거/재시도
- 브라우저 마이크 권한 처리 및 상태표시
- 실시간 상태 동기화(통화중/대기중/오류)
- 텍스트 채팅 동시 사용 보장

## 비적용

- 영상통화, 녹음 저장
- 운영자용 통화 품질 대시보드

## 핵심 전제

- 1:1 통화 MVP로 시작
- 통화 신호/시그널 계약은 yeon-3에서 선행 확정 후 구현 반영
- 운영은 베타 플래그 기반으로 단계 확대

## 성공 기준

- 같은 룸 내 최소 1:1 음성통화 성공 연결
- 통화 실패 시 명확한 fallback 메시지와 텍스트 채팅 지속
- 주요 브라우저에서 권한 거부 시 사용자 안내 정상 동작

## 회귀 주의

- 채팅, 참가자 상태, 룸 이동/퇴장 플로우가 음성통화 중에도 안정적으로 유지되어야 함

## 1차 — 공통 계약/시그널 정의

### 작업내용

- `packages/race-shared`에 `VOICE_EVENTS`, 통화 상태, SDP/ICE/종료/음소거 메시지 타입을 추가한다.
- 서버/웹이 같은 이벤트 문자열과 payload 타입을 참조하도록 한다.

### 논의 필요

- 장기적으로 서버 룸 상태 snapshot에 통화 표시를 포함할지 여부.

### 선택지

1. 이벤트 relay 중심으로 MVP 구현 후 UI는 클라이언트 세션 상태에서 파생
2. 서버 room state에 통화 상태까지 영속 반영

### 추천

- 선택지 1. WebRTC 시그널은 일시 상태이므로 race-server room 메모리 relay에 한정한다.

### 사용자 방향

- 추천 기준으로 진행.

## 2차 — race-server 실제 릴레이

### 작업내용

- 카드방/타자방 Colyseus room에 offer/answer/ICE/end/mute relay를 추가한다.
- 중복 통화, 자기 자신 호출, 대상 없음, 미응답 timeout, 퇴장/network 종료를 방어한다.
- answer 이후에는 초대 timeout을 해제해 정상 통화가 30초 후 강제 종료되지 않게 한다.

### 논의 필요

- 운영 이후 TURN 서버 도입 시점.

### 선택지

1. STUN 기반 브라우저 P2P로 MVP 출시
2. TURN 인프라까지 먼저 추가

### 추천

- 선택지 1. 기능 플래그 뒤에서 실제 제품 경로로 배포하고 실패 케이스를 관측한다.

### 사용자 방향

- 추천 기준으로 진행.

## 3차 — 웹 클라이언트/룸 UI 통합

### 작업내용

- 공통 `useRoomVoiceCall` 훅과 `RoomVoiceCallPanel`을 추가한다.
- 카드방과 타자방 대기/레이스 화면에서 실제 Colyseus room 인스턴스와 실제 참가자 목록을 사용한다.
- 마이크 권한 거부, 브라우저 미지원, 통화 거절/종료/재시도 안내를 한국어로 제공한다.

### 논의 필요

- 패널 배치가 카드/채팅 우선순위를 방해하는지 실제 화면에서 조정 필요.

### 선택지

1. 기존 룸 패널 안에 작게 삽입
2. 통화 전용 overlay로 분리

### 추천

- 선택지 1. 텍스트 채팅과 룸 흐름을 유지하는 MVP 목적에 맞다.

### 사용자 방향

- 추천 기준으로 진행.

## 4차 — 기능 제어/검증

### 작업내용

- `.env.example`에 `NEXT_PUBLIC_ENABLE_ROOM_VOICE_CALL=false`를 추가한다.
- 타입체크, lint, race-server build, web production build로 회귀를 검증한다.

### 논의 필요

- 실제 운영 활성화 시 `true`를 적용할 환경(dev/prod) 범위.

### 선택지

1. dev 환경부터 true
2. prod 내부 베타까지 true

### 추천

- 선택지 1 후 수동 브라우저 2대 통화 검증을 거쳐 prod 내부 베타로 확대.

### 사용자 방향

- 추천 기준으로 진행.
