# 랜딩 8개 서비스 카드 배경 통일

> 상태: 완료

## 1차 — 백지학습 배경을 공통 SSOT로 승격

- 작업내용: 백지학습 카드의 배경 이미지를 랜딩 서비스 카드 공통 배경 상수로 정의한다.
- 논의 필요: 카드별 배경 경로를 복제하면 이후 일부 카드만 달라질 수 있으므로 단일 경로에서 파생해야 한다.
- 선택지:
  - A. 공통 배경 경로를 한 번 선언하고 모든 카드가 참조한다.
  - B. 8개 카드 설정에 동일 문자열을 반복한다.
- 추천: A. 배경 일관성이 코드 구조에서도 강제된다.
- 사용자 방향: 2번 백지학습 배경을 1~8번 모두에 적용한다.

## 2차 — 4~8번 카드에 공통 비주얼 영역 적용

- 작업내용: 현재 전경이 없는 커뮤니티·오늘 보드·Discord AI·YEON 뉴스·게임 카드에도 같은 배경 비주얼 영역과 순번을 적용한다.
- 논의 필요: 전경이 없는 카드에서도 제목·상태·설명·CTA 위계와 카드 높이를 일관되게 유지해야 한다.
- 선택지:
  - A. 전경은 선택값으로 두고 공통 배경 카드 구조를 8개 모두 사용한다.
  - B. 1~3번만 배경을 통일하고 4~8번은 기존 텍스트 카드로 남긴다.
- 추천: A. 사용자가 요청한 8장 전체 배경 일관성을 충족한다.
- 사용자 방향: 8장이 모두 같은 배경을 사용한다.

## 3차 — 반응형 시각 검증과 배포

- 작업내용: 375px·1040px·1440px에서 8개 카드의 배경, 순번, 카드 높이, 가로 스크롤을 확인하고 main PR에 반영한다.
- 논의 필요: 동일 배경 반복이 제목·상태·CTA 가독성을 방해하지 않아야 한다.
- 선택지:
  - A. 현재 프레임브레이크 카드의 흰색 헤더와 본문을 유지한다.
  - B. 배경 위에 텍스트를 직접 올린다.
- 추천: A. 이미지 일관성만 바꾸고 정보 위계와 대비는 유지한다.
- 사용자 방향: 배경 외의 기존 전경과 카드 콘텐츠는 유지한다.

## 완료 조건

- 1~8번 카드가 모두 `baekji-frame-break-background.webp`를 사용한다.
- 배경 경로는 한 곳에서만 선언한다.
- 1~3번 전경과 기존 세로 위치는 유지된다.
- 4~8번에도 순번과 동일한 이미지 영역이 표시된다.
- 모바일·데스크톱에서 가로 스크롤과 콘텐츠 가림이 없다.

## 완료 결과

- `LANDING_CARD_BACKGROUND_SRC` 한 곳에서 백지학습 배경 경로를 선언하고 1~8번 카드가 모두 참조하도록 변경했다.
- 커뮤니티·오늘 보드·Discord AI 어시스턴트·YEON 뉴스·게임에도 4~8번 순번과 동일한 배경 이미지 영역을 적용했다.
- 전경을 선택값으로 분리해 1~3번의 기존 캐릭터·소품 전경과 세로 위치는 그대로 유지했다.
- 더 이상 참조되지 않는 타자연습 v3 배경과 플래시카드 덱 전용 배경 에셋을 제거했다.
- Playwright DOM 검사에서 375px·1040px·1440px 모두 동일 배경 이미지 8개와 가로 스크롤 부재를 확인했다.
