# 랜딩 4·5번 프레임브레이크 구도 재작업

> 상태: 완료

## 1차 — 프레임브레이크 실패 원인 교정

- 작업내용: 커뮤니티와 YEON Today 전경을 1·2·3번과 같은 와이드 비율과 낮은 책상선으로 다시 제작한다.
- 논의 필요: 현재 전경은 3:2 세로 비중이 크고 모니터·책상이 카드 이미지 영역을 덮어 공통 배경과 돌출 경계가 보이지 않는다.
- 선택지:
  - A. 기존 전경의 CSS 크기만 줄인다.
  - B. 전경 자체를 약 2.3:1 와이드 구도로 재구성해 인물·전등만 헤더로 돌출시키고 배경 노출 면적을 확보한다.
- 추천: B. CSS 축소만 하면 인물까지 작아지고 카메라 중심 문제는 남는다.
- 사용자 방향: 1·2·3번과 같은 완성도의 명확한 프레임브레이크로 다시 만든다.

## 2차 — 카메라 뷰와 서비스별 시각 중심 보정

- 작업내용: 4번은 인물 중심과 핵심 말풍선만 남기고, 5번은 인물·할 일 보드를 중심으로 두며 모니터를 축소하고 의자를 더 노출한다.
- 논의 필요: 원본의 모든 소품을 유지하면 작은 카드에서 주제가 분산되고 공통 배경이 가려진다.
- 선택지:
  - A. 원본 전체 장면을 동일 비율로 축소한다.
  - B. 서비스 정체성을 나타내는 최소 소품만 남긴 전경 구도로 재편집한다.
- 추천: B. 1·2·3번처럼 한눈에 인물 행동이 읽히고 공통 배경이 시각적으로 연결된다.
- 사용자 방향: 인물은 카드 중앙 쪽, 모니터는 덜 보이고 의자는 더 보이는 카메라 뷰로 맞춘다.

## 3차 — 반응형 비교 검증

- 작업내용: 375px·1040px·1440px에서 1·2·3번과 4·5번의 전경 크기, 돌출 높이, 배경 노출, 제목 충돌을 비교한다.
- 논의 필요: 4·5번만 따로 좋아 보여도 첫 줄 카드와 시각 리듬이 다르면 통일 목적을 달성하지 못한다.
- 선택지:
  - A. 4·5번 카드만 확인한다.
  - B. 1~5번을 같은 화면에서 비교하고 수치·스크린샷을 함께 남긴다.
- 추천: B. 프레임브레이크 완성도는 인접 카드와의 상대 비교가 핵심이다.
- 사용자 방향: localhost에서 바로 확인할 수 있게 유지한다.

## 완료 조건

- 4·5번에서 공통 배경이 충분히 보이고 인물·전등이 헤더 경계를 명확히 넘는다.
- 4번의 말풍선과 5번의 모니터가 주인공보다 강하게 보이지 않는다.
- 인물은 카드 중앙에 가깝고 의자 외곽은 자연스럽게 노출된다.
- 제목·상태 배지와 전경이 충돌하지 않으며 가로 스크롤이 없다.

## 완료 결과

- 4번 전경을 1455×711 와이드 에셋으로 교체해 인물·전등·의자와 핵심 커뮤니티 UI만 남겼다.
- 5번 전경을 1479×748 와이드 에셋으로 교체해 인물과 할 일 보드를 중앙에 두고 모니터 비중을 줄였다.
- 375px·1040px·1440px에서 공통 배경 노출, 헤더 돌출, 제목 충돌, 가로 스크롤을 확인했다.
- 공통 배경의 SHA-256이 `origin/main`과 동일함을 확인했다.
