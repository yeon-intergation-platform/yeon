# 카드방 공통 마크다운 렌더러 적용 및 덱 복귀 링크 수정 (2026-05-18)

## 배경

카드 상세 화면의 카드 목록에서는 HTML/마크다운 내용이 정상적으로 렌더링되지만, 카드방 학습 화면에서는 같은 카드 내용이 `<p>...</p>` 문자열로 그대로 표시된다. 같은 카드 본문을 화면마다 새로 구현하면서 렌더링 규칙이 갈라진 상태라 공통 렌더러로 통일해야 한다. 또한 덱 상세의 “내 덱으로 돌아가기” 링크는 `/card-service`가 아니라 덱 목록 `/card-service/decks`로 이동해야 한다.

## 1차

### 작업내용

- 덱 상세의 “내 덱” 복귀 링크 href를 `/card-service/decks`로 수정한다.
- 카드방 학습 카드의 앞면/뒷면 렌더링에서 raw HTML 문자열을 그대로 출력하지 않도록 수정한다.
- 카드서비스에서 이미 사용하는 공통 `MarkdownContent` 렌더러를 카드방에도 재사용한다.
- 새 하드코딩 렌더링 분기를 만들지 않고 공통 모듈 의존으로 정리한다.

### 논의 필요

- 카드방의 큰 폰트/중앙 정렬 학습 카드 스타일을 어디까지 유지할지 확인이 필요하다.
- 추후 카드서비스 전체 렌더러 위치를 `features/card-service/components`에서 더 공용 경로로 승격할지 검토가 필요하다.

### 선택지

1. 카드방에서 기존 `MarkdownContent`를 import해 즉시 렌더링 규칙을 통일한다.
2. 렌더러를 `features/card-service/components` 밖 공용 위치로 이동한 뒤 양쪽을 모두 재연결한다.
3. 카드방 전용 sanitize/render 로직을 새로 작성한다.

### 추천

1번. 현재 버그는 raw HTML 렌더링 누락이므로 기존 렌더러를 재사용하는 좁은 수정이 가장 안전하다. 공용 위치 승격은 별도 구조 정리로 분리한다.

### 사용자 방향

추천 기준으로 진행한다.
