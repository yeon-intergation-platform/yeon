# 2026-05-17 모바일 안드로이드 이중 앱 출시 구성

## 차수 1

### 작업내용

- `apps/mobile`을 단일 코드베이스로 익명친구 앱(`anonymous`)과 카드서비스 앱(`card`)을 분기 실행하도록 정리.
- `pnpm dev:all`에서 익명친구/카드 서비스 모바일 dev 서버를 동시에 띄우도록 포트 분기 포함(`MOBILE_PORT`, `MOBILE_CARD_PORT`).
- 익명친구 앱에서 전화번호 OTP 인증 절차를 건너뛰도록 환경변수 플래그를 추가 (`EXPO_PUBLIC_SKIP_ANONYMOUS_CHAT_PHONE_AUTH`).
- API 클라이언트/세션 토큰 헤더 분기(게스트 세션: `guest:<profileId>` 처리 및 `X-Yeon-Chat-Profile-Id`).

### 논의 필요

- 실제 배포(앱스토어 업로드) 시 앱 ID/번들명/스크림 정책을 어느 값으로 고정할지.

### 선택지

1. 카드/익명친구 모두 운영 빌드에서 동일 앱 ID 사용(단일 앱): 빌드 충돌 위험 감소.
2. 앱 스토어용으로 앱 ID 분리(현재 코드 기준): 사용자 오인 방지·독립 배포 가능.

### 추천

- 운영 배포는 앱 ID/slug 분리(현재 구현 방향 유지)로, 두 앱을 명확히 분리 배포.

### 사용자 방향

- `현재 익명친구는 인증 스킵이 기본, 카드서비스는 기존 인증 플로우 유지`로 진행.
