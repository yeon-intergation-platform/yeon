# 커뮤니티 채팅/게스트 인증 사이드바 개선

## 1차

### 작업내용

- 커뮤니티 채팅 메시지의 시간 표시 위치와 간격을 정리하고 날짜까지 보이게 한다.
- 채팅 영역에 "채팅은 3일 뒤 사라진다"는 보존기간 안내를 명확히 보여준다.
- community-chat 메시지 목록은 3일 보존기간 안의 메시지만 반환하고, 만료 메시지는 Spring backend에서 정리한다.
- 게스트 닉네임/비밀번호 입력을 글쓰기 버튼 옆 inline 영역에서 분리하고 오른쪽 사이드바의 작은 카드로 옮긴다.
- 닉네임/비밀번호를 등록하면 글/댓글 작성·수정·삭제 흐름에서 자동으로 사용되게 한다.
- 모바일에서는 오른쪽 사이드바가 본문 위/아래의 별도 카드로 접히며, 글쓰기 버튼과 인증 입력이 결합되지 않게 한다.

### 논의 필요

- community-chat 3일 보존기간은 운영 정책으로 문서화할지 별도 결정이 필요하다.
- 게스트 비밀번호를 장기 저장하면 보안 위험이 있으므로 이번 작업은 클라이언트 로컬 저장 기반의 편의 기능으로 한정하고, 서버 인증 원천은 기존 Spring 검증을 유지한다.

### 선택지

- A. Spring repository/service에서 3일 이전 채팅을 목록에서 제외하고 삭제하며, 웹 UI는 안내/표시/인증 카드만 담당한다.
- B. 웹 query에서만 3일 이전 메시지를 숨긴다.
- C. DB schema에 TTL 컬럼/스케줄러를 추가한다.

### 추천

- A. Next.js가 장기 상태의 원천이 되면 안 되고, B는 실제 데이터가 남아 정책과 어긋난다. C는 현재 요구보다 무겁다. Spring repository/service에 cutoff와 cleanup을 두는 방식이 가장 작고 명확하다.

### 사용자 방향

- 추천 기준으로 진행한다.

### 실행 결과

- Spring community-chat에 3일 cutoff 조회와 만료 메시지 정리를 추가한다.
- 웹 채팅 헤더에 3일 보존 안내를 표시하고 timestamp를 날짜+시간 meta 행으로 정리한다.
- 게스트 닉네임/비밀번호를 글쓰기 영역에서 제거하고 우측/모바일 상단 인증 카드로 분리한다.
- 저장된 인증값은 글/댓글 작성·수정·삭제에 자동 사용한다.
- Playwright 검증 중 모바일 가로 오버플로와 저장된 인증값 하이드레이션 mismatch를 발견해 함께 수정한다.
