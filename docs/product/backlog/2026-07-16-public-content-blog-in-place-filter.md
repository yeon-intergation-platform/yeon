# 블로그 홈 인플레이스 카테고리 필터

## 목표

블로그 홈의 `기술 글`·`제품 글`·`개발 일지`·`에세이`를 컬렉션 페이지 이동이 아닌 같은 목록 화면의 필터로 동작하게 한다. 선택 상태는 URL에 보존하되, 카드 영역만 바뀌고 스크롤 위치와 헤로는 유지한다. 로컬에서만 확인하며 PR, push, `main` 병합, 배포는 하지 않는다.

## 1차 — URL 기반 필터 상태

### 작업내용

- 필터 URL을 `/blog?category=<category>`로 통일하고, 기본 `전체`는 쿼리를 제거한다.
- 유효한 카테고리만 파싱해 목록에 적용하고, 잘못된 값은 기본 전체 목록으로 안전하게 처리한다.
- 선택된 카테고리는 해당 카테고리의 모든 글을 보여 주고, 전체는 최근 글 일부를 유지한다.
- 같은 화면 전환이므로 `router.replace`와 `scroll: false`를 사용해 히스토리와 스크롤 점프를 막는다.

### 논의 필요

- 필터 전환에 별도 애니메이션을 넣을지 여부.

### 선택지

1. 카드 그리드 전체에 전환 애니메이션을 넣는다.
2. 위치와 헤로를 고정하고 목록 내용과 선택 상태만 즉시 바꾼다.

### 추천

2. 블로그의 편집물 같은 정돈된 분위기를 유지하고, 필터라는 사용자 기대와 가장 잘 맞는다.

### 사용자 방향

제품 글 같은 분류는 화면이 확 바뀌는 페이지 이동이 아니라 같은 목록의 필터로 동작해야 한다.

## 진행 결과

- 카테고리 항목을 컬렉션 링크 대신 실제 `button` 필터로 전환했다.
- `제품 글` 선택 시 `/blog?category=product`만 교체하고, 블로그 홈 헤더·필터·스크롤 위치는 유지한다.
- URL 직접 접근과 새로고침에서도 선택 상태와 목록이 복구된다.
- 데스크톱·모바일 화면 증빙은 `ai-log/hyeonjun/2026-07-16/블로그-인플레이스-필터-screenshots/`에 저장했다.
- 로컬 검증만 완료했으며 PR, push, `main` 병합, 배포는 수행하지 않는다.
