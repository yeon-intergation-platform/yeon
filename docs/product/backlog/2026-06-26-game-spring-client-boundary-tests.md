# 게임 Spring client 경계 테스트 보강

## 1차

### 작업내용

- 게임 좋아요, 라이브러리, 경험치 Spring client의 URL/헤더/본문/오류 처리 테스트를 추가한다.
- 기능 로직은 변경하지 않고 Spring BFF 호출 계약을 테스트로 고정한다.

### 논의 필요

- 없음. 운영 장애 원인 추적을 돕는 테스트 보강이다.

### 선택지

- A. client 단위 테스트로 Spring BFF 호출 계약을 고정한다.
- B. route와 client 통합 테스트로 확장한다.
- C. 실제 Spring 서버를 띄운 E2E를 추가한다.

### 추천

- A. 빠르게 회귀를 막고 기존 기능 동작을 바꾸지 않는다.

### 사용자 방향

- 추천 기준으로 진행한다.
