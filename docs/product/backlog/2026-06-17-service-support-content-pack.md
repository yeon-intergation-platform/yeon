# 3종 서비스 support 콘텐츠팩

## 배경

공개 콘텐츠 정책은 `support.yeon.world`, `news.yeon.world`, `blog.yeon.world`를 분리한다. 사용자가 동의한 최종 구조는 support를 문제 해결형 검색 유입 채널로 두고, news는 공식 소식, blog는 개발기와 기술 글로 운영하는 방식이다.

NEXA 초기 support 문서는 먼저 확장했다. 다음 차수에서는 Yeon 유지보수 대상 서비스인 타자, 카드, 커뮤니티의 검색형 도움말을 서비스별로 늘린다. 상담관리/상담 워크스페이스는 동결 정책에 따라 제외한다.

## 1차

### 작업내용

- `support.yeon.world/typing`에 연습 덱 변경 가이드와 타자방 접속 문제 해결 문서를 추가한다.
- `support.yeon.world/card`에 게스트 덱 계정 이관 가이드와 복습 모드 단축키 가이드를 추가한다.
- `support.yeon.world/community`에 게스트 닉네임/비밀번호 가이드와 글·댓글 등록 실패 문제 해결 문서를 추가한다.
- admin 공개 콘텐츠 통계 테스트의 article count를 registry와 맞춘다.
- 각 문서는 실제 구현 파일을 `sourcePaths`로 남긴다.

### 논의 필요

- admin에서 본문 수정/삭제 기능을 이번 차수에 넣을지 여부.
- 각 서비스별 FAQ를 더 많이 늘릴지, 우선 실제 구현 근거가 확실한 2개씩만 넣을지 여부.

### 선택지

- A. 타자, 카드, 커뮤니티에 검증 가능한 support 글을 2개씩 추가한다.
- B. 세 서비스의 FAQ까지 한 번에 대량 추가한다.
- C. admin CMS 편집 기능을 먼저 만든 뒤 콘텐츠를 나중에 넣는다.

### 추천

A를 추천한다. SEO에는 빈 메뉴와 얇은 글보다 실제 구현에 근거한 문제 해결형 글이 낫다. 현재 단계에서 admin 본문 수정/삭제는 사용자가 필요 없다고 판단했으므로 넣지 않는다.

### 사용자 방향

사용자는 상담관리를 제외한 서비스들을 전부 다루되, 가이드와 FAQ는 서비스별로 나누자고 했다. admin은 초기에는 직접 수정/삭제보다 공개 페이지와 `/admin/content`에서 상태를 확인하고 정적 registry 또는 이후 CMS 설계로 본문을 관리하는 방향을 선호했다.
