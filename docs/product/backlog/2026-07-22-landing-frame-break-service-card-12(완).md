# 랜딩 타자연습 카드 인물 중심 카메라 구도 조정

> 상태: 완료

## 1차 — 실제 전경 에셋의 가로 구도 재구성

- 작업내용: 사용자 제공 원본에서 누끼 전경을 다시 만들고 투명 캔버스를 넓혀 오른쪽 의자 영역을 보존한다.
- 논의 필요: 전등 전체를 유지하면서 왼쪽 모니터 비중을 줄이고 인물을 카드 중앙에 가깝게 배치해야 한다.
- 선택지:
  - A. 원본 픽셀은 그대로 두고 투명 여백과 표시 배율로 카메라 구도를 조정한다.
  - B. CSS 도형이나 생성형 보정으로 잘린 영역을 연장한다.
- 추천: A. 실제 이미지 자산만 사용하므로 전등과 의자의 형태를 왜곡하지 않는다.
- 사용자 방향: 의자를 더 보여주고 모니터를 덜 보여주며 인물을 중앙에 배치한다.

## 2차 — 기존 전경 배율 유지와 반응형 검증

- 작업내용: 공통 전경 배율은 유지하고 타자연습 에셋 내부의 투명 여백만 조정한 뒤 모바일·데스크톱에서 구도를 확인한다.
- 논의 필요: 백지학습·플래시카드 구도는 바꾸지 않고 제목·상태 배지와의 간격도 유지해야 한다.
- 선택지:
  - A. 기존 공통 너비를 유지하고 에셋 캔버스만 조정한다.
  - B. 서비스별 너비 설정을 새로 추가한다.
- 추천: A. 불필요한 UI 분기를 만들지 않고 변경 범위를 1번 이미지로 한정한다.
- 사용자 방향: 현재 제목과 전등 간격은 유지하고 가로 카메라 뷰만 오른쪽으로 조정한다.

## 완료 조건

- 전등이 잘리지 않고 실제 원본 픽셀로 표시된다.
- 이전보다 의자가 더 보이고 모니터가 덜 보인다.
- 인물이 카드 중앙에 더 가까워진다.
- 375px와 1040px에서 제목·상태·CTA가 가려지지 않고 가로 스크롤이 없다.

## 완료 결과

- 사용자 제공 1574×792 원본의 누끼를 다시 추출하고 2225×792 투명 캔버스의 x=230 지점에 배치했다.
- 오른쪽 의자 원본 픽셀이 잘리지 않도록 캔버스를 확장했고, 카드 중앙 기준으로 전경 내용이 왼쪽에 놓여 카메라가 오른쪽을 보는 구도가 됐다.
- 기존 공통 `w-[min(170%,35rem)]`와 타자연습 `-bottom-8`은 그대로 유지해 다른 카드와 제목 간격을 바꾸지 않았다.
- CSS 도형이나 생성형 보정 없이 실제 원본 픽셀과 투명 여백만 사용했다.
- 375px와 1040px에서 전등 전체, 인물 중심성, 의자 노출, 가로 스크롤 부재를 확인했다.
