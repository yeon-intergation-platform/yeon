# 타자방 상세 레이아웃 정리 (2026-05-12)

## 상태

- 완료

## 1차 — 헤더와 방 상단 정보 재배치

### 작업내용

- `TypingServiceHeader`에서 브랜드와 현재 화면 제목이 `YEON YEON 타자방`처럼 중복 노출되지 않도록 title 표시 구조를 정리한다.
- 타자 서비스 내비게이션은 헤더 중앙 정렬로 이동한다.
- 타자방 상세 화면의 `초대`와 `나가기` 버튼은 헤더에서 제거한다.
- `나가기`는 main 내부 왼쪽 상단의 화살표 CTA로 둔다.
- 방 상태 카드 오른쪽 하단에 방 설명과 시작/준비 버튼을 배치한다.

### 논의 필요

- 초대 버튼은 헤더에서 제거하되 방 상단 카드 안에 유지할지, 완전히 숨길지 선택이 필요하다.

### 선택지

1. 방 상단 카드 안에 작은 보조 버튼으로 유지한다.
2. 초대 기능을 화면에서 제거한다.

### 추천

- 초대 기능은 유용하므로 헤더에서는 빼고 방 상단 카드 액션 영역에 보조 버튼으로 유지한다.

## 2차 — 참여자 카드와 캐릭터 표시

### 작업내용

- 참가자 스냅샷에 선택 캐릭터 ID를 전달해 타자방에서도 선택 캐릭터가 보이도록 한다.
- 참여자 목록은 리스트가 아니라 최대 2x2 카드 그리드로 표시한다.
- 빈 슬롯도 카드 형태로 보여 현재 인원/최대 인원을 직관적으로 알게 한다.

### 논의 필요

- 참가자 캐릭터 정보가 race-server 프로토콜에 포함되어야 하므로 `packages/race-shared`, `apps/race-server`, `apps/web`을 함께 수정한다.

### 선택지

1. 프로토콜에 `characterId`를 추가한다.
2. 클라이언트에서 모든 참가자를 기본 캐릭터로만 표시한다.

### 추천

- 실제 선택 캐릭터를 보여야 하므로 `characterId`를 프로토콜에 추가한다.

## 3차 — 채팅 영역 확장

### 작업내용

- 기존 3컬럼에서 채팅이 320px에 갇히는 구조를 제거한다.
- 방 설정/참여자를 좌측 보조 패널로 묶고, 채팅은 우측에서 최대 960px까지 넓게 배치한다.

### 논의 필요

- 좁은 화면에서는 모바일 단일 컬럼으로 자연스럽게 접힌다.

### 선택지

1. 데스크톱에서 `minmax(0, 960px)` 우측 채팅 컬럼을 둔다.
2. 채팅을 전체 하단 섹션으로 둔다.

### 추천

- 사용자가 명시한 960px 기준을 반영해 우측 메인 컬럼으로 둔다.
