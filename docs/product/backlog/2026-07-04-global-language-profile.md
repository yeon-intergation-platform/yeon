# 글로벌 언어 설정과 프로필 화면 복구

## 1차

### 작업내용

- 웹 공통 언어 설정을 `ko`/`en`으로 저장하는 작은 클라이언트 SSOT를 만든다.
- 타자 서비스에만 있던 locale 개념을 게임 서비스에서도 재사용할 수 있게 분리한다.
- 기존 타자 설정의 locale은 가능한 범위에서 새 공통 설정과 동기화한다.

### 논의 필요

- 전체 플랫폼의 모든 문구를 한 번에 번역하면 변경 범위가 과해진다.
- 먼저 사용자가 지적한 게임 서비스와 프로필 화면을 실제 사용 가능 수준으로 맞춘다.

### 선택지

- A. 전역 i18n 프레임워크를 도입한다.
- B. 현재 저장소 패턴에 맞춰 localStorage 기반 공통 언어 설정과 서비스별 텍스트 테이블을 만든다.

### 추천

- B. 의존성을 늘리지 않고 기존 타자 서비스의 구조와 맞출 수 있어 작고 검증 가능하다.

### 사용자 방향

- 추천 기준으로 진행한다.

## 2차

### 작업내용

- 게임 서비스 홈/상세의 주요 UI 문구를 언어 설정에 맞춰 한국어/영어로 표시한다.
- 언어 설정이 영어면 기본 게임 지역도 미국/영어권으로 잡고, 한국어면 한국 추천을 우선한다.
- 게임 카드에는 게임 언어/지역이 어긋나지 않도록 사용자가 선택한 언어 기준의 진입 상태를 명확히 보여준다.

### 논의 필요

- 외부 iframe 게임 내부 UI는 제어할 수 없으므로, 사용자가 선택한 언어와 맞는 게임 묶음을 먼저 보여주는 방식으로 해결한다.

### 선택지

- A. 외부 게임 iframe 내부까지 번역하려고 시도한다.
- B. 플랫폼 UI와 게임 선택/지역 추천을 언어 설정에 맞추고, 게임 자체 언어를 명시한다.

### 추천

- B. iframe 내부는 제어 경계 밖이라, 선택과 안내를 정확히 하는 것이 안정적이다.

### 사용자 방향

- 추천 기준으로 진행한다.

## 3차

### 작업내용

- `/profile`이 인증/비인증/오류 상태에서 빈 화면이 되지 않게 프로필 보기 화면을 보강한다.
- 프로필 저장 UI의 색상은 기존 흰 배경 디자인 시스템 값으로 정리한다.
- 로컬 Playwright로 프로필 화면과 게임 언어 설정 흐름을 확인한다.

### 논의 필요

- 프로필 API 자체가 실패하는 경우에는 화면을 죽이지 않고 오류 카드와 재시도/로그인 경로를 보여준다.

### 선택지

- A. 서버 컴포넌트에서 모든 프로필 데이터를 직접 불러오고 실패하면 throw 한다.
- B. 서버 세션 정보로 기본 화면을 항상 렌더하고, 클라이언트에서 프로필/경험치 부가정보를 안전하게 불러온다.

### 추천

- B. 빈 화면 방지와 장애 격리에 유리하다.

### 사용자 방향

- 추천 기준으로 진행한다.
