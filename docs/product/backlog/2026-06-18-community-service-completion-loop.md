# 커뮤니티 서비스 완성 루프

## 목표

- `community` 서비스만 범위로 고정해 실제 사용자 흐름 기준의 품질 루프를 수행한다.
- Playwright로 첫 진입, 작성, 수정, 삭제, 새로고침, 모바일, 실패 입력, 느린 네트워크를 확인한다.
- 발견 이슈를 P0/P1/P2로 분류하고 P0/P1은 같은 사이클에서 해결한다.
- 수정마다 테스트, Playwright smoke, CI preflight, schema check, lint rule, 문서 규칙 중 하나 이상의 재발 방지 장치를 남긴다.

## 1차 - 상태 분리와 기준선 확인

### 작업내용

- `origin/main` 기준 새 브랜치에서 시작한다.
- 기존 dirty worktree와 이번 루프 작업을 분리한다.
- 이전 커뮤니티 QA 로그와 현재 코드의 source of truth를 다시 확인한다.

### 논의 필요

- 없음. 사용자가 루프 실행을 승인했다.

### 선택지

- A. `yeon-2` dirty worktree에서 계속한다.
- B. 깨끗한 기존 작업 워크트리에서 `origin/main` 기준 새 브랜치를 만든다.

### 추천

- B. 다른 에이전트의 타자/i18n 변경과 섞이지 않는다.

### 사용자 방향

- 추천 기준으로 진행한다.

## 2차 - Playwright 실제 사용 감사

### 작업내용

- 데스크톱과 모바일에서 커뮤니티 목록, 채팅, 게스트 인증, 글, 댓글 흐름을 직접 조작한다.
- 실패 입력과 새로고침, 저장 인증 hydrate, 느린 네트워크/오프라인에 가까운 조건을 확인한다.
- 발견한 이슈를 P0/P1/P2로 분류한다.

### 논의 필요

- 없음. P2는 사용성 개선 후보로 남기고 P0/P1 우선 수정한다.

### 선택지

- A. 기존 테스트만 실행한다.
- B. Playwright로 실제 사용자 시나리오를 먼저 돌리고 테스트 커버리지 구멍을 찾는다.

### 추천

- B. 사용자가 요구한 완성도 루프의 핵심은 실제 사용 시뮬레이션이다.

### 사용자 방향

- 추천 기준으로 진행한다.

## 3차 - 작은 묶음 수정과 재발 방지

### 작업내용

- 한 사이클에 하나의 작은 문제 묶음만 수정한다.
- 수정 범위에 맞는 단위 테스트, Playwright smoke, 백엔드 테스트, 타입체크 중 하나 이상을 추가하거나 보강한다.
- 같은 실패가 3번 반복되면 구현을 멈추고 원인과 필요한 결정을 보고한다.

### 논의 필요

- 외부 서비스 계정, 운영 데이터 삭제, 권한 정책 변경이 필요하면 중단 후 보고한다.

### 선택지

- A. 발견 즉시 큰 리팩터링으로 묶어서 수정한다.
- B. P0/P1 중심으로 작게 수정하고 재발 방지 게이트를 붙인다.

### 추천

- B. PR 위험을 낮추고 CI 실패 원인을 좁힐 수 있다.

### 사용자 방향

- 추천 기준으로 진행한다.

## 4차 - 검증, PR, main 반영

### 작업내용

- 관련 lint/typecheck/test/Playwright 검증을 실행한다.
- 변경이 있으면 한국어 커밋 메시지로 커밋하고 PR을 `main` 대상으로 만든다.
- PR checks가 green이면 merge하고, CI 실패가 있으면 로그를 근거로 다음 사이클을 시작한다.

### 논의 필요

- 없음. 프로젝트 정책상 변경 후 main 반영까지 진행한다.

### 선택지

- A. 로컬 검증만 하고 멈춘다.
- B. PR(main)까지 만들어 CI와 merge까지 본다.

### 추천

- B. 요청한 루프의 종료 조건에 CI/PR 경로가 포함되어 있다.

### 사용자 방향

- 추천 기준으로 진행한다.

## 1차 루프 완료 결과

### 작업내용

- 커뮤니티 목록/상세의 게스트 인증 실패 경계와 글 수정 흐름을 실제 Playwright 사용 기준으로 확인했다.
- 저장된 게스트 비밀번호가 틀린 경우 mutation 실패가 Next runtime error로 번지는 P1을 수정했다.
- 글 수정 시 raw 저장 포맷이 노출되어 제목/카테고리가 깨지는 P1을 구조화 수정 폼으로 수정했다.
- 재발 방지로 커뮤니티 단위 테스트 2개 파일과 opt-in 온라인 Playwright E2E를 보강했다.

### 논의 필요

- 없음. 운영 정책 변경, DB 스키마 변경, 외부 계정 권한 변경 없이 프론트 상태 경계와 테스트만 수정했다.

### 선택지

- A. 이번 루프를 커뮤니티 P0/P1 수정 완료로 닫고 PR(main)로 반영한다.
- B. 같은 PR에서 커뮤니티 P2 UI 개선까지 추가로 묶는다.

### 추천

- A. 발견된 P1 두 건과 재발 방지 게이트가 같은 작은 변경 묶음으로 닫혔다. P2 개선은 별도 루프가 안전하다.

### 사용자 방향

- 추천 기준으로 진행한다.
