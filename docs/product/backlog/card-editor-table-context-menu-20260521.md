# 카드 에디터 표 우클릭 삭제 UX 백로그 (2026-05-21)

## 배경

카드 직접 작성 에디터에서 표 삽입은 가능하지만 삽입된 표를 명확하게 삭제할 방법이 없다. 사용자는 웹사이트 전반에 Windows/macOS처럼 오른쪽 클릭 UX를 도입하려고 하며, 이번 표 삭제 기능도 공용 컨텍스트 메뉴 도구로 확장 가능한 형태여야 한다.

## 1차

### 작업내용

- 웹 공용 컨텍스트 메뉴 컴포넌트/타입을 만든다.
- 카드 rich editor 영역에서 표를 오른쪽 클릭하면 "표 삭제" 메뉴를 표시한다.
- 메뉴 실행 시 현재 표 전체를 삭제한다.
- 메뉴는 바깥 클릭, Escape, 스크롤/리사이즈 상황에서 닫히게 한다.
- 기존 표 편집/입력 동작을 깨지 않는다.

### 논의 필요

- 없음. 사용자가 오른쪽 클릭 삭제와 공용화 방향을 명시했다.

### 선택지

1. 카드 에디터 내부에 임시 메뉴를 직접 구현한다.
   - 빠르지만 다른 화면 재사용성이 낮다.
2. 웹 공용 컨텍스트 메뉴 컴포넌트를 만든 뒤 카드 표 삭제에 적용한다.
   - 이번 요구와 장기 방향에 맞다.
3. 브라우저 기본 contextmenu를 그대로 둔다.
   - 표 삭제 액션을 제공하지 못한다.

### 추천

2번. 기능 요구와 운영체제형 우클릭 UX 방향을 동시에 만족한다.

### 사용자 방향

추천 기준으로 진행한다.

## 완료 기준

- 표 위에서 오른쪽 클릭 시 "표 삭제" 메뉴가 나온다.
- "표 삭제"를 누르면 해당 표가 삭제된다.
- 공용 컨텍스트 메뉴 컴포넌트가 카드 서비스 전용이 아닌 위치에 존재한다.
- web lint/typecheck/build를 통과한다.
