# 카드 에디터 Enter 들여쓰기 유지 (2026-05-21)

## 1차 — 현재 Enter 처리 경계 확인

### 작업내용

- 카드 서비스 TipTap 에디터의 `handleKeyDown`에서 Enter 처리 흐름을 확인한다.
- 목록, 코드블록, 표 노드의 기본 Enter 동작은 유지하고 일반 문단에서만 직접 입력한 선행 공백/탭을 다음 줄로 복제한다.

### 논의 필요

- 들여쓰기를 일반 문단에만 적용할지, 제목/인용문에도 적용할지.

### 선택지

1. 일반 문단에만 적용한다.
2. 모든 텍스트 블록에 적용한다.

### 추천

- 1안. 목록/코드블록/표 같은 구조화 노드는 TipTap 기본 동작과 충돌하면 안 된다.

### 사용자 방향

- 직접 들여쓴 글자는 Enter 후 다음 줄도 같은 들여쓰기가 유지되어야 한다.

## 2차 — Enter 들여쓰기 복제 구현

### 작업내용

- 커서가 일반 문단 안에 있고 줄 앞 공백/탭이 존재할 때 Enter를 가로채 `splitBlock` 후 같은 들여쓰기 텍스트를 삽입한다.
- 커서가 줄 맨 앞이거나 들여쓰기 없는 줄이면 기본 Enter를 유지한다.

### 논의 필요

- 줄 중간에서 Enter를 누를 때도 들여쓰기를 복제할지.

### 선택지

1. 현재 줄의 선행 공백/탭은 줄 중간 Enter에서도 복제한다.
2. 줄 끝에서 Enter할 때만 복제한다.

### 추천

- 1안. 일반 텍스트 편집기 관성에 맞고 구현도 예측 가능하다.

### 사용자 방향

- 추천 기준으로 진행한다.

## 3차 — 검증과 main 반영

### 작업내용

- targeted lint/typecheck를 실행한다.
- 문서/로그를 남기고 PR(main)로 머지한다.

### 논의 필요

- 로컬 브라우저 확인까지 할지.

### 선택지

1. 키 처리 단위 변경이라 lint/typecheck와 코드 검토로 진행한다.
2. dev server + Playwright로 확인한다.

### 추천

- 1안. 작은 키 처리 변경이고 build trigger는 아니다.

### 사용자 방향

- 작업 후 main 반영한다.
