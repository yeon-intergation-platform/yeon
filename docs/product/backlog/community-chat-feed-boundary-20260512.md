# 커뮤니티 채팅/피드 경계 복구

- 작성일: 2026-05-12
- 상태: 진행
- 범위: `apps/web/src/features/community` 채팅 위젯과 커뮤니티 피드 UI

## 1차

### 작업내용
- 커뮤니티 상단 `실시간 채팅` 입력이 피드 글 작성 API를 호출하는 결합을 제거한다.
- 채팅 위젯은 채팅방/메시지 API만 사용하고, 피드 글 작성은 `글쓰기` 패널만 담당하게 한다.
- 커뮤니티 페이지 상단 채팅 영역 높이를 키우고 메시지 목록이 자체 스크롤되도록 레이아웃을 조정한다.

### 논의 필요
- 공개 전체 채팅방을 별도 Spring 도메인으로 만들지, 기존 1:1 채팅방 목록 중 첫 방을 커뮤니티 위젯에 노출할지 여부.

### 선택지
1. 즉시 복구: 채팅 입력에서 피드 작성 API 호출을 제거하고 기존 채팅방 API만 사용한다.
2. 새 기능: 공개 커뮤니티 채팅 도메인/API를 Spring에 추가한다.

### 추천
- 사용자 피해가 큰 “채팅 입력 → 글 생성” 오작동은 1번으로 즉시 차단한다.
- 공개 채팅 도메인이 필요하면 별도 백로그에서 Spring API로 설계한다.

### 사용자 방향
- 채팅은 채팅, 피드는 피드로 분리한다.
