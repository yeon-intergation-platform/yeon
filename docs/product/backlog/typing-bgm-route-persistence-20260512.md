# 타이핑 BGM 라우트 이동 지속 백로그

## 1차수

### 작업내용

- 타이핑 BGM 오디오 인스턴스를 버튼 컴포넌트 생명주기에서 분리한다.
- Next.js 클라이언트 라우트 이동이나 컴포넌트 remount가 발생해도 같은 브라우저 탭의 BGM 인스턴스를 유지한다.
- 기존 BGM 버튼의 ON/OFF 표시와 차단 안내는 유지한다.

### 논의 필요

- 브라우저 전체 새로고침까지 자동 재생을 복원할지는 별도 정책이 필요하다. 이번 범위는 앱 내 화면 이동 지속으로 제한한다.

### 선택지

- A. 루트 layout에 숨김 audio 엘리먼트를 상시 렌더링한다.
- B. `globalThis` 기반 BGM controller를 만들고 버튼은 controller를 구독/제어한다.

### 추천

- B. UI 렌더 위치와 오디오 source of truth를 분리해 라우트 이동과 버튼 remount에 강하게 만든다.

### 사용자 방향

- BGM을 켠 뒤 화면 이동해도 끊기지 않게 한다.
