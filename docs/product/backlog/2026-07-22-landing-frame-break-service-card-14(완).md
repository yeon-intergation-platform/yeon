# 랜딩 타자연습 전경 크기 정합성 보정

> 상태: 완료

## 1차 — 투명 캔버스 여백 축소

- 작업내용: 타자연습 전경의 보이는 원본 픽셀은 유지하고 불필요한 투명 여백만 제거한다.
- 논의 필요: 인물 크기를 2·3번과 맞추면서 이전에 확정한 오른쪽 카메라 구도, 전등, 의자 노출을 유지해야 한다.
- 선택지:
  - A. 2225×792 캔버스를 1830×792로 잘라 보이는 픽셀의 CSS 배율을 높인다.
  - B. 1번에만 별도 CSS 확대율과 최대 너비를 추가한다.
- 추천: A. 서비스별 CSS 예외 없이 자산의 불필요한 투명 여백만 바로잡는다.
- 사용자 방향: 작게 보이는 1번 인물을 2·3번과 비슷한 크기로 수정한다.

## 2차 — 시각 및 수치 검증

- 작업내용: 375px·1040px·1440px에서 1번 전경 높이, 전등·의자 노출, 배경 일관성, 가로 스크롤을 확인한다.
- 논의 필요: 캔버스 축소로 전등이나 의자가 다시 잘리지 않아야 한다.
- 선택지:
  - A. 전경 이미지의 렌더링 높이를 2·3번과 수치로 비교한다.
  - B. 육안 확인만 한다.
- 추천: A. Playwright DOM 수치와 스크린샷을 함께 남긴다.
- 사용자 방향: 화면에서 즉시 비교할 수 있게 로컬 서버를 유지한다.

## 완료 조건

- 1번 전경 높이가 2·3번과 유사해진다.
- 전등과 의자의 실제 픽셀이 잘리지 않는다.
- 오른쪽 카메라 구도와 공통 배경은 유지된다.
- 모바일·데스크톱에서 가로 스크롤과 콘텐츠 가림이 없다.

## 완료 결과

- v9 전경의 2225×792 캔버스에서 x=200 기준 1830×792 영역만 잘라 v10으로 교체했다.
- 알파 전경 1558×791 원본 픽셀은 그대로 유지되고 투명 시작 여백만 x=230에서 x=30으로 줄었다.
- 1440px에서 1번 전경 높이가 199.36px에서 242.66px로 증가해 2번과 정확히 같아졌다.
- 375px에서는 196.83px에서 239.13px로 증가했으며 가로 스크롤이 없다.
- 전등 갓과 오른쪽 의자, 오른쪽 카메라 구도, 8개 공통 배경은 유지됐다.
- CSS 확대 예외를 추가하지 않고 기존 공통 전경 스타일을 그대로 사용했다.
