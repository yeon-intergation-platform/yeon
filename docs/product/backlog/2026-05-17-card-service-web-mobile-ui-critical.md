# 카드 서비스 웹 모바일 UI critical 깨짐 수정 백로그

## 배경

- 운영 URL `https://yeon.world/card-service`를 iPhone 14 Pro 폭(393px)에서 확인했을 때 홈 카드 영역이 뷰포트 오른쪽을 넘어간다.
- 원인은 카드 서비스 홈의 모바일 패널 내부 여백과 `TypingProfileCard`의 고정 폭(`w-[380px]`)이 합쳐져 부모 폭을 초과하는 구조다.

## 차수 1

- 작업내용:
  - 운영 모바일 화면에서 critical UI 깨짐 원인을 재현한다.
  - 카드 서비스 홈과 프로필 카드의 고정 폭/패딩 구조를 모바일 우선 반응형 구조로 수정한다.
  - 같은 화면의 CTA 카드가 부모 폭을 넘지 않는지 정적/브라우저 검증한다.
- 논의 필요: 없음. 사용자가 먼저 critical UI 깨짐 탐지와 리팩토링을 요청했다.
- 선택지:
  1. 카드 서비스 홈만 국소 패치한다.
  2. `TypingProfileCard` 자체를 반응형으로 바꿔 같은 컴포넌트를 쓰는 서비스에도 안전하게 만든다.
- 추천: 2번. 원인 고정 폭은 공용 프로필 카드에 있으므로 컴포넌트 레벨에서 폭을 고치고, 홈 패널도 `min-w-0`/모바일 패딩을 보강한다.
- 사용자 방향: 추천 기준으로 진행한다.

## 완료 기준

- 393px 모바일 뷰포트에서 `document.body.scrollWidth <= window.innerWidth`.
- 카드 서비스 홈의 프로필 카드/오늘의 시작 CTA가 오른쪽으로 잘리지 않는다.
- `pnpm --filter @yeon/web lint`와 관련 타입 검증을 통과한다.
