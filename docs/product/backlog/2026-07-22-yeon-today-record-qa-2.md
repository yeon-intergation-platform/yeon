# YEON Today 하루 기록 기능·시각·서버 QA 2차 백로그

## 목표

`/today/record`의 사용자 기능과 데스크톱·모바일 화면을 실제 브라우저에서 검증하고, Spring 조회 경로의 N+1 가능성과 상태 변경 로직을 점검해 발견한 문제를 수정한다.

## 1차 — 재현과 기능 기준선

- 작업내용: 인증된 독립 QA 계정으로 활동 선택, 시간 블록 생성·교체·삭제, 새로고침 지속성, 활동 생성·비활성화, 날짜·상위 탭 공유를 검증한다.
- 논의 필요: 운영 데이터가 아닌 로컬 PostgreSQL과 개발 로그인을 사용한다.
- 선택지: 기존 운영 화면을 읽기 전용으로 확인 / 로컬 전체 흐름을 변경까지 확인.
- 추천: 로컬 전체 흐름을 검증하고 운영 화면은 배포 후 smoke test만 수행한다.
- 사용자 방향: 기능이 모두 문제없이 작동하는지 실제 화면에서 확인한다.

## 2차 — 프론트엔드 시각·접근성 수정

- 작업내용: 활동 선택 버튼의 선택·포커스 표시 잘림, 반응형 overflow, 탭·타임라인·달력의 정렬과 키보드 접근성을 검증하고 최소 수정한다.
- 논의 필요: 선택 표시를 버튼 바깥 ring으로 유지할지 내부 border로 바꿀지 결정한다.
- 선택지: 컨테이너 여백 확장 / 선택 표시를 inset ring으로 변경.
- 추천: 가로 스크롤과 키보드 포커스가 함께 안전하도록 컨테이너에 ring 여유 공간을 명시한다.
- 사용자 방향: 첨부 화면에서 잘려 보이는 부분을 포함해 프론트엔드 관점으로 점검한다.

## 3차 — Spring 조회·상태 로직 점검

- 작업내용: Controller→Service→Repository 호출 수를 endpoint별로 계수하고, 슬롯 조회 JOIN, 기본 활동 초기화, 소유권·시간 경계·비활성 활동·동시 수정 로직을 검사한다.
- 논의 필요: 고정 크기 초기화 쿼리와 데이터 크기에 비례하는 N+1을 구분한다.
- 선택지: 코드 정적 점검만 수행 / repository 호출 수 회귀 테스트도 추가.
- 추천: N+1 부재를 호출 수 테스트로 고정하고, 발견한 상태 정합성 문제만 수정한다.
- 사용자 방향: 백엔드의 N+1 문제와 로직을 함께 점검한다.

## 4차 — 회귀 검증과 배포

- 작업내용: Today 관련 Spring·Web 테스트, lint, typecheck, Playwright 데스크톱·모바일 QA, before/after 스크린샷을 실행한 뒤 main 배포와 운영 smoke test를 확인한다.
- 논의 필요: 배포 전 빌드는 CI 게이트를 기준으로 하되 라우팅·환경변수 변경이 생기면 로컬 빌드도 실행한다.
- 선택지: 로컬 검증까지만 수행 / PR merge와 운영 확인까지 수행.
- 추천: 저장소 정책대로 PR(main) merge 후 운영 화면과 API 상태를 확인한다.
- 사용자 방향: 수정 결과가 실제 서비스에서도 문제없이 동작하는 상태까지 확인한다.

## 완료 기준

- 활동 선택 표시가 어느 viewport에서도 잘리지 않는다.
- 핵심 기록·관리·날짜 공유 흐름이 새로고침 후에도 유지된다.
- 기록 조회의 DB 호출 수가 슬롯 수에 따라 증가하지 않음을 테스트로 증명한다.
- 관련 자동화 검사와 Playwright QA가 통과하고 시각 증거가 작업 로그에 연결된다.

## 실행 결과

- 활동 선택 행에 상하좌우 4px 여유를 확보해 선택·포커스 ring이 스크롤 경계에서 잘리지 않도록 수정했다.
- 기록 조회는 슬롯과 활동을 한 번의 JOIN으로 가져오며, 슬롯 24개에서도 repository 조회가 1회임을 테스트로 고정했다.
- 기록 조회·기록 대상 검증 전에 실행되던 불필요한 활동 목록 초기화 조회를 제거했다. 기본 활동 초기화 책임은 활동 목록·생성 경로에 유지한다.
- Playwright로 시간 블록 생성·교체·새로고침 지속·삭제, 활동 생성·숨김, 날짜 변경, 상위 탭 날짜 유지, 390/1024/1440px overflow와 버튼 접근성 이름을 검증했다.
- Web lint·typecheck·전체 1,141개 테스트와 production build, Today Spring 13개 테스트, SSOT 검증이 통과했다.
- Spring 전체 633개 테스트 중 기존 `LayeredArchitectureTest` 2건만 실패했다. 실패 대상은 테스트 내부 컨트롤러 패키지와 게임 서비스 테스트 순환참조이며 Today 변경과 무관하다.
