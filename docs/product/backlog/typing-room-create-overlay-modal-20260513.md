# 타자방 만들기 overlay 모달 전환 (2026-05-13)

## 배경

타자방 로비에서 방 만들기 버튼을 눌렀을 때 생성 폼이 페이지 흐름 안에 들어간 카드처럼 보이고, 빈 상태 영역이 아래로 밀려 모달처럼 인지되지 않는다. 방 만들기 UI는 기존 레이아웃을 변경하지 않는 viewport 기준 overlay modal이어야 한다.

## 1차

### 작업내용

- 타자방 로비 방 만들기 폼을 fixed overlay 모달로 표시한다.
- 모달 open 상태에 따라 빈 상태 영역의 정렬/여백이 바뀌는 분기를 제거한다.
- dim 배경을 fixed inset 전체 화면으로 깔고 뒤쪽 요소 클릭을 차단한다.
- 모달 카드는 overlay 내부에서 viewport 중앙에 정렬한다.
- X 버튼, dim 배경 클릭, ESC 키로 닫히게 한다.

### 논의 필요

- dim 강도는 현재 흰 배경 미니멀 UI를 유지하면서 모달임을 인지할 수준이어야 한다.
- 모달이 열릴 때 body 스크롤 잠금까지 적용할지 확인이 필요하다.

### 선택지

1. 현재 `TypingRoomLobbyScreen` 내부 모달 마크업을 유지하고 fixed overlay/닫기 동작만 보강한다.
2. 공용 모달 컴포넌트로 분리한다.
3. `/typing-service/rooms/new` 페이지로만 이동하게 하고 로비 모달을 제거한다.

### 추천

1번. 현재 요구는 레이아웃 밀림 제거와 overlay 동작이므로 작은 수정으로 해결하고, 방 생성 흐름과 라우팅은 유지한다.

### 사용자 방향

추천 기준으로 진행한다.
