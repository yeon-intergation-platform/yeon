# 카드 추가 편집/미리보기 4칸 동일 높이 레이아웃 백로그 (2026-05-20)

## 배경

카드 추가 모달에서 카드 질문/답변 편집기와 오른쪽 미리보기 카드가 모달의 남은 세로 공간을 충분히 쓰지 못하고, 질문 편집기와 질문 미리보기 안쪽에 각각 스크롤이 생긴다. 사용자가 기대하는 구조는 질문/답변 2행이 남은 공간을 반반 나눠 쓰고, 왼쪽 편집기와 오른쪽 미리보기까지 총 4칸이 같은 높이로 보이는 것이다.

## 1차: 직접 작성 모드 4칸 equal grid fill

### 작업내용

- 카드 추가 직접 작성 레이아웃의 본문 영역을 남은 모달 높이를 채우는 grid/flex 구조로 바꾼다.
- 데스크톱에서는 2열 x 2행 구조에서 질문 편집기/질문 미리보기/답변 편집기/답변 미리보기가 같은 행 높이를 공유하게 한다.
- 질문 row와 답변 row가 남은 세로 공간을 `1fr 1fr`로 반반 나눠 먹게 한다.
- 질문 편집기와 질문 미리보기 내부 자체 스크롤을 줄이고, 카드 박스 높이가 커지게 한다.
- 기존 박스형 툴바, 라벨/뱃지, 저장/탭 동작은 유지한다.

### 논의 필요

- 내용이 1행 높이를 넘어서는 아주 긴 입력까지 내부 스크롤 없이 무한 확장하면 모달 footer가 사라질 수 있다. 이번 요구는 모달 안 남은 공간을 최대 활용하는 것이므로, 기본 상태에서는 내부 스크롤을 없애고 모달/outer 영역이 최종 스크롤을 담당하게 한다.

### 선택지

1. 각 editor에 고정 `h-[...]`를 더 키운다.
   - 화면 크기별로 또 깨질 가능성이 높고 남은 공간 반반 요구를 만족하지 못한다.
2. 직접 작성 content 영역을 `grid-rows-[minmax(0,1fr)_minmax(0,1fr)]`로 바꾸고 각 칸을 `min-h-0 h-full`로 채운다.
   - 요구에 가장 직접적으로 맞고 4칸 동등 구조를 만들 수 있다.
3. 모달 자체 높이를 더 크게 한다.
   - 원인인 내부 grid 분배를 해결하지 못한다.

### 추천

2번으로 진행한다. 행 단위 source of truth를 직접 작성 grid에 두고, 각 editor/preview는 부모 칸을 채우게 만든다.

### 사용자 방향

추천 기준으로 진행한다.

## 완료 기준

- 데스크톱 카드 추가 모달에서 질문/답변 행이 남은 공간을 반반 나눠 쓴다.
- 질문 편집기와 질문 미리보기의 높이가 같은 행 안에서 동일하다.
- 답변 편집기와 답변 미리보기의 높이가 같은 행 안에서 동일하다.
- 질문 편집기/질문 미리보기 내부에 불필요한 작은 스크롤이 생기지 않는다.
- `pnpm --filter @yeon/web lint` 통과.
- `pnpm --filter @yeon/web typecheck` 통과.
- `localhost:3001` Playwright로 카드 추가 모달 레이아웃을 확인한다.
