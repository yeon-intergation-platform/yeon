# 차수39 동시성 Write API 경합 검증
- 생성일시: 2026-05-09T21:40:00+09:00
- 작업내용: 채팅/카드/상담기록/공간 계열의 동시 쓰기 API에서 중복 요청, 순서 역전, 중복 처리 경합을 재현해 Spring 전환 전후 동작 차이를 정량화한다.
- 논의 필요: 경합 충돌 시 409/400/멱등성 키 정책 중 어떤 실패 응답을 1차 기준으로 고정할지.
- 선택지:
  1. 기존 Next 동작(중복 요청 허용/최신값 우선) 유지
  2. 멱등성 키 기반 명시적 재시도 정책 도입
- 추천: 선택지 1 기반으로 시작하고, 경합이 증폭되는 라우트만 단계적으로 idempotency 키 적용
- 사용자 방향: 추천 기준 진행
- 확장 산출:
  - 대상 라우트 후보: `step35`의 43개 mutation 경로 우선검증
  - 최소 동시성 테스트: `artillery` 또는 `k6`로 p95 50회/초, 60초 스트레스
  - 체크포인트: 중복 요청 시 상태 코드, 응답 본문 `errorCode`, DB 최종 정합성
