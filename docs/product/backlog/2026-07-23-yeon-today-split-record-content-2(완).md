# YEON Today 이중 기록 카드 정보 노출 2차

## 목표

한 시간에 기록을 두 번 추가했을 때 대각선 분할 형태는 유지하면서 두 기록 모두 아이콘·활동명·설명을 읽을 수 있게 한다.

## 1차 — 클리핑 구조 교정

- 작업내용
  - 첫 기록은 왼쪽 위 삼각형의 넓은 영역, 두 번째 기록은 오른쪽 아래 삼각형의 넓은 영역에 정보를 배치한다.
  - 메뉴 버튼을 `clip-path`가 적용된 영역 밖으로 분리해 버튼 전체가 보이게 한다.
- 논의 필요
  - 대각선 형태를 유지하면서 작은 모바일 카드에서도 읽을 수 있는 정보량을 정한다.
- 선택지
  - 대각선 분할을 없애고 상하 두 줄로 나눈다.
  - 대각선 분할을 유지하고 정보 배치와 크기를 교정한다.
- 추천
  - 기존 시각 언어를 유지하는 두 번째 선택지를 적용한다.
- 사용자 방향
  - 두 기록 상태에서도 아이콘과 설명이 모두 보여야 한다.

## 2차 — 실제 화면 검증

- 작업내용
  - 데스크톱과 모바일에서 아이콘, 활동명, 설명, 메뉴 버튼 노출을 확인한다.
  - 우클릭 메뉴와 각 기록의 더보기 버튼이 기존처럼 독립 동작하는지 확인한다.
- 논의 필요
  - 없음.
- 선택지
  - 정적 검증만 수행한다.
  - Playwright 동작 검증과 전후 스크린샷을 남긴다.
- 추천
  - 실제 브라우저 검증과 전후 스크린샷을 남긴다.
- 사용자 방향
  - 실제 카드에서 모든 정보가 보이는 상태로 반영한다.

## 3차 — 검증과 배포

- 작업내용
  - lint, typecheck, 관련 테스트, 패리티 검증을 실행한다.
  - 커밋·push·PR(main)·병합을 완료한다.
- 논의 필요
  - 없음.
- 선택지
  - 로컬 수정만 남긴다.
  - 검증 후 main에 병합한다.
- 추천
  - 프로젝트 정책에 따라 main에 병합한다.
- 사용자 방향
  - 즉시 사용할 수 있는 상태로 반영한다.

## 완료 결과

- 대각선 경계를 완만한 상하 분할로 바꿔 각 기록의 정보 영역을 확보했다.
- 두 기록 모두 아이콘·활동명·설명을 표시한다.
- 더보기 버튼을 클리핑 영역 밖으로 분리해 28×28 전체가 보이게 했다.
- 데스크톱·모바일 Playwright와 자동 검증을 통과했다.
