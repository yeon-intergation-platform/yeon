# 카드 일괄 추가 덮어쓰기 버튼 웹 적용 백로그

## 1차

### 작업내용

- 웹 카드 덱 상세의 일괄 추가 폼에서 기존 카드 전체 삭제 후 붙여넣은 카드 목록으로 교체하는 덮어쓰기 액션을 사용자에게 명확히 노출한다.
- 이미 존재하는 게스트/서버 `replaceCards` 흐름을 확인하고, 버튼 배치·문구·확인 메시지를 실제 사용 흐름에서 놓치지 않게 다듬는다.
- 카드 서비스 web 변경이므로 모바일의 동일 기능 존재 여부와 Universal UI parity 영향을 확인한다.

### 논의 필요

- 서버 인증 사용자 덮어쓰기는 현재 개별 삭제 후 bulk 생성 흐름이다. 백엔드 전용 atomic replace API가 없으므로 이번 작업은 프론트 표면과 기존 repository 포트를 활용한다.

### 선택지

1. 현재 일괄 추가 폼 안에 덮어쓰기 버튼을 추가/강조한다.
2. 별도 덮어쓰기 모달을 새로 만든다.
3. Spring에 atomic replace API를 새로 만든다.

### 추천

- 선택지 1. 사용자가 요청한 앞단 버튼 노출을 가장 작고 안전하게 해결한다. 삭제가 번거로운 문제는 기존 `replaceCards` 포트로 즉시 해소할 수 있다.

### 사용자 방향

- 일괄 추가 버튼 옆에 덮어쓰기 버튼을 추가한다. 덮어쓰기는 기존 카드를 모두 없애고 일괄 추가 내용으로 완전히 교체한다.
