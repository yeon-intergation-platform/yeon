# news editorial 품질 게이트

상위 계획: `docs/product/backlog/2026-06-17-public-content-network-500-step-plan.md` 9차 220~225
범위: 업계 뉴스 출처 링크, 원문 복사 방지, 과장/보도자료 톤 방지, 긴 공지 목차 기준

## 1차

### 작업내용

1. 외부 업계 뉴스 해설이 공개 본문에 출처 링크를 표시할 수 있게 본문 `links` block을 추가한다.
2. 업계 뉴스 해설은 짧은 요약과 출처 링크를 audit로 강제한다.
3. news 글의 과장된 마케팅 표현을 audit로 차단한다.
4. 업계 뉴스 해설에서 보도자료처럼 보이는 표현을 audit로 차단한다.
5. 긴 공지는 기존 heading 기반 본문 목차가 제공된다는 점을 원장에 완료 표시한다.

### 논의 필요

- 외부 출처를 article metadata에 둘지, 본문 block으로 공개 노출할지.
- 원문 복사 방지를 완전히 자동 판정할지, 길이/표현 기반 gate로 최소 방어할지.

### 선택지

- A. 공개 본문 `links` block을 추가하고 audit가 업계 뉴스에 외부 출처 링크와 짧은 요약을 요구한다.
- B. `sourcePaths`를 그대로 공개 출처로 재사용한다.
- C. 문서 정책만 추가하고 코드 audit는 만들지 않는다.

### 추천

A를 추천한다. `sourcePaths`는 내부 traceability이고 공개 출처 링크와 역할이 다르다. 본문 block으로 공개 링크를 표시하면 사용자와 검색엔진 모두 출처를 확인할 수 있고, audit도 같은 데이터를 검사할 수 있다.

### 사용자 방향

사용자 방향이 비어 있으면 추천 기준으로 진행한다.

### 결과

완료. 공개 본문 `links` block을 추가해 업계 뉴스 해설에 외부 출처 링크를 표시하고, editorial audit가 짧은 요약, 공개 출처 링크, 긴 원문 복사 의심 문단, 과장 표현, 보도자료식 표현을 검사한다. 9차 220~225도 완료 표시했다.
