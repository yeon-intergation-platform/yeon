# 랜딩 커뮤니티 대화 중심 프레임브레이크 전경 교체

> 상태: 완료

## 1차 — 대화 중심 투명 전경 제작

- 작업내용: 제공된 커뮤니티 이미지를 바탕으로 인물·전등·모니터·대화 UI를 하나의 투명 전경 에셋으로 제작한다.
- 논의 필요: 기존 전경은 대화 말풍선이 좌우 가장자리로 밀려 카드에서 대부분 잘린다.
- 선택지:
  - A. 기존 전경의 CSS 위치만 이동한다.
  - B. 대화 UI를 중앙 안전영역에 모은 새 전경으로 교체한다.
- 추천: B. 카드 비율에서도 대화 내용이 보이고 커뮤니티 서비스 정체성이 유지된다.
- 사용자 방향: 새 제공 이미지를 사용해 기존 커뮤니티 전경을 갈아끼운다.

## 2차 — 카드 연결과 구도 보정

- 작업내용: 4번 커뮤니티 카드에 새 투명 전경을 연결하고, 제목과 충돌하지 않으면서 인물·전등이 이미지 영역 위로 돌출되도록 배치한다.
- 논의 필요: 대화 UI를 살리기 위해 전체 장면을 과도하게 축소하면 프레임브레이크 강도가 약해질 수 있다.
- 선택지:
  - A. 전경 전체를 축소해 모든 요소를 노출한다.
  - B. 인물과 전등의 돌출 크기를 유지하면서 대화 묶음을 중앙에 배치한다.
- 추천: B.
- 사용자 방향: 공용 배경은 변경하지 않는다.

## 3차 — 반응형 검증

- 작업내용: 375px·1040px·1440px에서 대화 가독성, 제목 충돌, 전경 잘림과 가로 스크롤을 확인한다.
- 논의 필요: 없음.
- 선택지:
  - A. 데스크톱만 확인한다.
  - B. 모바일·태블릿·데스크톱을 함께 확인한다.
- 추천: B.
- 사용자 방향: localhost에서 바로 확인한다.

## 완료 조건

- 4번 카드에서 여러 대화 말풍선과 공지 패널이 카드 안쪽에 보인다.
- 인물과 전등이 제목을 가리지 않고 자연스럽게 프레임을 넘는다.
- 공용 배경 파일은 변경되지 않는다.
- 세 화면폭에서 가로 스크롤이 없다.

## 완료 결과

- 제공 이미지의 인물·전등·모니터·키보드와 대화 말풍선 5개, 공지 패널, 접속자 패널을 한 전경으로 재구성했다.
- `community-frame-break-foreground-v4.webp` 1560×1018 투명 WebP를 적용하고, 카드 전용 폭을 `w-[min(150%,31rem)]`로 맞췄다.
- 기존 `community-frame-break-foreground-v3.webp`는 참조를 교체한 뒤 제거했다.
- 공용 배경의 git blob 해시가 `origin/main`과 동일함을 확인했다.
- 375px·1040px·1440px에서 공용 배경 9개, 전경 8개, 가로 overflow 0을 확인했다.
