# AI 스프라이트 프레임 QA 에디터 전환

## 1차

### 작업내용

- 기존 `/sprite-editor`의 픽셀 드로잉 중심 기능을 제거하고, AI 생성 프레임 검수/수정 요청 워크플로우 도구로 전환한다.
- 다중 프레임 업로드, 순서 변경, 재생 검수, 기준선/중심선/바운딩 박스/onion skin, 프레임 상태/메모, AI 수정 프롬프트 생성, 선택 프레임 교체, 최종 export를 제공한다.

### 논의 필요

- 향후 Codex/image-gen과 직접 연결해 앱 내부에서 재생성까지 자동화할지, 당분간 프롬프트 복사 기반으로 둘지 결정 필요.

### 선택지

1. 브라우저 내부 QA 도구 + 프롬프트 복사 방식으로 시작한다.
2. 서버 API를 추가해 이미지 생성 요청까지 연결한다.
3. 기존 픽셀 편집 기능을 유지하며 QA 기능을 덧붙인다.

### 추천

- 1번. Next.js가 신규 백엔드 역할을 맡지 않도록 하고, 우선 사람의 검수·지시문 생성 속도를 높이는 내부 도구로 만든다.

### 사용자 방향

- 추천 기준으로 진행한다.
