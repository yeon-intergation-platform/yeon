# 카드 상세 인라인 TipTap 편집 복구 (2026-05-12)

## 배경

`383ce33 feat: 카드 서비스 추가·수정 흐름을 중앙 모달 편집기로 통합` 이후 카드 행 클릭이 모달 편집기로 연결되어, 카드 목록 안에서 바로 질문/답변을 고치던 흐름이 사라졌다. 카드 상세 목록의 기본 조작은 모달이 아니라 해당 카드 row 안에서 바로 수정하는 흐름이어야 한다.

## 1차 — 인라인 편집 상태 복구

- 작업내용
  - `CardRow`를 보기/편집/삭제 상태로 분리한다.
  - 카드 본문 클릭 시 모달 없이 해당 row 안에서 질문/답변 편집 UI를 연다.
  - 삭제 버튼/스와이프 삭제는 유지하되 편집 중에는 저장/취소와 충돌하지 않도록 숨기거나 비활성화한다.
- 논의 필요
  - 편집 중 다른 카드를 클릭할 때 기존 편집 내용을 자동 취소할지 차단할지.
- 선택지
  - A. row별 독립 편집을 허용한다.
  - B. 한 번에 한 카드만 편집하고 dirty 상태는 확인창으로 보호한다.
- 추천
  - B. 한 번에 한 카드만 편집하고 dirty 상태는 `window.confirm`으로 보호한다.
- 사용자 방향
  - 추천 기준으로 진행한다.

## 2차 — 카드 전용 TipTap 리치 마크다운 에디터

- 작업내용
  - 카드 수정에는 `@uiw/react-md-editor` 대신 카드 전용 TipTap 기반 `CardRichMarkdownEditor`를 사용한다.
  - 질문/답변 각각 드래그앤드롭, 붙여넣기, 파일 선택으로 이미지를 현재 커서 위치에 삽입한다.
  - 이미지 리사이즈 width 속성을 HTML에 보존한다.
- 논의 필요
  - 카드 추가 폼까지 즉시 TipTap으로 바꿀지, 수정 흐름만 먼저 복구할지.
- 선택지
  - A. 수정 흐름만 TipTap으로 바꾸고 추가 폼은 기존 MarkdownEditor를 유지한다.
  - B. 추가/수정 모두 TipTap으로 통일하고 기존 단일 이미지 aside를 제거한다.
- 추천
  - B. 사용자가 이미지 삽입을 에디터 안으로 통일하라고 했으므로 추가/수정 모두 같은 에디터를 쓴다.
- 사용자 방향
  - 추천 기준으로 진행한다.

## 3차 — HTML/Markdown 호환 렌더링과 저장 호환성

- 작업내용
  - `MarkdownContent`가 HTML이면 DOMPurify로 sanitize 후 렌더링하고, 기존 Markdown이면 기존 렌더링 경로를 유지한다.
  - TipTap HTML은 `frontText`/`backText`에 저장한다.
  - 기존 `imageStorageKey/imageUrl` 단일 첨부 카드는 보기 화면에서 유지하고, 새 편집 화면에서는 별도 첨부 UI를 제거한다.
- 논의 필요
  - 본문 HTML 이미지 URL의 미사용 이미지 GC는 이번 범위에 포함할지.
- 선택지
  - A. 이번에 GC까지 포함한다.
  - B. 새 DB 스키마 없이 본문 HTML 저장만 처리하고 GC는 후속으로 둔다.
- 추천
  - B. 이번 작업은 인라인 편집 복구와 안전한 렌더링에 집중한다.
- 사용자 방향
  - 추천 기준으로 진행한다.
