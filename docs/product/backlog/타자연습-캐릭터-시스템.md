## 1차수: registry 기반 시스템 + 캐릭터 2종 추가

### 작업 내용

**A. SoT 정의**

- `apps/web/src/features/typing-service/characters/types.ts`: `CharacterDef` 타입.
- `apps/web/src/features/typing-service/characters/index.ts`: `TYPING_CHARACTERS` 배열, `findCharacter`, `toEnginePlayerCharacter`.
- 라벨 i18n은 registry의 `label: { ko, en }`로 직접 보유 (TRANSLATIONS 키 추가 없음).

**B. 자산 컨벤션 + 정규화 도구**

- 표준 위치: `apps/web/public/sprites/characters/<id>/run.png` (8프레임 가로 시트).
- 정규화 스크립트: `scripts/sprites/extract-character-run.py`.
  - 입력 시트가 9행×8열 layout일 때 2번째 줄(오른쪽 달리기 8프레임)만 추출.
  - `flip_horizontal` 옵션으로 좌측 시트도 우측으로 변환 가능.
- 카멜은 기존 `/sprites/camel-run.png` 유지 (96×96 6프레임).

**C. 카드 메타 기반 재배선**

- `typing-profile-card.tsx`의 `FRAME_SIZE/FRAME_COUNT/FRAME_COLS` 하드코딩 제거.
- `CharacterSprite` 컴포넌트가 `CharacterDef` 메타에서 frameWidth/Height/Count/Cols/fps 읽어 동적 계산.
- 캐릭터 height가 다르더라도 표시 height(`CARD_DISPLAY_HEIGHT=160`) 기준으로 정규화, width는 비율 유지.

**D. 프로필 훅 stale 방어**

- `use-typing-profile.ts`에서 localStorage 값을 `findCharacter` 통해 검증. registry에서 빠진 ID는 기본값으로 폴백.

**E. 엔진 옵션화**

- `mountTypingRaceEngine`에 `playerCharacter?: TypingRacePlayerCharacter` 추가.
- preload에서 NPC용 카멜 + 플레이어 캐릭터 두 시트 로드.
- `LANE_DISPLAY_HEIGHT=46` 기준 자동 scale → 캐릭터 height가 96이든 208이든 동일 시각 크기.
- 화면 진입 시 프로필 hydrate 대기 후 마운트 (default 캐릭터로 잘못 시작하는 깜빡임 방지).

**F. 추가 캐릭터**

- `asuna` (Yuuki Asuna 모티브 chibi, 192×208 8프레임).
- `linnea` (Augury Bird fae chibi, 192×208 8프레임).
- 2026-05-03 사용자가 두 자산 모두 공개 배포 허가 부여. `credit.license = "approved"`로 갱신, 자산 그대로 main 배포에 포함.
