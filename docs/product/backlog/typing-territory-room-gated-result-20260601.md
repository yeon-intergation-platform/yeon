# 타자 점령전 타자방 선입장 및 결과 UI 백로그

## 1차: 타자방 참가 후 점령전 진입으로 고정

- 작업내용
  - 홈의 점령전 CTA가 점령전 화면으로 직접 이동하지 않고 타자방 로비로 안내되게 변경한다.
  - 타자방 대기 화면에서 현재 roomId를 포함한 점령전 입장 CTA를 제공한다.
  - `/typing-service/territory` 직접 접근은 타자방 선입장 안내 화면으로 막고, roomId 쿼리가 있을 때만 점령전 화면을 보여준다.
- 논의 필요
  - 기존 타자방 Colyseus 방과 점령전 Colyseus 방의 서버 세션/참가자 완전 승계를 별도 프로토콜로 통합할지 여부.
- 선택지
  - A. 이번 차수는 URL/UX 게이트로 사용자 플로우를 고정하고, 점령전 서버는 기존 방을 유지한다.
  - B. race-server 프로토콜까지 확장해 타자방 참가자를 점령전 방에 완전 승계한다.
- 추천
  - A. 사용자가 지적한 즉시 오류는 직접 진입 플로우이므로 작은 변경으로 선입장 규칙을 먼저 고정한다.

## 2차: 레퍼런스형 점령전 종료 결과판 적용

- 작업내용
  - 프로젝트 내부 레퍼런스 이미지(`screenshot/typing-territory/result-reference-2026-06-01-195518.png`)를 기준으로 중앙 승리 배지, 좌우 팀 순위, 주황팀/보라팀 점수 카드, 하단 방 나가기/대기방 액션을 구성한다.
  - 프로토콜 팀 값은 유지하되 UI 라벨은 red=주황팀, blue=보라팀으로 매핑한다.
  - 종료 후 일정 시간 뒤 원래 타자방 대기방으로 돌아가도록 안내/자동 이동한다.
- 논의 필요
  - 보너스 게임 점수와 획득 포인트의 실제 서버 원천을 향후 별도 스코어 필드로 확장할지 여부.
- 선택지
  - A. 현재 snapshot의 team score/player score/captured count만으로 결과판을 표시한다.
  - B. shared protocol에 result breakdown 필드를 추가한다.
- 추천
  - A. 현재 서버 원천으로 표현 가능한 결과판을 먼저 구현하고, breakdown 확장은 별도 후속으로 분리한다.
