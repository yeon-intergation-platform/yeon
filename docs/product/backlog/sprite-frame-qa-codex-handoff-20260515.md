# AI 스프라이트 QA 에디터 Codex handoff 전환

## 1차

### 작업내용

- `/sprite-editor`에서 외부 이미지 API를 전제로 보이는 AI 수정 프롬프트 UI를 제거한다.
- 사람이 사이트에서 할 일은 재생 검수, 문제 프레임 표시, 메모, 수정 큐 정리, 프레임 교체, export로 제한한다.
- Codex에게 별도 요청할 때 사용할 수 있는 검수 리포트/수정 큐 JSON만 제공하고, 사이트가 직접 AI 생성 요청을 수행하는 흐름으로 보이지 않게 정리한다.

### 논의 필요

- 이후 Codex가 실제 프레임 이미지를 다시 생성한 뒤 어떤 디렉터리에 결과물을 저장하고, 사이트가 그 결과를 자동 재로드할지 결정이 필요하다.

### 선택지

1. MVP에서는 브라우저가 QA 리포트/수정 큐만 export하고, Codex 수정은 채팅에서 별도 요청한다.
2. 사이트에서 외부 이미지 API 호출 버튼까지 제공한다.
3. 기존 프롬프트 텍스트 영역을 유지하되 명칭만 바꾼다.

### 추천

- 1번. 비용이 드는 외부 이미지 API 호출 흐름을 사이트에 넣지 않고, Codex 작업을 위한 검수 데이터만 만든다.

### 사용자 방향

- 추천 기준으로 진행한다.
