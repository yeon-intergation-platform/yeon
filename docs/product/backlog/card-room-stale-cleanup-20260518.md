# 카드방 잔존 데이터 정리 백로그

## 1차

### 작업내용
- 과거 race-server 퇴장 누락으로 남은 빈 카드방이 로비에 계속 노출되는 경로를 Spring에서 차단한다.
- 활성 참가자가 0명인 공개 카드방을 `finished`로 전환하는 repository/service 정리 로직을 추가한다.
- 애플리케이션 기동 후 주기적으로 오래된 빈 카드방을 닫는 cleanup 작업을 추가해 네트워크/프로세스 누락에도 방이 쌓이지 않게 한다.
- 신규 정리 로직의 단위 테스트와 backend 검증을 수행한다.

### 논의 필요
- 없음. 사용자가 후속 정리를 요청했고, 과거 데이터와 누락 방어는 Spring이 소유하는 상태 정합성 문제다.

### 선택지
- A. Spring 서비스에 빈 방 TTL cleanup을 추가하고 기동/주기 실행한다.
- B. 운영 DB에 일회성 수동 쿼리만 실행한다.
- C. 클라이언트 로비에서만 0명 방을 숨긴다.

### 추천
- A를 적용한다. 수동 쿼리는 과거 데이터만 해결하고, 클라이언트 숨김은 상태 원천을 고치지 못한다.

### 사용자 방향
- 추천 기준으로 진행한다.

## 완료 기록
- Spring 카드방 정리 스케줄러를 추가해 기동 시점과 주기 실행에서 빈 방/오래된 방을 `finished`로 닫도록 했다.
- 과거에 참가자 `left_at`이 누락되어 남은 방도 TTL 기준으로 로비에서 사라지게 했다.
