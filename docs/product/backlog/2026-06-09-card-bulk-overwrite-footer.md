# 카드 일괄 추가 덮어쓰기 버튼 footer 고정

## 1차

### 작업내용

- 웹 카드 덱의 `카드 추가 > 일괄 추가` 모달에서 기존 `추가` 버튼 옆에 `덮어쓰기` 버튼을 항상 같은 footer 액션 영역에 노출한다.
- 덮어쓰기는 입력으로 인식된 카드 목록을 기준으로 기존 카드를 모두 삭제하고 새 목록으로 교체하는 기존 `replaceCards` 흐름을 사용한다.
- 사용자가 실수로 전체 카드를 삭제하지 않도록 확인 다이얼로그와 비활성 조건을 유지한다.

### 논의 필요

- 서버에 원자적 replace endpoint를 추가할지 여부. 이번 요청은 “앞단” 범위이므로 기존 프론트 repository 포트의 replace 흐름을 유지한다.

### 선택지

1. 기존 form 내부 하단에 덮어쓰기 버튼을 둔다.
2. 모달 footer에 추가/덮어쓰기 액션을 함께 고정한다.

### 추천

- 2번. 사용자가 본 화면처럼 footer에 추가 버튼만 보이는 혼란을 없애고, 스크롤 위치와 무관하게 두 액션을 나란히 인지할 수 있다.

### 사용자 방향

- 추천 기준으로 진행한다.
