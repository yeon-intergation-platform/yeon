# 게임 catalog 정합성 커버리지 보강

## 1차수

### 작업내용

- 정적 게임 카탈로그가 29개 노출 대상을 유지하는지 테스트로 고정한다.
- slug, 제목, 요약, 상세 설명, 제공자, 조작법이 비어 있거나 URL 비호환 형식으로 깨지는 회귀를 막는다.
- embed 식별자 중복을 테스트로 고정해 같은 외부 게임이 중복 노출되는 운영 누락을 줄인다.

### 논의 필요

- 없음. 기능 변경 없이 기존 catalog 값의 무결성 테스트만 보강한다.

### 선택지

- A. 기존 `game-catalog.test.ts`에 값 정합성 케이스를 추가한다.
- B. 별도 catalog validator 스크립트를 만든다.

### 추천

- A. 현재 카탈로그는 TypeScript 정적 소스라 단위 테스트가 가장 작고 빠르게 회귀를 잡는다.

### 사용자 방향

- 추천 기준으로 진행한다.
