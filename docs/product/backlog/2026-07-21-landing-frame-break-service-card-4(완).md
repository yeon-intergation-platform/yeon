# 랜딩 프레임 브레이크 전등 원본 에셋 복원

> 상태: 완료

## 1차 — CSS 보정 제거와 래스터 에셋 복원

- 작업내용: 전등 절단면을 덮던 CSS 캡을 제거하고, 투명 전경 이미지에서 잘린 전등 관절과 지지대를 실제 이미지 픽셀로 복원한다.
- 논의 필요: 인물·책상·전등 갓의 기존 외형과 배치를 보존하면서 전등 상단만 자연스럽게 이어야 한다.
- 선택지:
  - A. 기존 전경을 보존하고 이미지 생성 결과에서 전등 상단 복원 영역만 투명 레이어로 합성한다.
  - B. 전경 전체를 새 이미지로 교체한다.
- 추천: A. 생성 결과의 변경 범위를 전등 상단으로 제한해 인물과 기존 UI 정합성을 보존한다.
- 사용자 방향: CSS 캡을 제거하고 투명 전경 이미지의 전등을 실제로 복원한다.

## 완료 조건

- 전등 상단이 CSS 도형 없이 실제 이미지 에셋으로 이어진다.
- 인물·책상·전등 갓의 기존 시각 요소가 유지된다.
- 제목·번호·상태 배지와 전경이 겹치지 않는다.
- 모바일 및 데스크톱 실제 화면에서 절단면과 가로 스크롤이 보이지 않는다.

## 완료 결과

- CSS로 그린 검은 캡을 완전히 제거했다.
- `imagegen`으로 복원한 전등 상단만 기존 전경에 합성한 1934×1013 투명 WebP 에셋을 추가했다.
- 헤더와 이미지 영역에 전경을 두 번 복제하지 않고 복원 에셋 한 장을 걸쳐 배치해 전등·인물 연결을 일치시켰다.
- 제목 묶음에 흰색 안전 배경과 오른쪽 패딩을 적용해 전등 지지대가 글자를 침범하지 않게 했다.
- 375px, 480px, 1040px에서 이미지 로드와 가로 스크롤 부재를 확인했다.
