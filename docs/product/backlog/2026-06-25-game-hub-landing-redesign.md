# 게임 허브 랜딩 재구축 (섹션형 + 카테고리 확장)

## 배경

사용자 제공 목업처럼 게임 허브 홈을 **섹션형 랜딩**으로 재구축한다. 기존은 단일
평면 그리드 + 국가 토글이었으나, 목업은 ① 아이콘 카테고리 탭 바 ② 운영자 추천
히어로 섹션 ③ 추억의 플래시 게임 행 ④ 인기 게임 행 ⑤ "전체 게임 보러가기"
버튼으로 구성된다.

## 작업내용

1. **카테고리 탭 확장**: 전체 / 추천 / 추억게임 / 인기게임 / 쯔꾸르 / 퍼즐 / 액션 /
   슈팅 / 레이싱 / 스포츠 / 아케이드 / 캐주얼 / **2인용**. 아이콘 포함.
2. **컬렉션 모델 도입**(`game-catalog.ts`): 추천·추억게임·인기게임·쯔꾸르·2인용은
   큐레이션 slug 컬렉션(genre category와 분리). 장르 탭(퍼즐/액션/…)은 기존 category
   필터를 그대로 쓴다. feed 게임은 자동으로 장르에만 매핑되므로 컬렉션은 curated 전용.
   - `retro`(추억게임) = `kind === "swf"` 자동 도출(허락받은 플래시만).
   - `twoPlayer`(2인용) = 온라인 대전/멀티 게임 큐레이션(smash-karts/bullet-force/snake-io).
   - `rpgmaker`(쯔꾸르) = 현재 공개 게임 없음 → "준비 중" 빈 상태.
3. **랜딩/그리드 모드 분기**(`page.tsx`):
   - 랜딩(파라미터 없음): 섹션형. 운영자 추천(region featured) 히어로 + 추억의 플래시 +
     인기 게임 행 + "전체 게임 보러가기".
   - 그리드(`?category=`/`?collection=`/`?view=all`/`?q=`): 필터 그리드 + 페이지네이션.
4. **검색**: 헤더/상단 검색 폼. no-JS GET(`?view=all&q=`)으로 제목 부분일치 필터.
5. **카드 태그 칩**: `getGameTags(game)` = 카테고리 라벨 + curated 태그(slug 맵). GameEntry
   스키마는 바꾸지 않고 보조 맵으로 도출(과침습 회피).

## 논의 필요

- 쯔꾸르 탭은 공개 게임이 0개 → 빈 상태로 노출할지 숨길지. 사용자가 탭 목록에 명시했으므로
  "준비 중" 빈 상태로 노출.
- 헤더 중앙 검색바를 CommonProductHeader에 넣을지, main 상단에 둘지. 공통 헤더 침습을 피해
  main 상단 검색 행으로 배치.

## 선택지

- (A) category에 rpgmaker/twoPlayer를 추가 → feed 매핑·테스트 영향. ❌
- (B) 컬렉션(curated slug)로 분리 → genre 모델 불변, 테스트 영향 없음. ✅ 채택

## 추천

(B) 컬렉션 분리 + 섹션형 랜딩 + no-JS 검색. region 토글은 기존 기능이므로 유지.

## 사용자 방향

목업대로 재구축, 2인용 카테고리 추가, 3번째 섹션은 "가볍게" 대신 "인기게임".
