# 카드방 런타임 연결 복구 백로그 (2026-05-18)

## 1차: dev:all 카드방 런타임 연결 복구

- 작업내용
  - `pnpm dev:all`이 배정한 backend/race-server 포트를 web과 race-server가 같은 값으로 공유하게 만든다.
  - 카드방 상세 화면에서 race-server가 카드방 상태를 불러올 때 잘못된 기본 backend 포트로 붙지 않게 한다.
  - 사용자 화면의 카드방 문구에서 내부 프레임워크명을 제거한다.
- 논의 필요
  - 없음. 현재 사용자가 직접 재현한 카드방 입장 실패이며, 내부 구현명 노출은 즉시 제거 대상이다.
- 선택지
  1. dev:all 환경 전달만 고친다.
  2. dev:all 환경 전달과 사용자 문구를 함께 고친다.
  3. 카드방 구조를 크게 재설계한다.
- 추천
  - 2번. 원인은 런타임 환경 전달 누락에 가깝고, 화면 문구는 별도 리스크 없이 바로 정리 가능하다.
- 사용자 방향
  - 추천 기준으로 진행.
