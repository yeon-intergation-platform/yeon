# community board layout 2026-05-12

## 배경

`/community`가 트위터형 타임라인처럼 보이면서 실제 서버 값이 아닌 리포스트/좋아요/조회수/우측 추천 목데이터를 노출하고 있다. 사용자가 준 와이어프레임 기준의 흰 배경 게시판 구조로 정리한다.

## 1차

### 작업내용

- `/community`를 단일 중앙 컬럼 게시판 레이아웃으로 정리한다.
- 상단 실시간 채팅, 카테고리 필터, 닉네임/비밀번호/글쓰기 행, 카드형 게시글 순서로 배치한다.
- 트위터형 탭, 우측 사이드바, 핸들/아바타/리포스트/북마크/공유/가짜 좋아요/가짜 조회수를 제거한다.
- 채팅 API/폴링/게스트 전송 동작은 유지하고 빈 상태와 placeholder만 실제 동작에 맞춘다.
- `/community` 메타데이터에서 피드/타임라인 표현을 제거한다.

### 논의 필요

- 없음. 서버/API/DB 변경 없이 현재 클라이언트 UI만 정리한다.

### 선택지

1. 최소 변경: 기존 컴포넌트 상태/훅을 유지하고 렌더링만 게시판형으로 교체한다.
2. 전면 분리: 새 게시판 컴포넌트 구조로 파일을 쪼갠다.

### 추천

- 1번. 동작 보존이 중요하므로 `useCommunityFeed`, `CommunityChatWidget` API는 유지하고 표시 요소만 걷어낸다.

### 사용자 방향

- 추천 기준으로 진행한다.
