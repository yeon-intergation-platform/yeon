# 제품 헤더 시각 규격 통일 2차 백로그

> 상태: 완료. 2026-07-14에 키보드 타자연습·커뮤니티·오늘보드·게임을 플래시카드 기준의 제품 헤더 및 페이지 제목 규격으로 통일했다.

## 1차 — 기준과 공통 source of truth 확정

### 작업내용

- 플래시카드 홈을 기준으로 상단 제품 헤더(61px), 브랜드 글자(모바일 19px·데스크톱 23px), 아이콘 버튼(44px), 페이지 제목(모바일 27px·데스크톱 34px) 규격을 확정한다.
- 키보드 타자연습, 커뮤니티, 오늘보드, 게임의 현재 DOM·운영 화면을 비교해 헤더 프레임과 서비스 제목의 편차를 기록한다.
- 웹 전용 페이지 제목 컴포넌트를 product-shell에 두고, 모바일 header chrome은 parity registry의 `platform-divergent` 정책을 유지한다.

### 논의 필요

- 집중 모드(`/todo-service/focus`)는 몰입형 작업 화면이므로 일반 서비스 진입 헤더 규격에서 제외할지 판단이 필요하다.

### 선택지

1. 각 서비스에서 Tailwind 클래스를 직접 맞춘다.
2. 상단 제품 헤더와 페이지 제목을 각각 공통 source of truth로 두고 서비스별 내용만 주입한다.

### 추천

- 2번. 반복되는 크기와 글자 규격을 한 곳에서 관리하고, 서비스별 기능·카피는 유지한다.

### 사용자 방향

- 플래시카드의 헤더 크기·글자 크기·요소 크기를 다른 서비스에도 일관되게 적용한다.

## 2차 — 대상 서비스 적용

### 작업내용

- 키보드 타자연습, 커뮤니티, 오늘보드, 게임 홈의 페이지 제목·설명·보조 정보 영역을 공통 제목 규격으로 교체한다.
- 오늘보드와 게임의 상단 제품 헤더도 플래시카드와 같은 공통 유틸리티 크기·배치를 사용하게 한다.
- 서비스별 검색창, 날짜, 통계, 채팅, 게임 분류처럼 본문 기능은 기존 위치와 동작을 보존한다.

### 논의 필요

- 서비스별 도움말의 존재 여부와 BGM의 재생 상태는 서비스 기능이므로, 외형 규격만 강제하고 콘텐츠 슬롯은 기존 동작을 유지한다.

### 선택지

1. 모든 헤더의 문구와 기능을 완전히 동일하게 만든다.
2. 프레임·타이포그래피·터치 타깃만 동일하게 만들고 서비스별 내용·동작은 유지한다.

### 추천

- 2번. 정보 구조와 기능을 훼손하지 않으면서 사용자가 요구한 시각 일관성을 만든다.

### 사용자 방향

- 추천 기준으로 진행한다.

## 3차 — 화면 검증과 main 반영

### 작업내용

- 각 대상 서비스의 desktop(1440px)·mobile(375px) 화면을 Playwright로 캡처해 헤더 프레임, 브랜드 글자, 버튼 크기, 페이지 제목을 비교한다.
- `@yeon/web` lint/typecheck와 `git diff --check`를 통과시킨다.
- 변경 파일만 한국어 커밋으로 커밋하고 PR(main)을 생성·병합한다.

### 논의 필요

- 운영 배포 반영은 GitHub Actions 비동기 흐름을 사용하므로 PR merge 직후 장시간 폴링하지 않는다.

### 선택지

1. 코드 검증만 수행한다.
2. 코드 검증과 실제 브라우저 화면 증거를 함께 남긴다.

### 추천

- 2번. 사용자가 지적한 문제 자체가 시각 편차이므로 브라우저 증거가 필요하다.

### 사용자 방향

- 추천 기준으로 진행한다.
