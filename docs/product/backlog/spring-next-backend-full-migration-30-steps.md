# Next.js 백엔드 전체 스프링 이관 60단계 백로그 (2026-05-09)

## 범위 및 기준

- 대상: `apps/web/src/app/api/v1` 하위의 백엔드 라우트(`route.ts`)를 Next.js에서 Spring Boot(`apps/backend`)로 이관
- 현재 기준 스냅샷:
  - 총 라우트: 111개 (테스트 제외)
  - 도메인별 라우트 수: `auth(1), card-decks(10), chat-service(17), counseling-records(11), integrations(19), life-os(4), members(1), mobile(1), public-check-sessions(4), space-templates(3), spaces(30), typing-character-frames(2), typing-decks(6), users(1)`
  - `@/server/services/*` 직접 의존 라우트: 46개
- 규칙
  - 기능 동작 동일성 우선
  - `apps/web` UI/BFF는 전환 완료 전까지 라우팅/프록시 역할로 점진 축소
  - 차수는 `작업내용/논의 필요/선택지/추천/사용자 방향` 필수 표기

- 41~50차수: 클라이언트/배포/운영 자동화 및 모니터링
- 51~60차수: 운영 전환, 보안/감사/정리 및 조직적 정착
