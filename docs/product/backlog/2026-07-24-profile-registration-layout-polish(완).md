# 프로필 등록 폼 간격과 닉네임 입력 개선

- 상태: 완료
- 결과: 사진 편집과 닉네임 편집을 반응형 흐름으로 분리하고, 닉네임 입력을 높이 48px의 캡슐형 필드로 변경했다.

## 1차수

- 작업내용: 프로필 등록 카드의 제목, 사진 선택, 닉네임 입력, 저장 행동 사이 간격과 정렬을 하나의 흐름으로 정리한다.
- 논의 필요: 프로필 사진과 닉네임 중 어느 항목을 더 먼저 강조할지 정해야 한다.
- 선택지:
  - 사진을 크게 강조하고 닉네임 입력을 아래에 유지한다.
  - 사진은 보조 항목으로 줄이고 닉네임 입력과 저장 행동을 핵심 흐름으로 강조한다.
- 추천: 닉네임 입력과 저장을 핵심 흐름으로 두고 사진은 같은 편집 영역 안의 보조 항목으로 묶는다.
- 사용자 방향: 현재의 어색한 간격을 고치고, 갑자기 튀어나온 직사각형 닉네임 입력 디자인을 바꾼다.

## 2차수

- 작업내용: 닉네임 라벨을 입력창 위에 고정하고 입력창·사진 버튼·저장 버튼의 둥근 형태와 높이를 통일한다.
- 논의 필요: 입력창을 완전한 캡슐형으로 만들지, 작은 라운드 사각형으로 유지할지 정해야 한다.
- 선택지:
  - 기존 `rounded-lg`를 유지하고 간격만 수정한다.
  - `rounded-full` 캡슐형 필드로 바꿔 주변 버튼과 시각 언어를 통일한다.
- 추천: 캡슐형 필드로 바꿔 직사각형 인상을 없애고 포커스 링으로 입력 상태를 분명히 한다.

## 3차수

- 작업내용: 인증된 `/profile` 화면을 데스크톱과 모바일 viewport에서 Playwright로 확인하고 before/after 증거를 남긴다.
- 논의 필요: 로컬 인증 세션을 재현하지 못하면 컴포넌트 단독 증거로 대체할지 정해야 한다.
- 선택지:
  - 로컬 개발 로그인을 사용해 실제 라우트를 검증한다.
  - 인증이 불가능할 때만 정적 렌더링 증거와 컴포넌트 테스트로 대체한다.
- 추천: 로컬 개발 로그인을 사용해 실제 라우트를 검증한다.
