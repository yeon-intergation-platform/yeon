# Support 실제 안내와 오류 신고 진입 개선

## 1차

### 작업내용

- NEXA 디스코드 봇 추가 가이드에 실제 설치 페이지와 직접 초대 링크를 명시한다.
- NEXA 데스크톱 앱 설치와 디스코드 봇 추가를 같은 절차처럼 보이지 않게 분리한다.
- “YEON 계정으로 로그인하는 방법” 같은 기본 사용법 글은 support 런타임 목록에서 제외한다.
- 로그인 풀림 글은 브라우저/주소/계정 상태 확인 중심으로 다시 작성한다.
- 오류 신고 글은 긴 체크리스트가 아니라 어디서 바로 신고하는지와 최소 정보만 안내한다.
- support 홈에 “오류 신고하기” 버튼을 추가한다.

### 논의 필요

- 오류 신고 접수 위치를 별도 폼으로 만들지, 현재 공개 문의 이메일로 먼저 받을지 결정이 필요하다.

### 선택지

- 선택지 A: 현재 공개 문의 이메일로 `mailto:` 오류 신고 버튼을 먼저 제공한다.
- 선택지 B: 별도 신고 폼과 백엔드 저장소를 만든다.

### 추천

- 현재 단계에서는 선택지 A가 적절하다. 별도 폼은 서버 저장·스팸 방지·개인정보 처리 범위가 늘어나므로 지금은 공개 문의 이메일을 명확한 신고 진입점으로 사용한다.

### 사용자 방향

- support는 실제 사용자가 바로 따라 할 수 있는 수준으로 고친다.
- 오류 신고는 어렵게 만들지 않고 서비스 메인에서 바로 누를 수 있게 한다.
