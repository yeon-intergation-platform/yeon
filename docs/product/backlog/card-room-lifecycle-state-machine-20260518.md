# 카드방 로비-학습-종료 상태 전이 안정화 백로그

## 배경

카드방에서 브라우저 새로고침(F5) 또는 일시적인 연결 끊김이 명시적 퇴장처럼 처리되어 참가자와 카드방 상태가 사라지는 문제가 있다. 현재 화면도 입장 즉시 학습 패널이 노출되어 로비/준비/시작/종료의 제품 시나리오가 분리되어 있지 않다.

## 1차

### 작업내용

- 카드방 실시간 이벤트에 명시적 `START`, `END`, `LEAVE`를 추가한다.
- Colyseus `onLeave`는 네트워크/새로고침 이탈로만 보고 Spring 참가자 삭제를 호출하지 않는다.
- 명시적 나가기 버튼 또는 이벤트에서만 Spring 참가자 퇴장을 호출한다.
- Spring 카드방 참가를 `roomId + userId/guestId` 기준 재입장 가능하게 만들어 F5/세션 복구 시 기존 참가자와 카드 스냅샷을 재사용한다.
- Spring 카드방 시작/종료 API를 추가하고 방장/역할/준비 상태 검증을 둔다.
- 웹 카드방 화면을 `WAITING` 로비와 학습 진행 화면으로 분리한다.

### 논의 필요

- 종료 상태를 기존 `finished`와 별도 `closed`로 유지할지 여부.
- 진행 중 새 참가자를 관전자로 받을지, 재입장만 허용할지 여부.

### 선택지

1. 최소 수정: `onLeave` 삭제 호출만 막고 UI는 유지한다.
2. 권장 수정: 로비/시작/종료 이벤트와 UI를 함께 추가한다.
3. 대규모 수정: 참가자 온라인/오프라인 상태 컬럼까지 추가한다.

### 추천

2번. 지금 문제는 네트워크 이벤트와 제품 상태 전이가 섞인 것이므로, 명시적 시작/종료 이벤트를 추가하고 새로고침은 재접속으로 처리해야 한다.

### 사용자 방향

권장 수정 기준으로 진행한다.
