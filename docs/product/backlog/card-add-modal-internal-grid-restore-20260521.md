# 카드 추가 모달 내부 그리드 정렬 복구 (2026-05-21)

## 1차 — main 누락 범위 재확인

### 작업내용

- 현재 main에는 카드 추가 모달의 바깥 2열/2행 grid만 있고, 라벨/툴바/상태/본문을 같은 내부 grid 축으로 맞추는 작업은 남아 있지 않다.
- 카드 질문/답변 편집 영역의 내부 구조를 다시 grid 기준으로 정렬한다.

### 논의 필요

- 본문 시작 위치를 라벨 열 아래까지 확장할지, 툴바 컨트롤 열과 같은 x축에 맞출지.

### 선택지

1. 본문을 전체 폭으로 유지한다.
2. 라벨 열을 고정 gutter로 두고 본문은 툴바 컨트롤 열과 같은 x축에서 시작한다.

### 추천

- 2안. 사용자가 지적한 “다 그리드” 요구에 맞게 라벨/툴바/본문의 열 기준을 맞춘다.

### 사용자 방향

- 이전에 버린 내부 그리드 정렬 작업을 복구한다.

## 2차 — compact editor 내부 grid 복구

### 작업내용

- compact toolbar를 `라벨 / 도구 버튼 / 상태 pill` 3열 grid로 바꾼다.
- editor body를 `라벨 gutter / 본문` 2열 grid로 바꿔 질문/답변 모두 같은 x축에서 입력이 시작되게 한다.
- 표 편집 bar, 업로드/오류 메시지는 full-width 행으로 유지한다.

### 논의 필요

- 라벨 열 폭을 고정값으로 둘지 반응형으로 둘지.

### 선택지

1. 기존 라벨 최소 폭 72px 근처를 유지한다.
2. toolbar와 본문 정렬이 더 안정적인 88px 고정 열로 둔다.

### 추천

- 2안. 질문/답변 라벨 폭 차이에도 내부 grid 축이 흔들리지 않는다.

### 사용자 방향

- 추천 기준으로 진행한다.

## 3차 — 검증과 main 반영

### 작업내용

- lint/typecheck/diff check를 실행한다.
- branch → PR(main) → merge로 반영한다.

### 논의 필요

- 브라우저 visual QA까지 할지.

### 선택지

1. 정적 검증 후 PR 반영한다.
2. dev server를 다시 띄워 Playwright로 확인한다.

### 추천

- 1안. 현재 요청은 누락 diff 복구이며 CSS/grid 구조 변경이 작다.

### 사용자 방향

- 작업 후 main 반영한다.
