# 랜딩 플래시카드 덱 카드 프레임 브레이크 적용

> 상태: 완료

## 1차 — 플래시카드 덱 원본 에셋 분리

- 작업내용: 사용자 제공 이미지를 플래시카드 덱용 배경 WebP와 투명 전경 WebP로 분리한다.
- 논의 필요: 카드·전등 빛·고양이·화분의 밝은 영역은 보존하면서 외부 흰 배경만 제거해야 한다.
- 선택지:
  - A. 이미지 외곽과 연결된 흰 배경만 투명화해 내부 밝은 요소를 보존한다.
  - B. 밝은 픽셀 전체를 일괄 투명화한다.
- 추천: A. 연결 영역 기반 제거로 플래시카드와 조명 디테일의 손실을 방지한다.
- 사용자 방향: 기존 프레임 브레이크 카드와 동일한 두 장 구조로 적용하고 main에 병합한다.

## 2차 — 플래시카드 덱 카드 설정 추가

- 작업내용: `card-service` slug에 순서 3, 배경, 전경, 대체 텍스트를 등록해 공용 프레임 브레이크 렌더링을 재사용한다.
- 논의 필요: 제목·상태 배지 안전영역과 기존 타자·백지 카드의 표현을 유지해야 한다.
- 선택지:
  - A. 기존 `FRAME_BREAK_ARTWORK_BY_SERVICE` 설정에 카드 서비스 데이터만 추가한다.
  - B. 플래시카드 전용 JSX를 복제한다.
- 추천: A. 시각적 차이는 에셋 데이터로만 관리하고 레이아웃은 한 곳에서 유지한다.
- 사용자 방향: 제공한 플래시카드 덱 이미지를 기존 카드와 같은 프레임 브레이크로 적용한다.

## 완료 조건

- 플래시카드 덱 카드가 배경 1장과 투명 전경 1장으로 렌더링된다.
- 제목·번호·상태 배지가 전경에 가려지지 않는다.
- 타자·백지 학습 카드의 기존 표현과 동작이 유지된다.
- 375px, 480px, 1040px에서 이미지 로드와 가로 스크롤 부재를 확인한다.

## 완료 결과

- 보라색 방 배경을 1412×425 WebP로 분리했다.
- 외곽과 연결된 흰 영역만 제거해 전등·플래시카드·고양이·화분·인물을 보존한 1448×632 투명 WebP 전경을 만들었다.
- 기존 `FRAME_BREAK_ARTWORK_BY_SERVICE`에 `card-service` 순서 3과 두 에셋, 대체 텍스트를 추가했다.
- 375px, 480px, 1040px에서 제목·상태 배지 비겹침, 이미지 로드, 가로 스크롤 부재를 확인했다.
- web lint, typecheck, build와 Universal UI parity, 프로젝트 SSOT 검증을 통과했다.
