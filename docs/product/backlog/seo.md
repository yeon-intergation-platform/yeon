### 차수 2 — 핵심 퍼널 이벤트 계측 추가

- 작업내용: 공통 analytics 유틸을 만들고 `login`, `typing start`, `room create`, `deck open` 핵심 액션을 GA4 이벤트로 수집한다.
- 논의 필요: 이벤트명을 현재 커스텀 이벤트로 고정하고, 추천 이벤트명/전자상거래 스키마로 맞출지는 추후 분석 대시보드 요구에 따라 재정의한다.
- 선택지:
  - 옵션 A. 화면 액션 지점에 직접 `trackEvent`를 붙여 최소 범위로 시작한다.
  - 옵션 B. 라우터/도메인 훅 단위로 더 추상화해 전역 추적 계층을 먼저 만든다.
- 추천: 옵션 A. 현재는 퍼널 가시성이 급하고, 액션 지점이 명확해 빠르게 운영 데이터를 쌓을 수 있다.
- 사용자 방향: 추천 기준으로 진행.

### 차수 3 — 루트 카드서비스 비로그인 진입 복구

- 작업내용: 루트 랜딩의 카드서비스 접근 정책을 실제 게스트 사용 흐름과 일치시키고, 비로그인 상태에서도 로그인 모달 없이 `/card-service`로 진입되게 수정한다.
- 논의 필요: 카드서비스를 장기적으로 완전 공개 서비스로 볼지, 게스트 진입 후 로그인 전환 유도형 mixed 서비스로 유지할지 운영 기준을 정해야 한다.
- 선택지:
  - 옵션 A. 루트 랜딩에서 카드서비스 접근 정책을 `mixed`로 바꿔 비로그인 진입을 허용한다.
  - 옵션 B. 카드서비스 내부 게스트 기능을 제거하고 랜딩의 로그인 모달 정책을 유지한다.
- 추천: 옵션 A. 현재 구현과 사용자 기대가 이미 게스트 사용 가능 쪽이라 루트 정책만 맞추는 것이 최소 변경이다.
- 사용자 방향: 추천 기준으로 진행.
