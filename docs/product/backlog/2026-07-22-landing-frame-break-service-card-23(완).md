# 랜딩 쥔장 포트폴리오 프레임브레이크 전경 적용

> 상태: 완료

## 1차 — 포트폴리오 투명 전경 제작

- 작업내용: 제공된 포트폴리오 장면에서 전등·대형 모니터·키보드·태블릿·포트폴리오 문서를 투명 전경으로 제작한다.
- 논의 필요: 원본 전체를 카드 배경으로 쓰면 1~8번과 다른 구조가 되고 공용 배경이 가려진다.
- 선택지:
  - A. 원본 이미지를 카드 이미지로 통째로 사용한다.
  - B. 서비스 정체성을 보여주는 책상 장면만 투명 전경으로 분리한다.
- 추천: B. 공용 배경을 유지하면서 포트폴리오 화면과 문서가 선명하게 보인다.
- 사용자 방향: 제공 이미지를 쥔장의 포트폴리오 카드에 넣는다.

## 2차 — 9번 카드 연결과 구도 보정

- 작업내용: 9번 카드에 새 전경을 연결하고 전등과 모니터가 헤더 하단을 자연스럽게 넘도록 크기와 위치를 맞춘다.
- 논의 필요: 모니터 화면의 세부 UI를 살리면서 번호·제목과 충돌하지 않아야 한다.
- 선택지:
  - A. 전체를 작게 축소해 안전 여백을 우선한다.
  - B. 모니터와 전등 돌출은 유지하고 문서 하단 일부를 카드 본문 뒤로 숨긴다.
- 추천: B.
- 사용자 방향: 기존 공용 배경은 변경하지 않는다.

## 3차 — 반응형 검증

- 작업내용: 375px·1040px·1440px에서 제목 충돌, 전경 잘림, 가로 스크롤과 공용 배경 노출을 확인한다.
- 논의 필요: 없음.
- 선택지:
  - A. 데스크톱만 확인한다.
  - B. 모바일·태블릿·데스크톱을 함께 확인한다.
- 추천: B.
- 사용자 방향: localhost에서 바로 확인할 수 있게 한다.

## 완료 조건

- 9번 카드에 포트폴리오 화면과 작업 문서가 보이는 투명 전경이 적용된다.
- 전등과 모니터가 제목을 가리지 않고 프레임브레이크를 만든다.
- 1~9번이 같은 공용 배경을 유지한다.
- 세 화면폭에서 가로 스크롤이 없다.

## 완료 결과

- 제공 이미지의 전등·포트폴리오 모니터·키보드·태블릿·문서 5종을 한 투명 전경으로 재구성했다.
- `owner-portfolio-frame-break-foreground-v1.webp` 1876×852 투명 WebP를 9번 카드에 적용했다.
- 전경 폭 `w-[min(185%,38rem)]`와 `-bottom-6` 위치로 모니터 가독성과 헤더 돌출을 맞췄다.
- 공용 배경의 git blob 해시가 `origin/main`과 동일함을 확인했다.
- 375px·1040px·1440px에서 공용 배경 9개, 전경 9개, 가로 overflow 0을 확인했다.
