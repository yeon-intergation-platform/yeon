# 카드 표 최소 셀 크기와 렌더 경로 일치 백로그 (2026-05-21)

## 배경

편집기 표는 비어 있는 셀도 안정적인 최소 크기를 가져서 보기 좋다. 반면 카드 미리보기/실제 학습 카드에서는 비어 있는 열이 너무 좁아 보일 수 있다. 표 렌더링은 편집기, 카드 미리보기, 카드 학습 화면에서 같은 최소 셀 크기와 표시 규칙을 가져야 한다.

## 1차

### 작업내용

- 카드 표 렌더러 경로를 찾아 미리보기/학습 카드가 같은 MarkdownContent 표시 규칙을 쓰는지 확인한다.
- HTML/Markdown table 렌더링에 안정적인 최소 셀 폭/높이를 지정한다.
- 편집기 표도 같은 최소 셀 기준을 사용하게 맞춘다.
- 다른 표 표시 경로가 있으면 같은 규칙으로 정리한다.

### 논의 필요

- 없음. 사용자가 최소 크기와 미리보기/학습 표시 일치를 명시했다.

### 선택지

1. 편집기만 그대로 두고 미리보기만 보정한다.
   - 실제 학습 카드와 불일치 가능성이 남는다.
2. 카드 표 렌더 공통 컴포넌트/스타일 기준을 찾아 모든 카드 표시 경로에 같은 최소 셀 크기를 적용한다.
   - 요구와 일치한다.
3. 표 데이터 삽입값에 공백을 넣는다.
   - 렌더 문제를 데이터로 우회하는 땜빵이다.

### 추천

2번. 표시 계층에서 표 최소 크기를 정의해 편집/미리보기/학습 화면의 source of truth를 맞춘다.

### 사용자 방향

추천 기준으로 진행한다.

## 완료 기준

- 빈 셀/빈 열도 카드 미리보기와 실제 학습 카드에서 안정적인 최소 크기로 보인다.
- 편집기와 카드 표시의 표 밀도/최소 크기 기준이 일치한다.
- web lint/typecheck/build를 통과한다.
