# 랜딩 키보드 타자연습 카드 신규 에셋 교체

> 상태: 완료

## 1차 — 신규 타자연습 원본 에셋 분리

- 작업내용: 사용자 제공 이미지를 키보드 타자연습용 방 배경 WebP와 투명 전경 WebP로 분리한다.
- 논의 필요: 전등 빛과 모니터·키보드·머그컵·인물을 보존하면서 외부 흰 배경만 제거해야 한다.
- 선택지:
  - A. 이미지 외곽과 연결된 흰 배경만 투명화해 내부 밝은 요소를 보존한다.
  - B. 밝은 픽셀 전체를 일괄 투명화한다.
- 추천: A. 연결 영역 기반 제거로 전등 빛과 모니터 외곽의 손실을 방지한다.
- 사용자 방향: 제공한 새 이미지로 기존 키보드 타자연습 프레임 브레이크 카드를 만든다.

## 2차 — 타자연습 카드 에셋 교체

- 작업내용: 기존 `typing-service` 프레임 브레이크 설정이 새 버전 에셋과 실제 크기를 참조하도록 변경한다.
- 논의 필요: 브라우저/CDN 캐시 충돌을 피하면서 번호 1, 제목 안전영역, 상태 배지, CTA를 유지해야 한다.
- 선택지:
  - A. 새 버전 파일명을 사용하고 설정의 경로·크기·대체 텍스트만 교체한다.
  - B. 기존 파일 내용을 덮어쓰고 동일 URL을 유지한다.
- 추천: A. 새 파일명으로 배포해 이전 정적 이미지 캐시가 남지 않게 한다.
- 사용자 방향: 기존 카드 구조는 유지하고 새 이미지로 표현한다.

## 완료 조건

- 키보드 타자연습 카드가 새 배경 1장과 새 투명 전경 1장으로 렌더링된다.
- 전등은 CSS 도형 없이 원본 이미지 픽셀로만 표시된다.
- 제목·번호·상태 배지가 전경에 가려지지 않는다.
- 백지 학습·플래시카드 덱 카드의 기존 표현과 동작이 유지된다.
- 375px, 480px, 1040px에서 이미지 로드와 가로 스크롤 부재를 확인한다.

## 완료 결과

- 새 방 배경을 1412×510 WebP로 분리했다.
- 외곽 연결 흰 영역을 제거하고 전등 내부 조명을 원본 픽셀로 보존한 1448×552 투명 WebP 전경을 만들었다.
- `typing-service`가 캐시 충돌 없는 v2 경로와 실제 이미지 크기를 참조하도록 변경했다.
- CSS 도형을 추가하지 않고 번호 1, 제목, 상태 배지, CTA와 기존 공용 레이아웃을 유지했다.
- 375px, 480px, 1040px에서 제목·상태 배지 비겹침, 이미지 로드, 가로 스크롤 부재를 확인했다.
- web lint, typecheck, build와 Universal UI parity, 프로젝트 SSOT 검증을 통과했다.
