# 상담기록 프론트 노출 제거 백로그

## 배경

상담기록 서비스는 메인 서비스가 아니며 장기적으로 제거 대상이다. 사용자가 구글 검색, 브라우저 탭, 메인 화면에서 `상담기록` 용어를 접하지 않도록 프론트 노출 문자열을 제거한다.

## 1차

### 작업내용

- 웹 프론트에서 `상담기록`/`상담 기록`으로 노출되는 메타데이터, 내비게이션, 메인 화면 문구를 찾는다.
- 백엔드 비즈니스 로직과 상담 워크스페이스 기능 구조는 변경하지 않고, 사용자 노출 문구만 제거한다.
- 필요한 경우 검색 색인/탭 제목에 쓰이는 Next.js metadata를 비노출 또는 일반 서비스명으로 바꾼다.

### 논의 필요

- 상담 워크스페이스 자체를 완전히 숨길지, 용어만 숨길지.

### 선택지

1. 프론트 노출 용어만 제거하고 경로/기능은 유지한다.
2. 상담 워크스페이스 진입점까지 제거한다.
3. 라우트 자체를 삭제한다.

### 추천

- 1번. 현재 지시는 “앞단에서 용어를 접하지 않게”이므로 기능 삭제보다 검색/탭/메인 노출 문구 제거가 작고 안전하다.

### 사용자 방향

- 추천 기준으로 진행한다.
