# 카드서비스 프로필 hydration flicker 제거 (2026-05-18)

## 1차

### 작업내용

- `/card-service` 진입 직후 저장된 타자/카드 공용 프로필을 읽기 전 기본 캐릭터(camel)가 순간 렌더되는 문제를 제거한다.
- 프로필 저장소(`yeon:typing-profile`) hydrate 완료 전에는 캐릭터 스프라이트를 그리지 않는 placeholder를 유지해 레이아웃 점프와 잘못된 캐릭터 노출을 막는다.

### 논의 필요

- 없음. 사용자는 카드서비스 진입 시 낙타가 순간 보이는 flicker를 명시적으로 버그로 보고했다.

### 선택지

1. `useTypingProfile` 초기값을 localStorage 동기 읽기로 바꾼다.
2. `loaded`가 false일 때 프로필 카드를 숨기거나 skeleton을 보여준다.
3. 캐릭터 기본값을 최근 인기 캐릭터로 바꾼다.

### 추천

- 2번. 기존 hook 계약(`loaded`)을 활용하고 SSR/브라우저 저장소 접근 경계를 건드리지 않으면서 잘못된 캐릭터 노출만 막는다.

### 사용자 방향

- 추천 기준으로 진행한다.

## 2차

### 작업내용

- `/card-service/decks`에서 `/card-service` 홈으로 돌아가는 명시적 뒤로가기 네비게이션을 추가한다.
- 브라우저 히스토리에 의존하지 않고 항상 카드서비스 홈으로 이동하는 링크를 제공한다.

### 논의 필요

- 없음. 사용자가 덱 목록에서 홈으로 돌아가는 `<-` 네비게이션 부재를 명시했다.

### 선택지

1. `router.back()` 버튼을 둔다.
2. `/card-service` 고정 링크를 둔다.
3. 공용 헤더에 전역 뒤로가기 슬롯을 추가한다.

### 추천

- 2번. 사용자가 원하는 목적지가 명확하고, 다른 진입 경로에서도 카드서비스 홈으로 안정적으로 복귀한다.

### 사용자 방향

- 추천 기준으로 진행한다.
