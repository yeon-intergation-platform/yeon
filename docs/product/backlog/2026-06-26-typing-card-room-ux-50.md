# 타자방·카드방 사용감(UX) 점검 — 문제 50가지 + 수정

> 근거: 로컬(`localhost:3100`) Playwright 실측 스크린샷(데스크톱 1280 / 모바일 390) +
> 컴포넌트 코드 정독. 스크린샷: `ai-log/hyeonjun/2026-06-26/room-ux-screenshots/`.
> 대상: `/typing-service/rooms`(타자방), `/card-service/rooms`(카드방) 및 생성/캐릭터/홈.
> 상태 표기: ✅ 이번에 수정 / ⏳ 후속.

## A. 전역·레이아웃·모바일 충돌

1. ✅ **모바일에서 커뮤니티 챗 위젯(`fixed inset-x-4 bottom-3 z-40`)이 콘텐츠 하단을 덮음.**
   카드/타자 레이아웃 둘 다. 방 목록·에러·빈상태 하단이 가려진다. → 본문에 하단 패딩(safe gap) 부여.
2. ✅ **타자방 모바일 "방 만들기" 플로팅 버튼(`fixed right-5 bottom-5 z-30`)이 챗 위젯과 충돌**(겹침/잘림). z-30 < z-40.
3. ✅ 타자방 모바일 플로팅 생성 버튼 2개가 가로로 화면 밖으로 **넘침(overflow)**.
4. ⏳ 챗 위젯이 모든 방 화면에서 우선순위 높게 떠 있어, 처음 방문자가 "이게 뭐지?" 혼란. 첫 진입 시 축소/딜레이 노출 검토.
5. ✅ 히어로 제목 `text-[48px]`(모바일)~`56px`(데스크톱)이 과도하게 큼 → 화면 위쪽을 제목이 독점, 정작 기능(목록)은 아래로 밀림.
6. ✅ 히어로 섹션 `min-h-[174px] py-10`로 상단 여백 과다(특히 모바일). 빈 공간만 크고 밀도 낮음.
7. ⏳ 좌하단 "N" 원형 요소(전역) 정체 불명확·라벨 없음 → 접근성/의미 불명.
8. ⏳ 페이지 배경이 순백 단색 + 얇은 회색 보더만이라 영역 구분이 약함(밋밋함).

## B. 타자방 로비

9. ✅ 백엔드/레이스서버 미연결 시 "타자방 서버에 연결할 수 없어요." 한 줄만 → "다시 시도" 버튼 추가.
10. ✅ 에러 상태 영역이 `min-h-[520px]`로 과하게 큼(빈 회색 박스가 화면을 가득 채워 휑함).
11. ⏳ 에러/로딩/빈 상태에서 상단 필터·검색·생성 UI가 사라져(`!isInitialEmpty`) 사용자가 할 수 있는 행동이 없음.
12. ✅ 필터 칩(전체/공개방/입장가능) `h-[50px] px-7 text-[16px]`로 과대 → 칩이라기보다 큰 버튼, 위계 혼란.
13. ⏳ 검색 인풋과 생성 버튼이 같은 줄(desktop)인데 생성 버튼이 검은 Primary 2종이라 시각 소음.
14. ✅ 방 카드 메타가 "한국어 · 짧은 글 · 보통 · N라운드 · 종료 · 일반전" 식의 가운뎃점 나열 → 가독성 낮음(라벨화 필요).
15. ⏳ 방 카드에 호스트/인원 배지가 많아(상태/좌석/게임타입/코드/호스트/인원/입장) 정보 과밀.
16. ✅ "#roomCode"가 mono 작은 회색이라 잘 안 보임 + 복사 불가.
17. ⏳ 데스크톱 방 카드 그리드 `md:grid-cols-[1fr_auto]`인데 모바일은 세로 적층되며 "입장" CTA가 카드 하단에 묻힘.

## C. 카드방 로비

18. ✅ **빈/에러 상태에 일러스트 없음**(타자방은 키보드 일러스트 있음) → 서비스 간 일관성 결여, 휑함.
19. ✅ 에러 제목 `text-[28px]`/빈상태 `text-[32px]`가 모바일에서 너무 커 챗 위젯과 겹쳐 잘림.
20. ✅ 에러/빈 상태 `min-h-[280px]`로 짧아 상단 필터와 바짝 붙고 하단은 위젯에 가림.
21. ✅ "카드방 서버 상태를 확인해 주세요." → 사용자가 할 수 있는 게 없음(재시도 버튼 부재).
22. ✅ 필터 비활성 변형이 `secondary`(회색 버튼)라 활성 pill과 대비가 약하고 버튼처럼 보임.
23. ⏳ 히어로 설명 2줄이 모바일에서 3줄로 깨지며 줄간 답답.
24. ⏳ 카드방 카드 목록과 타자방 카드가 클래스만 비슷하고 배지/정보 구조가 제각각(공용화 부족).

## D. 방 만들기(생성 모달/폼)

25. ⏳ 카드 생성 모달이 길어 데스크톱에서도 하단 액션("카드방 만들고 입장하기")이 접힘/스크롤 필요.
26. ⏳ 카드 생성 모달 하단 액션이 3개(`내 덱 보기`/`로비로 돌아가기`/`카드방 만들고 입장하기`)인데 정렬·위계 불명확(주 CTA가 회색 비활성처럼 보임).
27. ✅ "덱 선택" 셀렉트가 비어 있을 때(덱 없음) 안내·생성 유도 없음 → 만들 수 없는 막다른 길.
28. ⏳ 모달 안에 "카드방 프로필"(캐릭터)과 "닉네임" 인풋이 중복적으로 보임(역할 구분 약함).
29. ⏳ 타자 생성 모달 visibility 라디오가 `border` 카드형인데 선택 표시가 미묘(inset shadow만)해 상태 인지 어려움.
30. ⏳ 타자 생성 모달 "공개/비공개" 외 핵심 옵션(난이도/모드/라운드)은 "입장 후 설정"으로 숨겨 기대-결과 불일치.
31. ⏳ 모달 제목과 본문 사이 여백/구분선 부족.
32. ✅ 생성 실패 시 에러가 인풋과 동일 톤 박스라 에러로 인지 안 됨(색 강조 부재).

## E. 캐릭터·프로필

33. ⏳ 타자 "캐릭터 바꾸기"가 모달이 아니라 **홈(`/typing-service`)으로 페이지 이동** → 로비 맥락 이탈, 돌아오기 번거로움.
34. ⏳ 카드 "캐릭터 바꾸기"는 카드 홈으로 이동(동일 문제) — 인라인 캐릭터 선택 필요.
35. ⏳ 캐릭터 요약 카드 썸네일 배경이 흰 박스 + 캐릭터가 하단 정렬이라 위쪽 빈 공간 큼.
36. ⏳ 게스트 닉네임이 "Guest"로 고정 노출 → 첫 인상이 "비회원" 느낌, 닉네임 변경 유도 약함.
37. ⏳ 홈 "캐릭터 선택"에서 3개만 보이고 "더 보기(20개)"라 선택 비용 큼(그리드/검색 부재).

## F. 빈·에러·로딩 상태(공통)

38. ✅ 로딩 상태가 텍스트 한 줄("…불러오는 중")뿐 → 스켈레톤/스피너 부재.
39. ✅ 에러와 빈 상태의 시각 언어가 서비스마다 달라(아이콘 유무/문구/높이) 통일 안 됨.
40. ✅ 에러 문구가 개발자 시점("서버 상태 확인")이라 사용자 행동 가이드 부족.

## G. 반응형(모바일)

41. ✅ 모바일 필터 칩이 가로 스크롤(`overflow-x-auto`)인데 스크롤 가능 힌트(그라데이션/페이드) 없음.
42. ✅ 모바일에서 생성 CTA 접근이 플로팅 버튼에만 의존 → 위젯과 겹쳐 사실상 누르기 어려움.
43. ⏳ 모바일 히어로 제목+설명이 화면 1스크롤을 거의 차지 → 목록 도달 전에 스크롤 필요.
44. ✅ 모바일에서 방 카드 좌우 패딩(`px-6`)·카드 내부 `p-5`로 정보 폭 좁음.

## H. 접근성·인터랙션

45. ✅ 필터 버튼 `data-active`만으로 상태 표현 → `aria-pressed`/`role=tab` 부재(스크린리더 상태 불명).
46. ✅ 검색 인풋 카드방은 `aria-label` 없음(타자방은 있음) — 일관성/접근성 결여.
47. ⏳ 방 카드 전체가 링크인데 내부에 또 "입장" 텍스트 버튼 모양 → 중복/혼란, 키보드 포커스 1개로 OK지만 시각적 오해.
48. ⏳ 플로팅 생성 버튼 `z-30`이 위젯 아래라 탭 시 위젯이 먹음(터치 타깃 가림).
49. ⏳ 색상 대비: 회색 `#aaa` placeholder·`#666` 보조텍스트가 옅어 가독성 경계.
50. ⏳ BGM 토글이 헤더에 있으나 상태(켜짐/꺼짐) 시각 구분 약함.

- 검증: web lint/typecheck 통과, Playwright 전후 스크린샷(데스크톱·모바일) 비교.

## 2차: 풀스택 플로우 검증 + 추가 수정 (2026-06-28)

로컬에 **DB(Postgres)+백엔드(Spring 8081)+race-server(2567)+web(3100)** 전부 기동해 실제 플로우 검증.

- ✅ **타자방 E2E**: 로비 → "일반 타자방 만들기" → 방 생성(race-server) → `/rooms/{id}` 입장 →
  대기실(참가자 슬롯/채팅/시작하기) 정상. 콘솔 에러 없음.
- ✅ **백엔드 연동**: 카드 `/api/v1/card-rooms` 200, 타자 로비 "연결 에러" 사라지고 진짜 빈상태 노출.
- ⚠️ **카드방 E2E(다중 참가자 동기화)**: 게스트 덱이 IndexedDB라 Playwright storageState로 못 옮겨
  자동화로 끝까지 재현 불가. 단 카드방은 타자방과 동일한 BFF/룸 인프라(검증됨)를 쓴다. 수동(브라우저
  로그인/덱+카드 생성)으로 최종 확인 권장.
- ✅ **추가 수정**: 카드 생성 덱-없음 막다른 길 → 덱 생성 안내 CTA(#27), 생성 에러 빨강 강조(#32),
  필터 칩 크기 정상화(#12), 필터 `aria-pressed`(#45), 카드 검색 `aria-label`(#46).

### 발견된 신규 이슈(방 내부, 후속 ⏳)

- R1. 타자 방 입장 직후 채팅에 "[시스템] Guest님과의 연결이 잠시 끊겼습니다" 오해 메시지(생성→리다이렉트 재연결 흔적).
- R2. ✅ 참가자 슬롯 아바타를 선택 캐릭터 스프라이트로 교체(typing-room-screen 하드코딩 🙂 제거, 표준/점령전 슬롯 공통 SlotAvatar).
- R3. 방 정보가 헤더 메타행과 좌측 패널/대기실 우측에 3중 중복.
