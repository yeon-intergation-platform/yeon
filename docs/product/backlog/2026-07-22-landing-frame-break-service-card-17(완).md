# 랜딩 서비스 카드 전경 3종 통일

> 상태: 완료

## 1차 — 제공 이미지 전경 에셋 분리

- 작업내용: 제공된 커뮤니티·YEON Today·뉴스 이미지를 원본 구도와 소품을 유지한 투명 전경 에셋으로 분리한다.
- 논의 필요: 원본 3장은 모두 흰색 또는 체커보드가 실제 픽셀로 합성되어 있어 그대로 겹치면 공통 배경을 가린다.
- 선택지:
  - A. 완성 이미지를 카드 배경으로 사용한다.
  - B. 배경만 제거한 투명 전경을 만들고 기존 공통 배경 위에 배치한다.
- 추천: B. 1~8번 카드의 배경 일관성을 유지하면서 1~3번과 같은 프레임브레이크 구조를 확장할 수 있다.
- 사용자 방향: 4·5·7번도 1·2·3번처럼 누끼 전경으로 적용하고 배경은 건드리지 않는다.

## 2차 — 공통 배경 SSOT와 전경 배치 통일

- 작업내용: 커뮤니티 전용 완성형 배경 override를 제거하고 4·5·7번 카드에 투명 전경을 연결한다.
- 논의 필요: 신규 9번 카드는 이번 요청 범위 밖이므로 변경하지 않고, 기존 1~8 중 제공 전경이 있는 카드만 갱신한다.
- 선택지:
  - A. 서비스마다 선택적 배경 경로를 계속 허용한다.
  - B. 프레임브레이크 카드 배경을 공통 상수 한 곳으로 고정하고 서비스별 차이는 전경만 선언한다.
- 추천: B. 배경 일관성을 코드 구조에서도 보장하고 다시 개별 배경이 섞이는 일을 막는다.
- 사용자 방향: 1~8 카드가 같은 배경을 사용하도록 유지한다.

## 3차 — 반응형 시각 검증과 배포

- 작업내용: 375px·1040px·1440px에서 전경 크기, 제목·상태 배지 충돌, 카드 이미지 영역, 가로 스크롤을 Playwright로 확인한다.
- 논의 필요: 전경 원본마다 비율이 달라 동일한 width 규칙을 쓰되 하단 위치는 서비스별로 미세 조정할 수 있다.
- 선택지:
  - A. 한 화면 크기에서만 육안 확인한다.
  - B. 모바일·태블릿·데스크톱 스크린샷과 계산값을 함께 확인한다.
- 추천: B. 프레임브레이크 돌출과 크롭은 뷰포트별 회귀가 생기기 쉬우므로 세 기준을 모두 검증한다.
- 사용자 방향: localhost에서 즉시 확인 가능하게 유지하고 검증 후 main에 반영한다.

## 완료 조건

- 4·5·7번에 투명 전경이 적용되고 공통 배경 파일은 변경되지 않는다.
- 1~8번 카드가 동일한 배경 경로를 사용한다.
- 제목과 상태 배지가 전경에 가리지 않고 전경 외곽이 잘리지 않는다.
- 375px·1040px·1440px에서 가로 스크롤이 없다.

## 완료 결과

- 커뮤니티·YEON Today·뉴스 원본을 built-in `imagegen` 편집 모드에서 평면 크로마키 배경으로 분리하고 공식 `remove_chroma_key.py` 도구로 알파 WebP 3종을 만들었다.
- 커뮤니티 전용 완성형 배경 override를 제거하고 모든 프레임브레이크 카드가 `baekji-frame-break-background.webp` 한 경로만 참조하도록 단순화했다.
- 공통 배경 파일의 Git 객체 해시가 `origin/main`과 동일한 `072b6d2a090388cd6260ff070cc7814f57070fc7`임을 확인했다.
- 375px·1040px·1440px에서 공통 배경 9개, 전경 6개, 기존 커뮤니티 완성형 배경 0개, 가로 overflow 0px를 확인했다.
- 제목과 상태 배지를 전경보다 높은 안전 레이어에 유지해 말풍선·머리카락과 겹쳐도 텍스트가 가려지지 않는다.
