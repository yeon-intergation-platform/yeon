## 화면별 상세

### 1. 홈 — 서비스 선택 `/` — 16건

**#1. [높음 · 타이포그래피] 제목이 두 줄로 어색하게 끊겨 단어 중간에서 줄바꿈됨**

- 근거: 상단 제목 '현재 3가지 서비스를 운영 중입니다.'가 '중입'과 '니다.'로 분리되어, 한 단어인 '중입니다'가 줄 경계에서 깨진다. 의미 단위가 끊겨 가독성과 완성도가 떨어진다.
- 개선: word-break/keep-all(overflow-wrap: break-word + word-break: keep-all)을 적용해 어절 단위로 줄바꿈하거나, 제목 폰트 크기를 약간 줄여 더 자연스러운 줄바꿈 지점을 만든다.

**#2. [높음 · 어포던스/명확성] '바로 이동' 링크가 버튼인지 텍스트 링크인지 어포던스 불명확**

- 근거: 각 카드 하단의 '바로 이동'이 밑줄 없는 검은 굵은 텍스트로만 표시되고, 화살표 아이콘이나 버튼 배경/테두리가 없다. 위 구분선과 함께 있어 클릭 가능한 액션인지 단순 라벨인지 한눈에 판단하기 어렵다.
- 개선: '바로 이동'에 화살표(→) 아이콘을 붙이거나 버튼 스타일(배경 또는 테두리)을 적용하고, 카드 전체가 클릭 가능하면 hover/press 상태를 명확히 한다.

**#3. [보통 · 시각적 위계] 세 카드가 모두 '운영 중' 동일 배지 + 동일 '바로 이동' CTA로 시각적 구분 부족**

- 근거: 타자/카드/커뮤니티 3개 카드가 우상단 검은 '운영 중' 배지, 동일한 '바로 이동' 텍스트 링크, 동일한 회색 카드 배경으로 거의 똑같이 보인다. 서비스 간 차별점이나 우선순위가 시각적으로 드러나지 않아 선택을 돕지 못한다.
- 개선: 각 서비스에 대표 아이콘/일러스트나 약간의 색상 악센트를 부여하고, 추천/대표 서비스가 있다면 그 카드만 강조(테두리 또는 primary CTA 버튼)해 위계를 만든다.

**#4. [보통 · 어포던스/명확성] 카드 전체 vs '바로 이동' 텍스트 중 어디가 탭 영역인지 모호**

- 근거: 카드 전체가 회색 박스로 묶여 있어 카드 전체가 클릭 가능해 보이지만, 하단에 별도 '바로 이동' 텍스트 링크가 있어 탭 타겟이 카드 전체인지 그 텍스트만인지 충돌한다. 사용자가 어디를 눌러야 할지 혼란스럽다.
- 개선: 카드 전체를 탭 타겟으로 통일하고 '바로 이동'은 시각적 보조 표시로만 두거나, 반대로 카드 본문은 비액션으로 하고 명확한 버튼 하나만 액션으로 둔다.

**#5. [보통 · 색상/대비] 본문 설명 텍스트가 저대비 회색이라 가독성 저하**

- 근거: 각 카드의 설명 문구('바로 연습을 시작하고...', '덱을 만들고...', '실시간 채팅과...')가 연한 회색으로 표시되어 흰/연회색 카드 배경 대비가 낮다. 모바일 야외 환경에서 읽기 어렵고 WCAG 본문 대비(4.5:1) 미달 가능성이 있다.
- 개선: 본문 텍스트 색상을 더 진한 회색(예: #4B5563 이상)으로 올려 본문 대비비 4.5:1을 확보한다.

**#6. [보통 · 타이포그래피] 커뮤니티 카드 설명이 '제공합니/다.'로 음절 단위 줄바꿈됨**

- 근거: 세 번째 카드 설명 '실시간 채팅과 커뮤니티 글·댓글 피드를 함께 제공합니다.'에서 '제공합니'와 '다.'가 분리되어 마지막 음절만 두 번째 줄로 넘어간다. 외톨이 글자(orphan)가 생겨 마무리가 지저분하다.
- 개선: keep-all 적용 또는 문구를 짧게 다듬어(예: '실시간 채팅과 커뮤니티 피드를 제공합니다.') 음절 단절을 없앤다.

**#7. [보통 · 접근성] 헤더 설정/프로필 아이콘이 라벨 없는 단독 아이콘**

- 근거: 우상단 톱니바퀴(설정)와 사람(프로필) 아이콘이 텍스트 라벨 없이 아이콘만 있는 테두리 버튼이다. 의미를 아이콘에만 의존해 스크린리더 사용자나 아이콘 해석이 익숙하지 않은 사용자에게 불명확하다.
- 개선: 각 아이콘 버튼에 aria-label('설정', '내 계정')을 부여하고, 가능하면 탭/롱프레스 시 툴팁 또는 텍스트를 제공한다.

**#8. [보통 · 시각적 위계] 숨김 서비스 보기 버튼의 위치/역할이 모호**

- 근거: '숨김 서비스 보기' 버튼이 서브헤딩과 카드 목록 사이에 단독으로 떠 있어, 무엇을 펼치는 토글인지 목록 필터인지 맥락이 약하다. 테두리만 있는 연한 버튼이라 주요 액션보다 시선을 먼저 끌 이유가 없는데 카드 위에 배치되어 흐름을 끊는다.
- 개선: 토글 의미를 드러내는 아이콘(눈/펼침 화살표)을 추가하고, 카드 목록 하단으로 옮기거나 '숨겨진 서비스 N개 보기'처럼 개수를 노출해 맥락을 준다.

**#9. [보통 · 콘텐츠/문구] 푸터 이메일이 실서비스 도메인과 무관한 placeholder성 주소**

- 근거: 푸터 연락처가 'hiclaudepro@gmail.com'으로, yeon.world 서비스 브랜드와 무관한 개인 gmail 주소처럼 보인다. 프로덕션 화면에 남으면 신뢰도가 떨어지고 placeholder 잔재로 의심된다.
- 개선: 서비스 도메인 기반 공식 이메일(예: contact@yeon.world)로 교체하거나, 미정이라면 노출하지 않는다.

**#10. [보통 · 레이아웃/간격] 히어로 영역 상하 여백이 과도해 첫 화면에 카드가 거의 안 보임**

- 근거: 제목 위/아래, 서브헤딩~숨김버튼~첫 카드까지 빈 여백이 크게 잡혀 있어 390px 모바일 첫 화면에서 첫 카드 상단만 겨우 노출되고 실제 선택지(서비스 카드)를 보려면 스크롤이 필요하다. 핵심 콘텐츠 도달이 늦다.
- 개선: 히어로 상하 패딩과 제목~카드 사이 간격을 줄여 첫 화면(above the fold)에 최소 한 장의 서비스 카드가 온전히 보이도록 한다.

**#11. [낮음 · 레이아웃/간격] 본문 좌측 정렬과 헤더 로고 정렬은 맞으나 카드 내부 우측 여백이 과함**

- 근거: 카드 설명 텍스트가 우측 가장자리에서 일찍 줄바꿈되며 카드 우측에 비교적 큰 빈 여백이 남는다. 반면 '운영 중' 배지는 거의 가장자리에 붙어 있어 우측 여백 기준이 일관되지 않아 보인다.
- 개선: 카드 내부 좌우 패딩을 동일 기준으로 맞추고, 배지와 텍스트의 우측 정렬선을 통일한다.

**#12. [낮음 · 콘텐츠/문구] '운영 중' 배지가 모든 카드에 붙어 정보 가치가 없음**

- 근거: 3개 카드 전부 동일하게 '운영 중' 검은 배지를 달고 있어, 모두 운영 중이라면 배지가 변별 정보를 주지 못하고 시각적 노이즈만 된다. '숨김 서비스'와 대비되는 상태 표시 의도라면 그 맥락이 화면에 드러나지 않는다.
- 개선: 전부 운영 중이면 배지를 생략하거나, 출시 예정/베타/신규 등 상태가 다른 경우에만 배지를 노출해 정보성을 살린다.

**#13. [낮음 · 레이아웃/간격] 카드 사이 간격 대비 카드 하단 구분선~CTA 간격이 좁아 위계 혼란**

- 근거: 카드 내부에서 설명과 '바로 이동' 사이에 가느다란 구분선이 있는데, 구분선 위 여백과 아래 '바로 이동'까지의 간격 리듬이 카드 간 큰 여백과 어울리지 않아 CTA가 카드에 눌린 듯 보인다.
- 개선: 구분선 위아래 여백을 균등하게 키워 CTA 영역에 숨 쉴 공간을 주고, 카드 내부 수직 리듬을 일관되게 정리한다.

**#14. [낮음 · 콘텐츠/문구] 푸터 링크들이 구분자 없이 공백으로만 나열되어 경계 모호**

**#36. [보통 · 어포던스/명확성] 카드마다 '자세히 보기'와 '연습하기' 두 버튼이 경쟁적으로 강조**

**#37. [보통 · 색상/대비] 본문/설명 회색 텍스트의 대비가 낮음**

**#38. [낮음 · 일관성] 두 배지(기본/언어)의 스타일이 달라 같은 줄에서 불일치**

- 개선: 두 배지의 모양(채움 vs 외곽선)을 통일하거나, 의도적으로 위계를 둘 경우 출처/언어 중 무엇이 상위인지 명확한 규칙으로 색·굵기를 차등 적용한다.

**#39. [낮음 · 레이아웃/간격] 첫 카드만 본문이 2줄, 나머지는 3줄로 카드 높이 들쭉날쭉**

**#40. [낮음 · 타이포그래피] 한글 카드 본문이 의미 단위와 무관하게 어색하게 줄바꿈됨**

- 근거: 진달래꽃 설명 '...에서 고른 한국어 기'에서 줄이 끊겨 '기'(추정 '기사'의 일부) 한 글자가 줄 끝에 남고 다음 줄로 넘어간다. 영문 카드처럼 단어 경계로 끊기지 않고 한 단어가 두 줄로 쪼개져 가독성이 떨어진다(버블 겹침과 별개로 줄바꿈 자체가 어색).

**#41. [낮음 · 콘텐츠/문구] '4개 표시 · 전체 4개' 카운트 문구가 중복·장황**

- 근거: 세그먼트 탭 아래 '4개 표시 · 전체 4개'는 표시 수와 전체 수가 동일한 상황에서 같은 숫자를 두 번 반복해 정보 가치가 거의 없다. 필터가 적용되지 않은 상태에서는 군더더기다.
- 개선: 표시 수와 전체 수가 같을 때는 '총 4개' 한 가지로 간결화하고, 필터로 일부만 보일 때만 '4개 중 N개 표시' 형태로 노출한다.

**#42. [낮음 · 어포던스/명확성] '기본 덱/내 덱/공개 덱' 세그먼트 탭의 비활성 탭 어포던스 약함**

- 개선: 세그먼트 컨트롤 전체에 옅은 배경 트랙을 깔고 비활성 탭에도 미세한 경계나 hover 상태를 주어 3개가 모두 선택 가능한 탭임을 드러낸다.

**#43. [낮음 · 접근성] 헤더 우상단 아이콘(설정/프로필)이 라벨 없는 단독 아이콘**

- 개선: word-break: keep-all 적용 및 본문 폭/폰트 사이즈 조정으로 '공간입니다.'가 한 덩어리로 유지되도록 한다.

**#54. [보통 · 어포던스/명확성] 입장 캐릭터 카드의 '캐릭터 바꾸기'가 링크인지 버튼인지 모호**

- 근거: '캐릭터 바꾸기'에 밑줄만 있어 텍스트 링크처럼 보이는데, 회색 카드 안에 있어 카드 전체를 누르는 것인지 텍스트만 누르는 것인지 어포던스가 불명확하다. 탭 타겟 크기도 텍스트 높이 수준으로 작아 보인다.
- 개선: '캐릭터 바꾸기'를 명확한 버튼 또는 chevron 아이콘이 붙은 행 형태로 만들어 탭 가능 영역을 44px 이상으로 키우고 클릭 가능성을 시각적으로 드러낸다.

**#55. [보통 · 빈 상태] 빈 상태인데도 방 검색창과 필터탭이 그대로 노출돼 불필요**

- 근거: 방이 0개인 빈 상태에서 상단에 '전체/공개방/입장 가능' 필터탭과 '방 검색' 입력창이 모두 활성화된 채 그대로 보인다. 검색할 대상이 없는데 검색창을 제공해 사용자 혼란과 헛수고를 유발한다.
- 개선: 방이 없을 때는 필터/검색을 비활성화(dimmed)하거나 숨기고, 첫 방 생성 CTA에 집중하도록 빈 상태를 단순화한다.

**#56. [보통 · 색상/대비] '방 검색' 플레이스홀더가 저대비 연회색이라 접근성 미달 우려**

- 근거: 검색 입력창의 '방 검색' 플레이스홀더 텍스트와 돋보기 아이콘이 옅은 회색으로, 흰 배경 대비 WCAG AA(4.5:1)에 미치지 못해 보일 정도로 흐리다. 저시력 사용자 가독성이 떨어진다.
- 개선: 플레이스홀더 색을 #6B7280 수준 이상으로 어둡게 조정해 본문 대비 4.5:1을 충족하고, 입력 필드임을 명확히 한다.

**#57. [보통 · 모바일 사용성] 필터탭 사이 간격은 넓으나 탭 자체 높이/세로 패딩이 작아 보임**

- 근거: '전체/공개방/입장 가능' pill 버튼들이 가로로는 충분하지만 세로 높이가 다소 낮아 보여, 390px 모바일에서 엄지로 누르기엔 탭 타겟 44px 권장 높이에 미달할 가능성이 있다.
- 개선: 필터 pill의 세로 패딩을 늘려 최소 44px 높이를 확보하고, 비활성 탭의 outline 대비도 약간 강화한다.

**#58. [보통 · 접근성] 헤더 우상단 설정/프로필 아이콘이 라벨 없이 단독 노출**

- 근거: 우상단 톱니(설정), 사람(프로필) 아이콘이 텍스트 라벨 없이 아이콘만 있어 의미 전달이 아이콘 인지에만 의존한다. 스크린리더 aria-label 누락 시 접근성 문제가 되고, 두 버튼이 거의 붙어 있어 오탭 위험도 있다.
- 개선: 각 아이콘 버튼에 aria-label을 부여하고, 두 버튼 사이 간격을 늘려 오탭을 방지한다. 탭 영역 44px 확보.

**#59. [낮음 · 시각적 위계] 빈 상태 카드의 외곽선이 거의 안 보일 만큼 흐려 영역 구분이 약함**

- 근거: 빈 상태를 감싸는 큰 카드의 테두리가 매우 옅은 회색이라 흰 배경과 거의 구분되지 않는다. 카드 영역이 어디까지인지 모호하고 콘텐츠가 허공에 떠 있는 인상을 준다.
- 개선: 카드 테두리 대비를 약간 높이거나 배경에 미세한 그레이 채움/그림자를 적용해 영역을 명확히 구분한다.

**#60. [낮음 · 레이아웃/간격] 본문 영역 좌우 여백 대비 빈 상태 카드 높이가 과도하게 길어 빈 공간 과다**

- 근거: 빈 상태 카드가 화면 하단까지 매우 길게 늘어나 일러스트/텍스트/버튼 아래로 빈 공간이 크게 남는다. 콘텐츠 밀도가 낮아 휑한 인상을 준다.
- 개선: 빈 상태 카드 높이를 콘텐츠에 맞게 줄이고(또는 콘텐츠를 카드 세로 중앙 정렬), 하단 여백을 절제해 균형을 잡는다.

**#61. [낮음 · 일관성] '전체' 활성 탭만 검정 채움이라 다른 탭과 위계 대비가 과함**

- 근거: 선택된 '전체' 탭은 검정 배경 흰 글씨로 강조가 강한 반면, 비선택 탭은 흰 배경에 옅은 회색 테두리로 매우 약하다. 검정 채움 강조가 하단 CTA 버튼들과 같은 색이라 '선택 상태'와 '주 버튼'의 시각 신호가 충돌한다.
- 개선: 선택 탭 강조를 채움보다 약한 방식(밑줄 또는 옅은 회색 채움)으로 낮춰, 페이지 내 검정 채움은 주 CTA에만 쓰도록 위계를 정리한다.

**#62. [낮음 · 콘텐츠/문구] 헤더 로고 'YEON 타자연습'과 페이지 제목 '타자방' 표현 불일치**

- 근거: 상단 로고는 '타자연습', 페이지 제목은 '타자방'으로 서비스 명칭 표기가 섞여 있다. 동일 서비스 내 용어 일관성이 약해 보인다.
- 개선: 서비스/기능 명칭을 통일하거나(예: '타자 연습 - 타자방'), 제목과 로고의 관계를 브레드크럼/서브타이틀로 명확히 한다.

**#63. [낮음 · 콘텐츠/문구] 'Guest · 낙타' 캐릭터 이름과 일러스트(기린류)가 불일치해 보임**

- 근거: 입장 캐릭터 라벨은 '낙타'인데 카드 썸네일의 픽셀 일러스트는 목이 길어 기린에 가까워 보여, 이름과 이미지가 어긋난 인상을 준다. 사용자가 자기 캐릭터 정체성을 혼동할 수 있다.
- 개선: 캐릭터 명칭과 일러스트 에셋의 매핑을 검수해 일치시키고, 의도된 디자인이라면 이름을 이미지에 맞춰 조정한다.

**#64. [낮음 · 콘텐츠/문구] 빈 상태 안내 문구가 제목과 의미상 중복되어 정보량이 약함**

- 근거: '아직 열린 타자방이 없어요' 제목 아래 '첫 방을 만들어 친구와 함께 시작해보세요.' 보조 문구가 제목 내용을 거의 반복한다. 새 정보(예: 방 생성 소요 시간, 비공개/공개 차이 안내) 없이 같은 메시지를 두 번 전달한다.
- 개선: 보조 문구에 실질적 가이드(공개/비공개 선택 가능, 친구 초대 링크 제공 등)를 담아 중복을 줄이고 행동 동기를 강화한다.

### 5. 개인 연습 `/typing-service/practice` — 15건

**#65. [높음 · 어포던스/명확성] 헤더 우측 두 아이콘 버튼이 라벨 없는 회색 원으로 의미 불명**

- 근거: 상단 헤더 우측에 회색 원형 아이콘 버튼 2개(톱니바퀴로 보이는 것, 사람/계정으로 보이는 것)가 텍스트 라벨 없이 나란히 놓여 있다. 두 버튼이 무엇을 하는지 아이콘만으로 구분이 어렵고, 크기도 작아 기능을 추측해야 한다.
- 개선: 각 아이콘에 aria-label과 가능하면 짧은 텍스트 라벨/툴팁을 추가하고, 설정과 계정 등 기능을 시각적으로 명확히 구분한다.

**#66. [높음 · 빈 상태] 하단에 입력 영역으로 보이는 큰 빈 회색 박스가 비어 있고 안내가 부족**

- 근거: 화면 맨 아래에 'AI 시작...'(일부만 보임)으로 시작하는 옅은 회색 텍스트가 담긴 넓은 빈 박스가 있다. 입력창인지 안내 영역인지 불명확하고, 카운트다운 10 상태에서 사용자가 무엇을 해야 하는지(곧 시작됨, 입력 대기 등) 명확한 안내가 없다.
- 개선: 입력 영역임을 알 수 있게 테두리/포커스 상태를 명확히 하고, '곧 시작됩니다' 또는 '여기에 입력하세요' 같은 명시적 플레이스홀더/안내 문구를 넣는다.

**#67. [보통 · 빈 상태] 카운트다운 '10' 외에 시작 흐름 안내가 없어 빈 상태가 불친절**

- 근거: 레이스 트랙 중앙에 큰 '10' 숫자만 떠 있고 '초 후 시작' 같은 단위/맥락 텍스트가 없다. 처음 보는 사용자는 10이 카운트다운인지 점수인지 즉시 알기 어렵다.
- 개선: 숫자 아래에 '초 후 시작' 같은 보조 텍스트를 추가하거나 '곧 시작! 10' 형태로 맥락을 함께 제시한다.

**#68. [보통 · 일관성] 지표 라벨 '타수/acc/progress/time'의 언어가 혼용됨**

- 근거: 지표 행에서 '타' 또는 '타수'는 한글, 'acc'·'progress'·'time'은 영어 약어로 표기되어 있어 한 줄 안에서 한/영이 섞여 있다. 한국어 앱의 일관성이 깨진다.
- 개선: '타수 / 정확도 / 진행도 / 시간'처럼 모두 한국어로 통일하거나, 정 영어를 쓰려면 전부 영어로 통일한다.

**#69. [보통 · 색상/대비] 지표 행 텍스트가 작고 저대비 회색이라 가독성이 낮음**

- 근거: '타수 0', 'acc 100%', 'progress 0%', 'time 0.0s' 지표 행에서 라벨이 매우 작은 옅은 회색으로 표시되어 흰 배경에서 대비가 부족하다. 모바일에서 핵심 지표인데도 눈에 잘 들어오지 않는다.
- 개선: 라벨 글자색을 더 진한 회색/검정으로 올리고 폰트 크기를 키워 WCAG 대비 기준을 충족시킨다.

**#70. [보통 · 레이아웃/간격] 4개 지표가 좁은 390px 폭에 균등 배분되지 않고 빽빽함**

- 근거: 한 줄에 '타수 / acc / progress / time' 4개 지표를 모두 넣어 항목 간 간격이 좁고 구분선/여백이 부족해 어디가 한 묶음인지 시각적으로 분리되지 않는다.
- 개선: 항목 사이 구분선이나 충분한 간격을 주거나, 2x2 그리드로 배치해 모바일에서 각 지표를 명확히 묶어 보여준다.

**#71. [보통 · 타이포그래피] 본문(타이핑할 문장) 텍스트의 줄바꿈이 어색하고 단어가 어절 중간에서 끊김**

- 근거: 하단 안내 문장에서 '손/가락보다'처럼 단어가 줄 끝에서 부자연스럽게 끊겨 다음 줄로 넘어가고, 행간도 좁아 한글 두 줄 이상이 빽빽하게 보인다.
- 개선: word-break를 keep-all로 설정해 어절 단위로 줄바꿈하고 행간(line-height)을 1.6 이상으로 늘려 가독성을 확보한다.

**#72. [보통 · 레이아웃/간격] 레이스 트랙 영역이 화면 세로의 절반 이상을 차지해 입력/지표 공간을 압박**

- 근거: 상단 라마 레이스 일러스트 카드가 화면의 절반 가까이를 차지하는 반면, 실제 타이핑 본문과 입력 영역은 하단 좁은 공간에 몰려 있다. 연습의 핵심인 입력보다 장식적 레이스가 더 큰 비중을 차지한다.
- 개선: 카운트다운/연습 중에는 트랙 높이를 줄여 본문과 입력 영역에 더 많은 세로 공간을 배분한다.

**#73. [보통 · 색상/대비] 플레이어/상대 라벨 'You/Guest'가 영어이고 트랙 위 텍스트 대비가 낮음**

- 근거: 레이스 트랙 각 레인 좌측에 'You', 'Guest', 'Guest', 'Guest' 라벨이 작은 흰/연한 글씨로 표시되어 흙색 배경 위에서 대비가 약하고, 한국어 앱인데 영어 라벨이다. 또한 1인 연습인데 Guest가 3개나 채워져 의미가 모호하다.
- 개선: 라벨을 '나/상대'처럼 한국어로 바꾸고, 배경 대비를 위해 외곽선/그림자를 넣는다. 개인 연습 화면에서 더미 Guest 3명의 필요성을 재검토한다.

**#74. [낮음 · 어포던스/명확성] 좌측 하단 말풍선(채팅) 아이콘 버튼의 용도/위치가 모호**

- 근거: 본문 영역 좌측에 회색 원형 말풍선 아이콘이 떠 있는데, 라벨이 없고 본문 텍스트 시작 위치와 겹쳐 보일 만큼 가까워 무엇을 위한 버튼인지(채팅? 도움말?) 알기 어렵다.
- 개선: 아이콘에 라벨/툴팁을 추가하고 본문 텍스트와 충분한 간격을 두며, 개인 연습에서 채팅이 불필요하면 노출 조건을 재검토한다.

**#75. [낮음 · 레이아웃/간격] 레이스 트랙 카드와 본문 사이 간격은 넓은 반면 지표 행은 붙어 있어 간격 리듬이 불균형**

- 근거: 트랙 카드 아래 지표 행, 그 아래 본문 박스 사이의 수직 여백이 일관되지 않다. 트랙-지표 사이는 좁고 지표-본문 사이는 상대적으로 떠 보여 섹션 구분 리듬이 고르지 않다.
- 개선: 섹션 간 수직 간격을 일정한 spacing 토큰(예: 16/24px)으로 통일해 시각적 리듬을 맞춘다.

**#76. [낮음 · 시각적 위계] 본문 첫 글자 '랩'이 사각형 박스로 강조되어 의미가 불분명**

**#113. [높음 · 콘텐츠/문구] '방 만들기'와 '카드방 만들기' 두 개의 검정 CTA가 한 화면에 중복**

**#131. [보통 · 시각적 위계] 게시글 카드의 제목과 본문이 동일 내용 중복으로 위계가 무너짐**

**#134. [보통 · 타이포그래피] 하단 '게시판 종류' 설명 문단이 장황하고 줄 간격이 빽빽**

**#135. [보통 · 콘텐츠/문구] 하단 '최근 글' 목록이 위 게시글 목록과 거의 동일해 중복 정보**

**#136. [보통 · 색상/대비] 본문 텍스트의 회색 대비가 낮아 가독성 우려**

**#137. [보통 · 레이아웃/간격] 채팅 피드와 글쓰기 폼/게시글 목록 간 섹션 구분이 약함**

**#138. [낮음 · 빈 상태] '댓글 0' 빈 카운트가 강조 없이 반복돼 빈 상태 안내 부재**

**#139. [낮음 · 정렬] 카드별 '잡담' 배지 위치가 작성자/날짜 줄에 끼어 정렬이 불안정**

- 개선: 카테고리 배지를 작성자 줄 우측 끝에 고정 정렬하거나 제목 위 별도 줄로 분리해 카드 간 위치를 일관되게 한다.

**#140. [낮음 · 어포던스/명확성] '실시간 채팅' 헤더의 초록 점+숫자 '2'의 의미가 라벨 없이 불명확**

**#141. [낮음 · 일관성] 게시글 카드와 하단 '최근 글' 카드의 스타일이 불일치**

### 10. 교차 — 화면 간 일관성/디자인 시스템 — 16건

**#143. [높음 · 일관성] 헤더 로고 표기가 화면마다 제각각 — 'YEON' / 'YEON 타자연습' / 'YEON 카드' / 'YEON 커뮤니티' 혼용**

- 근거: 홈(mobile-01)은 헤더 로고가 'YEON' 단독, 타자방(mobile-04)은 'YEON 타자연습', 카드방(mobile-06/08)은 'YEON 카드', 커뮤니티(mobile-09)는 'YEON 커뮤니티'로 표기된다. 같은 'YEON' 워드마크인데 어떤 화면은 서비스명을 붙이고(타자연습/카드/커뮤니티), 어떤 화면은 붙이지 않는다. 또 카드 서비스는 정식명이 '플래시카드 덱'(홈 카드)인데 헤더에서는 '카드'로 축약되어 동일 서비스의 명칭조차 두 가지다. 사용자가 자기가 어느 서비스에 있는지, 브랜드명이 무엇인지 혼란스럽다.
- 개선: 헤더 로고는 'YEON' 워드마크 하나로 고정하고, 서비스명은 별도의 페이지 타이틀(예: '타자방', '카드방')로 위계를 분리하거나, 로고 옆 서브라벨을 쓸 거면 모든 서비스(홈 포함)에 동일 패턴으로 적용한다. 'YEON 카드'와 '플래시카드 덱'처럼 한 서비스에 두 이름이 생기지 않도록 명칭 사전을 단일화한다.

**#144. [높음 · 일관성] '방 만들기' 버튼이 타자방은 떠다니는 작은 버튼, 카드방은 풀폭 막대 버튼으로 형태가 완전히 다름**

- 근거: 동일 기능('방 만들기')이 타자방(mobile-04)에서는 빈 상태 일러스트 위에 둥근 모서리의 작고 떠 있는 검은 버튼으로 콘텐츠 위에 겹쳐 떠 있고, 카드방(mobile-08)에서는 검색창 바로 아래 좌우 전체폭을 채우는 직사각형 막대 버튼으로 배치된다. 위치·크기·라운드·앵커링이 모두 달라 같은 서비스 패밀리로 보이지 않고, 타자방에서는 버튼이 일러스트 위에 떠서 시각적으로 어색하게 겹친다.
- 개선: 목록 화면의 1차 액션('방 만들기')은 두 서비스에서 동일한 위치(검색 바로 아래 풀폭 또는 동일 플로팅 규칙)·동일 크기·동일 라운드로 통일한다. 타자방의 콘텐츠 위에 겹쳐 떠 있는 형태는 제거한다.

**#145. [높음 · 시각적 완성도] 버튼 라운드(border-radius) 토큰 혼용 — pill vs rounded vs 직각**

- 근거: 타자방(mobile-04) 필터 칩('전체/공개방/입장 가능')은 약 8px 정도의 약한 라운드 사각형인데, 카드방(mobile-08)의 동일 필터 칩은 거의 완전한 pill(타원형) 형태다. 또 카드방의 '카드방 만들기' CTA(mobile-08)는 라운드가 거의 없는 직각 사각형인 반면, 타자방 '첫 방 만들기'(mobile-04)와 홈의 '숨김 서비스 보기'(mobile-01)는 둥근 모서리다. 같은 컴포넌트 종류(필터 칩, 1차 CTA)가 화면마다 radius가 달라 라운드 토큰이 통일되지 않았다.
- 개선: radius 토큰을 컴포넌트별로 1개씩 고정한다(예: 필터 칩 = pill, 1차 버튼 = 12px, 입력창 = 12px). 카드방의 직각 CTA와 직각 '방 만들기' 막대도 동일 radius로 맞춘다.

**#146. [높음 · 모바일 사용성] 좌하단/우하단 떠다니는 채팅 버튼이 빈 상태 제목·CTA와 겹침**

- 근거: 타자 서비스 소개(mobile-02)에서는 플로팅 채팅 아이콘이 본문 위 '...의 시작' 텍스트와 겹쳐 글자를 가리고, 카드 서비스 소개(mobile-06)에서도 같은 채팅 아이콘이 '시작' 텍스트와 겹친다. 타자방(mobile-04)에서는 좌측 채팅 버튼이 빈 상태 제목 '아직 열린 타자방이 없어요' 왼쪽에 붙어 글자에 근접하고, 카드방(mobile-08)에서는 채팅 버튼이 빈 상태 제목 '열린 카드방이 없어요'의 첫 글자 바로 옆에 겹쳐 제목 가독성을 해친다. 게다가 채팅 버튼 위치가 화면마다(소개=본문 중앙 좌측, 목록=빈 상태 좌측) 들쭉날쭉하다.
- 개선: 플로팅 채팅 버튼은 화면 우하단(또는 좌하단) 고정 위치 하나로 통일하고, 콘텐츠와 겹치지 않도록 safe-area/하단 여백을 확보한다. 빈 상태 영역 위에 떠서 텍스트를 가리지 않게 z-index와 위치를 재조정한다.

**#147. [보통 · 일관성] 우상단 헤더 아이콘(설정/계정)의 크기·배경이 화면마다 다름**

- 근거: 홈(mobile-01), 타자방(mobile-04), 카드방(mobile-08)에서는 우상단 설정(톱니)·계정 아이콘이 연한 회색 테두리 박스 안에 큼직하게(약 44px) 들어가 있는 반면, 타자 서비스 소개(mobile-02)·카드 서비스 소개(mobile-06)·커뮤니티(mobile-09)에서는 같은 두 아이콘이 테두리 없이 작게 붙어 있다. 동일한 글로벌 헤더 액션인데 어떤 화면은 버튼형, 어떤 화면은 플레인 아이콘형으로 보여 일관성과 탭 타겟이 흔들린다.
- 개선: 헤더 우상단 아이콘 버튼은 단일 컴포넌트(동일 크기·동일 테두리/배경·최소 44x44 탭 타겟)로 통일한다. 소개 화면들에서도 같은 버튼 스타일을 쓴다.

**#148. [보통 · 빈 상태] 빈 상태 패턴 불일치 — 일러스트 유무, 제목 줄바꿈, CTA 위치가 타자방/카드방 간 다름**

- 근거: 타자방 빈 상태(mobile-04)는 키보드 일러스트 + 제목('아직 열린 타자방이 없어요') + 보조문구 + 'first 방 만들기' CTA 구조이고, 카드방 빈 상태(mobile-08)는 일러스트가 없이 제목('열린 카드방이 없어요') + 보조문구 + '카드방 만들기' CTA만 있다. 동일한 '방 목록 비어있음' 상태인데 한쪽은 일러스트가 있고 한쪽은 없으며, 제목 문구도 '아직 열린 ~이 없어요' vs '열린 ~이 없어요'로 미묘하게 다르다. 빈 상태 컴포넌트가 통일돼 있지 않다.
- 개선: '방 없음' 빈 상태를 공통 컴포넌트로 만들어 일러스트 유무·제목 문구 패턴('아직 열린 OO방이 없어요')·보조문구·CTA 라벨/위치를 두 서비스에서 동일하게 맞춘다.

**#149. [보통 · 색상/대비] 페이지 배경색이 화면마다 흰색/연회색으로 혼용**

- 근거: 타자방(mobile-04)·카드방(mobile-08)·홈(mobile-01)은 순수 흰색(#fff) 배경 위에 연회색 카드를 얹는 구조인 반면, 타자 서비스 소개(mobile-02)·카드 서비스 소개(mobile-06)는 페이지 전체가 미묘한 연회색 톤이고 카드가 그 위에 더 옅은 회색으로 올라간다. 커뮤니티(mobile-09)는 또 흰색 카드들이 회색 분위기 배경에 놓여 다르게 보인다. 페이지 base 배경과 surface 배경의 토큰이 통일되지 않아 화면 전환 시 배경 밝기가 미세하게 점프한다.
- 개선: page 배경(예: #fff)과 surface/card 배경(예: #f5f5f5) 두 단계 토큰을 정하고 모든 화면이 동일하게 따르도록 한다. 소개 화면의 회색 base 배경을 목록 화면과 통일한다.

**#150. [보통 · 색상/대비] 카드/서피스 배경 회색 토큰 혼용 (#f0f0f0 계열 vs #f5f5f5 계열)과 보더 색 혼용**

- 근거: 홈(mobile-01)의 서비스 카드는 연회색 배경에 보더가 거의 보이지 않는 fill형이고, 타자방/카드방(mobile-04/08)의 프로필 카드·방 목록 컨테이너는 흰색 배경 + 연회색 1px 보더(outline형)다. 커뮤니티(mobile-09)의 글 카드는 또 다른 흰색 + 옅은 보더 조합이다. 같은 '카드' 컴포넌트인데 어떤 곳은 채움 회색, 어떤 곳은 테두리만 있는 흰 카드로 표면 처리가 일관되지 않고 보더 회색값도 미세하게 달라 보인다(#e5e5e5 vs #e5e7eb급 차이).
- 개선: 카드 컴포넌트를 fill형/outline형 중 하나로 정의하고 surface 색·border 색 토큰을 단일화한다(#e5e7eb 등 하나로 통일). 홈의 채움형 카드와 목록의 테두리형 카드 중 시스템 기준 하나로 수렴시킨다.

**#151. [보통 · 타이포그래피] 페이지 타이틀 타이포 스케일 불일치 — 홈은 거대한 멀티라인, 소개 화면은 작은 한 줄**

- 근거: 홈(mobile-01)의 메인 타이틀 '현재 3가지 서비스를 운영 중입니다.'와 타자방/카드방(mobile-04/08)의 '타자방'/'카드방'은 매우 큰 디스플레이 사이즈(40px+)인데, 타자 서비스 소개(mobile-02)·카드 서비스 소개(mobile-06)의 상단 타이틀 '바로 시작하는 타자 연습'/'바로 시작하는 카드공부'는 훨씬 작은 본문급 크기다. 동일 위계(페이지 최상단 제목)인데 화면 그룹마다 타이포 스케일 단계가 달라 H1 정의가 흔들린다.
- 개선: 페이지 H1 스케일을 1개로 정의하고(또는 '소개형'과 '히어로형' 두 종류만 명시적으로 구분) 소개 화면과 목록 화면의 최상단 제목 크기 위계를 통일한다.

**#152. [보통 · 타이포그래피] 큰 타이틀이 모바일 폭에서 어색하게 두 줄로 깨짐**

- 근거: 홈(mobile-01)의 '현재 3가지 서비스를 운영 중입니다.'가 '...운영 중입' / '니다.'로 끊겨 마지막 줄에 '니다.'만 남는다. 타자방(mobile-04)의 빈 상태 제목 '아직 열린 타자방이 없어' / '요'도 '요' 한 글자만 둘째 줄에 떨어진다. 카드방(mobile-08)의 보조문구 '...첫 카드방을' / '만들어 보세요.'도 어색하게 끊긴다. 큰 타이포에서 어절 단위 줄바꿈 제어가 안 되어 한 글자/한 어절만 다음 줄로 떨어지는 고아(orphan) 현상이 여러 화면에 공통으로 발생.
- 개선: 큰 제목·CTA 문구에 word-break/keep-all 및 적절한 max-width, 또는 줄바꿈 제어(<wbr>, &nbsp;, balance)를 적용해 한 글자/한 어절만 둘째 줄로 떨어지지 않게 한다. 시스템 차원의 한글 타이포 줄바꿈 규칙(word-break: keep-all 기본)을 정한다.

**#153. [보통 · 어포던스/명확성] '바로 이동' 링크가 버튼인지 텍스트 링크인지 어포던스 불명확 (홈 카드)**

- 근거: 홈(mobile-01)의 세 서비스 카드 하단 '바로 이동'은 굵은 검은 텍스트에 위쪽 구분선만 있을 뿐 버튼 배경·테두리·화살표 같은 클릭 어포던스가 없어 단순 라벨인지 액션인지 모호하다. 반면 타자방/카드방의 1차 액션은 명확한 검은 채움 버튼이다. 같은 '서비스 진입' 액션인데 한쪽은 버튼, 한쪽은 밋밋한 텍스트라 클릭 가능성 신호가 일관되지 않다.
- 개선: 카드 진입 액션을 화살표 아이콘(→)을 붙이거나 버튼 스타일로 바꿔 클릭 가능함을 명확히 하고, 카드 전체를 탭 타겟으로 만들거나 시스템의 '카드 내 액션' 패턴을 통일한다.

**#154. [보통 · 레이아웃/간격] 섹션/요소 간 수직 간격 스케일이 화면마다 불규칙**

- 근거: 타자방(mobile-04)·카드방(mobile-08)은 프로필 카드 아래에 큰 여백 + 가로 구분선으로 섹션을 크게 띄우는데, 홈(mobile-01)은 카드 사이 간격이 비교적 균일하고 소개 화면(mobile-02/06)은 섹션 간격이 더 촘촘하다. 카드방(mobile-08)에서는 검색창과 '방 만들기' 막대 버튼 사이 간격이 다른 요소 간격보다 타이트해 보인다. 8/16/24 같은 일관된 spacing scale이 적용되지 않아 화면 간 리듬이 다르다.
- 개선: 4 또는 8 배수 기반 spacing 토큰(8/16/24/32)으로 섹션·요소 간 수직 간격을 표준화하고, 모든 화면이 동일 간격 스케일을 따르게 한다. 특히 카드방 검색창-버튼 간 여백을 다른 간격과 맞춘다.

**#155. [낮음 · 시각적 위계] 본문/보조 텍스트 회색 톤이 화면마다 달라 위계가 불안정**

- 근거: 홈(mobile-01)의 카드 본문 설명은 비교적 진한 회색, 타자/카드 소개(mobile-02/06)의 FAQ 본문과 '주요 기능' 설명은 더 옅은 회색이며, 커뮤니티(mobile-09)의 글 본문/메타 텍스트는 또 다른 톤이다. secondary/tertiary 텍스트 색 토큰이 통일되지 않아 같은 '보조 설명' 역할의 글자가 화면마다 대비가 달라진다. 일부 옅은 회색은 모바일에서 저대비로 읽기 부담이 있다.
- 개선: 텍스트 색을 primary/secondary/tertiary 3단계 토큰으로 고정하고 모든 화면이 동일 값을 쓰게 한다. 본문 보조 텍스트는 WCAG AA(4.5:1) 대비를 만족하는 회색으로 통일한다.

**#156. [낮음 · 일관성] '운영 중' 상태 배지가 화면마다 다른 형태로 표현됨 (홈 pill vs 커뮤니티 점/숫자)**

- 근거: 홈(mobile-01)의 서비스 카드는 우상단에 검은 pill 배지 '운영 중'으로 상태를 표시하는데, 커뮤니티(mobile-09)의 '실시간 채팅' 헤더에서는 초록 점 + 숫자(2)로 온라인/접속 상태를 표시한다. 또 커뮤니티 글 카드의 '잡담' 카테고리 태그는 연한 파란 배지로 또 다른 배지 스타일이다. 상태/카테고리 배지의 색·형태 시스템이 통일되지 않아 같은 '배지'류가 제각각으로 보인다.
- 개선: 배지를 용도별(상태 배지, 카테고리 태그, 카운트 인디케이터)로 분류해 각 1개 스타일로 정의하고 색 의미(검정=운영상태, 파랑=카테고리, 초록=실시간 등)를 시스템 규칙으로 명시한다.

**#157. [낮음 · 모바일 사용성] 필터 칩 그룹의 탭 타겟 높이·정렬이 타자방/카드방에서 미묘하게 다름**

- 근거: 타자방(mobile-04)과 카드방(mobile-08)의 '전체/공개방/입장 가능' 필터 칩 그룹이 라운드뿐 아니라 높이·내부 패딩도 달라 보인다. 카드방 쪽 pill 칩이 더 둥글고 약간 더 큰 느낌이며, 두 화면 모두 활성('전체')은 검정 채움, 비활성은 흰 배경 테두리지만 칩 간 간격과 폭이 일정치 않다. 390px 폭에서 칩이 가로폭을 빠듯하게 채워 탭 타겟 여유가 화면마다 다르다.
- 개선: 필터 칩을 단일 컴포넌트로 통일(동일 높이≥40px, 동일 패딩, 동일 radius)하고 칩 간 gap 토큰을 고정한다. 두 서비스 목록 화면에서 동일하게 렌더되도록 한다.

**#158. [낮음 · 콘텐츠/문구] 커뮤니티 글 카드의 본문이 플레이스홀더성 텍스트로 채워져 빈/미완성 상태처럼 보임**

- 근거: 커뮤니티(mobile-09)의 글 목록 카드들이 'asdsda', 'sa', '누구세용', 'ㅁㄴㅇㄹㅁㄴㅇㄹ' 등 의미 없는 더미 텍스트로 채워져 있고, '최근 글' 섹션에도 같은 더미가 반복된다. 다른 화면들(홈/소개)은 정돈된 안내 문구인 데 반해 커뮤니티만 미완성/테스트 데이터처럼 보여 전체 완성도 인상을 떨어뜨린다. 또 카테고리 배지 '잡담'이 모든 글에 동일하게 붙어 다양성도 없어 보인다.
- 개선: 실제 데이터 미존재 시에는 더미 대신 빈 상태 안내(다른 화면과 동일한 빈 상태 패턴)를 노출하고, 데모/스크린샷용으로는 자연스러운 예시 콘텐츠를 채운다. 카테고리 태그 색·라벨도 시스템 배지 규칙에 맞춘다.
