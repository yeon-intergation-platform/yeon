# 추억의 플래시 게임 — 원작자 허락받기 가이드

> ⚠️ 이 문서는 법률 자문이 아니라 실무 참고용입니다. 큰 금액·분쟁 소지가 있으면 변호사 확인을 권합니다.
> 목적: yeon 게임 허브(game.yeon.world)에 옛 한국 플래시 게임을 **합법적으로** 넣기 위해 원작자에게 임베드/호스팅 허락을 받는 절차.

## 한 줄 요약

**① 원작자 찾기 → ② 연락처 찾기 → ③ 정중히 메일/DM 보내기 → ④ 답장(허락) 캡처·보관 → ⑤ SWF 파일 받기 → 우리가 Ruffle로 합법 실행.**
회사 IP(예: 아기고래 쿠아 = 한게임/NHN)는 개인이 허락 못 주니 **보류**하고, 개인 제작 게임부터 시작.

---

## 1단계 — 원작자(저작권자) 찾기

게임마다 만든 사람이 다릅니다. 아래 순서로 찾으세요.

1. **게임 화면 안 크레딧 확인**: 시작 화면/엔딩에 제작자 닉네임·이메일·홈페이지가 적혀 있는 경우가 많습니다.
2. **나무위키 게임 문서**: `나무위키 {게임이름}`으로 검색 → "제작자/제작" 항목에 닉네임이 있음. (예: 검 강화하기 = **NBS**)
3. **닉네임으로 추적**: 그 닉네임을 구글에 검색 → 블로그/유튜브/트위터/포트폴리오가 나오면 현재 활동 채널.
4. **아카이브 사이트에 문의**: 플래시아크·플래시모아·와플래시 같은 보존 사이트에 "이 게임 원작자 연락처 아시나요?" 문의.

> 💡 회사가 만든 게임(한게임·넥슨 등 포털 플래시)은 개인이 허락을 못 줍니다. 이런 건 회사 고객센터로 정식 문의해야 해서 난이도가 높습니다 → **개인 제작 게임부터 진행하세요.**

---

## 2단계 — 연락 채널 정하기

찾은 원작자의 채널 중 **지금도 살아있는 것** 하나를 고릅니다(우선순위):
1. 이메일 (가장 정중·공식)
2. 블로그 방명록/댓글
3. 유튜브/인스타/트위터 DM
4. 그 시절 커뮤니티 쪽지

---

## 3단계 — 허락 요청 메시지 (복붙용 템플릿)

아래를 그대로 복사해서 `{ }` 부분만 바꿔 보내세요. **수익(광고)이 붙는 사이트라는 사실을 반드시 정직하게 밝혀야** 나중에 문제가 없습니다.

```
안녕하세요, {원작자 닉네임}님.

저는 추억의 웹게임을 한곳에 모으는 사이트 'YEON 게임'(https://game.yeon.world)을 운영하는 {본인 이름}입니다.

{원작자}님이 만드신 플래시 게임 「{게임 이름}」을 정말 재미있게 했던 기억이 있어,
이 게임을 보존하고 더 많은 사람이 즐길 수 있도록 저희 사이트에 올리고 싶어 연락드립니다.

- 게임은 Ruffle(오픈소스 Flash 실행기)로 원작 그대로 구동하며, 내용을 수정하지 않습니다.
- 게임 화면과 상세 페이지에 "제작: {원작자}"로 출처를 분명히 표기하겠습니다.
- 솔직히 말씀드리면, 저희 사이트에는 광고가 붙어 약간의 수익이 발생할 수 있습니다.
  원하시면 수익 일부 배분이나 후원 링크 노출 등 조건도 협의하고 싶습니다.
- 원작자님이 원하시면 언제든 내려드리겠습니다.

혹시 「{게임 이름}」을 저희 사이트에 올려도 괜찮을지,
그리고 가능하다면 게임 SWF 파일을 공유해주실 수 있을지 여쭙고 싶습니다.

부담 없이 안 된다고 하셔도 괜찮습니다. 답장 주시면 감사하겠습니다.
{본인 이름} 드림 / 연락처: {이메일}
```

---

## 4단계 — 허락 답장 받으면 (꼭 보관!)

- 답장(이메일/DM)을 **캡처해서 보관**하세요. 이게 "허락받았다"는 증거입니다.
- 답장에서 아래 4가지를 분명히 해두면 분쟁이 안 생깁니다:
  1. **임베드/호스팅 허락 범위**: 우리 사이트에 올려도 되나? (Ruffle로 실행 포함)
  2. **수익화 OK 여부**: 광고 붙어도 되나?
  3. **출처 표기 조건**: 어떻게 표기할지
  4. **무료 / 유료(라이선스비) / 수익배분** 중 무엇인지
- 애매하면 한 줄로 다시 확인: *"정리하면, 출처 표기 조건으로 game.yeon.world에 광고 포함해 올려도 괜찮다는 말씀이 맞을까요?"*

---

## 5단계 — 게임 파일(SWF) 받기

- 원작자가 **SWF 원본 파일**을 주는 게 가장 좋습니다(가장 깨끗).
- 원작자가 파일이 없으면, "아카이브 사이트에 보존된 버전을 써도 될까요?"를 허락 메일에 같이 적어두면 됩니다.
- 받은 SWF는 yeon 쪽에 보관 → 저(개발)가 **Ruffle 플레이어로 게임 허브에 합법적으로 연결**합니다.

---

## 무응답 / 거절일 때

- 2주 정도 답이 없으면 **무응답 = 허락 아님**입니다. 올리지 마세요.
- 거절하면 깔끔하게 포기하고, **합법 임베드 게임(GameMonetize/CrazyGames) 중 비슷한 장르**로 대체합니다.
  - 검 강화하기류 → 강화/뽑기 캐주얼
  - 슈의 라면가게류 → 식당 경영 게임
  - 아기고래 쿠아류 → 물고기 먹기(feeding) 게임

---

## 개발 쪽 준비 (yeon 팀)

원작자 허락 + SWF만 확보되면 기술은 준비돼 있습니다:
- Ruffle(@ruffle-rs/ruffle) 플레이어 컴포넌트를 게임 허브에 통합
- `GameEntry`에 `kind: "swf"`(자체 호스팅) / `kind: "iframe"`(외부 임베드) 구분 추가
- 허락 게임은 출처·라이선스 메모를 카탈로그에 함께 기록

> 진행 순서 추천: **개인 제작 게임 1~2개 먼저 허락 시도 → 되면 Ruffle 통합 구현 → 점차 확대.**
