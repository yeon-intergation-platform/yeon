# Design Eye

## 1. "AI 디자인 냄새" 감지 체크리스트

- blur blob 배경을 습관적으로 넣지 않는다.
- 의미 없는 장식용 아이콘을 제목 옆에 붙이지 않는다.
- 한국어 서비스에서 영어 레이블을 남발하지 않는다.
- 과도한 그라데이션과 과도한 수식어를 피한다.
- 카드 안에 또 카드가 들어가는 이중 구조를 기본값처럼 쓰지 않는다.
- 빈 공간만 크고 정보 구조가 약한 레이아웃을 피한다.

## 2. 비주얼 위계

- 가장 중요한 것이 가장 먼저 보여야 한다.
- 강조 색상은 1~2곳에만 쓴다.
- 같은 위계의 정보는 같은 스타일을 사용한다.
- 굵기와 크기를 과도하게 섞지 않는다.

## 3. CTA 원칙

- 버튼은 동사로 시작하고 행동이 명확해야 한다.
- Primary CTA는 화면당 1개를 기본값으로 삼는다.
- `확인`, `더보기` 같은 범용 문구는 문맥이 없으면 피한다.

## 4. 레이아웃 판단

- 카드 내 빈 공간이 콘텐츠보다 크면 구조를 재설계한다.
- `justify-between`으로 관계 있는 요소를 불필요하게 멀리 떼어놓지 않는다.
- 모바일 375px 기준으로 먼저 본다.
- 터치 타겟은 최소 44px를 의식한다.

## 5. 이 저장소 전용 스타일링 판단

- 기본 Tailwind 유틸리티 사용을 허용한다.
- 기본 scale로 충분한데 굳이 토큰을 새로 만들지 않는다.
- 반복 의미가 생기는 값만 토큰화한다.
- 전역 CSS에 화면 전용 스타일을 추가하지 않는다.

## 6. 실행 루틴

1. 현재 UI를 스캔해 AI 디자인 냄새를 제거한다.
2. 핵심 행동과 CTA가 가장 두드러지는지 확인한다.
3. 컴포넌트 간 일관성과 레이블 언어를 맞춘다.
4. 모바일 기준으로 어색한 간격과 위계를 먼저 잡는다.
5. 왜 문제인지, 무엇을 어떻게 바꿀지, 결과가 어떻게 좋아지는지를 설명 가능하게 만든다.
