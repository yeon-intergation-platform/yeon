# Session Insights — 과거 세션 분석

과거 작업 이력을 분석하여 패턴과 인사이트를 도출한다.

## 데이터 소스

아래 소스를 종합하여 분석한다:

1. **git log** — 커밋 히스토리, 브랜치별 활동, 시간대별 패턴
2. **docs/** + **ai-log/** — 공식 문서, 날짜별 AI 작업 기록
3. **MEMORY.md + 메모리 파일** — 축적된 피드백, 프로젝트 메모리
4. **.claude/skills/** — 현재 등록된 스킬 목록

## 분석 항목

### 1. 활동 요약

- 분석 기간 내 커밋 수, 브랜치 수, PR 수
- 가장 많이 수정된 파일/디렉토리 top 5
- 작업 빈도가 높은 시간대 (git log 타임스탬프 기반)

### 2. 작업 패턴 분석

- 자주 반복되는 작업 유형 (feat/fix/refactor/docs 비율)
- 평균 PR 크기 (변경 라인 수)
- 한 브랜치에서 커밋 → PR → 머지까지 걸린 커밋 수 패턴

### 3. 메모리 활용도

- 현재 메모리 파일 수와 카테고리별 분포
- 최근 추가/수정된 메모리
- 오래되어 검증이 필요한 메모리 후보

### 4. 스킬 활용 제안

- 반복 패턴 중 아직 스킬로 등록되지 않은 것
- 기존 스킬 중 개선이 필요한 것

## 출력 형식

```
## Session Insights

### 활동 요약 (최근 N일)
| 항목 | 값 |
|------|-----|
| 커밋 수 | N |
| 브랜치 수 | N |
| PR 수 | N |
| 가장 활발한 영역 | path/to/dir |

### 자주 수정된 파일 Top 5
1. `파일경로` — N회 수정
2. ...

### 작업 패턴
- feat: N% | fix: N% | refactor: N% | docs: N%
- 평균 PR 크기: +N / -N lines

### 메모리 현황
- 총 N개 (user: N, feedback: N, project: N, reference: N)
- 검증 필요: [오래된 메모리 목록]

### 제안
- [ ] A. [구체적 제안]
- [ ] B. [구체적 제안]
```

## 옵션

- `/session-insights` — 전체 기간 분석 (기본값)
- `/session-insights 7d` — 최근 7일
- `/session-insights 30d` — 최근 30일

## 핵심 원칙

- 데이터가 없는 항목은 "데이터 부족"으로 표시하고 억지로 채우지 않는다.
- git log가 주요 데이터 소스이므로, 커밋이 적으면 분석 범위가 제한됨을 명시한다.
- 제안은 실행 가능한 것만 포함한다.
- **동시 작업 인식**: git log에는 여러 에이전트의 커밋이 섞여 있을 수 있다. 커밋 메시지 패턴이나 브랜치 명으로 "이 에이전트의 작업"을 구분할 수 있을 때만 에이전트별 분석을 시도하고, 불분명하면 전체 활동으로 통합 분석한다.
