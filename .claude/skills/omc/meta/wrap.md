# Wrap — 세션 마무리 시스템

세션 종료 전 이 명령어를 실행하면, 이번 세션에서 수행한 작업을 분석하여 5가지 관점에서 정리한다.

## 실행 절차

5개의 서브에이전트를 **병렬로** 실행한다. 각 에이전트는 이번 세션의 대화 맥락과 변경된 코드를 기반으로 작업한다.

### Agent 1: 문서 업데이트 감지

- 이번 세션에서 변경된 코드가 기존 문서(CLAUDE.md, CLAUDE.local.md, skills/\*.md, MEMORY.md)와 불일치하는지 점검
- 업데이트가 필요한 문서와 구체적인 변경 내용을 제안
- 새로운 메모리로 저장할 가치가 있는 정보가 있는지 확인

### Agent 2: 반복 패턴 자동화 제안

- 이번 세션에서 반복적으로 수행한 작업 패턴을 식별
- hook, skill, 또는 설정으로 자동화할 수 있는 것이 있는지 분석
- 구체적인 자동화 방법을 제안

### Agent 3: TIL (Today I Learned) 추출

- 이번 세션에서 새롭게 알게 된 기술적 사실, 디버깅 경험, 아키텍처 결정을 추출
- `ai-log/{person}/YYYY-MM-DD/TIL.md` 형식으로 정리할 내용을 준비
- 이미 알고 있던 것의 반복은 제외하고, 진짜 새로운 발견만 포함

### Agent 4: 후속 작업 정리

- 이번 세션에서 완료하지 못했거나, "나중에" 하기로 미룬 작업을 수집
- 다음 세션에서 바로 시작할 수 있도록 구체적인 시작점(파일, 라인, 맥락)을 정리
- 우선순위를 부여하여 정렬

### Agent 5: 중복 검증

- Agent 1~4의 결과가 서로 중복되거나 모순되는 부분이 없는지 검증
- 이미 MEMORY.md나 기존 문서에 있는 내용과 중복되는 제안을 제거
- 최종 결과를 정제

## 출력 형식

5개 에이전트의 결과를 종합하여 **객관식 체크리스트**로 제시한다.

```
## 세션 마무리 결과

아래에서 적용할 항목을 선택해주세요.

### 문서 업데이트
- [ ] A. CLAUDE.md에 [구체적 내용] 추가
- [ ] B. 메모리 저장: [구체적 내용]

### 자동화 제안
- [ ] C. [패턴]을 hook으로 자동화
- [ ] D. [패턴]을 새 skill로 등록

### TIL
- [ ] E. TIL 저장: [구체적 내용]

### 후속 작업
- [ ] F. [작업명] — `파일경로:라인` 부터 시작
- [ ] G. [작업명] — [시작점 설명]

(해당 카테고리에 항목이 없으면 생략)
```

사용자가 번호를 선택하면 해당 항목만 실행한다. 선택하지 않은 항목은 무시한다.

## 핵심 원칙

- 억지로 항목을 만들지 않는다. 각 카테고리별로 해당 사항이 없으면 생략.
- 모든 제안은 구체적이어야 한다. "문서를 업데이트하면 좋겠다" 같은 모호한 제안 금지.
- 사용자가 선택하기 전에 자동으로 파일을 수정하지 않는다.
- **동시 작업 인식**: 이 세션의 git log나 변경 파일에 다른 에이전트의 커밋이 섞여 있을 수 있다. 분석 시 "이번 세션에서 내가 한 작업"과 "다른 에이전트가 한 작업"을 구분하여 TIL, 후속 작업, 문서 제안이 현재 에이전트의 실제 작업 범위에서 도출되도록 한다.
