## 진단

- 추출된 시트의 프레임별 alpha bbox 검출 → 가로 중심 ±0.5px, 세로 baseline 동일 → **자산 정렬 문제 아님**.
- 트랙 sprite scale: 카멜 0.479 / 아스나·린네아 0.221 (모두 비정수). Phaser 기본 antialiasing이 sub-pixel scale에서 프레임마다 다른 픽셀 분포를 생성 → 떨림.
- 카드 CSS: `displayHeight=160` / `frameHeight=208` 비정수 scale → background-position 1px rounding 불일치 → 카드 떨림 가능성.
- fps=12: 8프레임을 12fps로 → 0.67초 한 사이클. 카멜(6프레임 10fps)의 0.6초보다 빠르게 보일 수 있고 다리 동작이 빨라 보임.

## 수정

- `packages/typing-race-engine/src/index.ts`: `Phaser.Game` config에 `pixelArt: true` 추가. nearest-neighbor 샘플링 + roundPixels 자동 → 떨림 제거 + 픽셀아트 선명도 유지.
- `apps/web/src/features/typing-service/typing-profile-card.tsx`: `CharacterSprite`에 `snapDisplayHeight` 도입. frameHeight의 정수 분수 중 maxHeight(160) 이하의 가장 큰 값으로 snap → asuna/linnea = 104, camel = 96. background-size·position이 모두 정수 픽셀.
- `apps/web/src/features/typing-service/characters/index.ts`: asuna·linnea fps `12` → `8` (다리 속도 완화).
