## 배경

- 4/21 백로그(`docs/product/backlog/history/2026-04-21/2-타자연습-프로필-캐릭터선택_BACKLOG.md`) 1차수에서 캐릭터 선택 UI 골격은 잡혔지만, 실제 캐릭터는 카멜 1종 + 96×96 6프레임 하드코딩.
- 사용자가 외부 chibi 스프라이트시트 2종(asuna, linnea, 모두 1536×1872 / 9행×8열 / 셀 192×208) 제공.
- 양 시트 모두 2번째 줄(y 213~410)이 8프레임 오른쪽 달리기로 검증됨(facing offset 분석으로 RIGHT 확인).
- 카멜 96×96 6프레임과 신규 캐릭터 192×208 8프레임이 동거해야 하므로, 카드/엔진의 frame 메타 하드코딩을 메타데이터 기반으로 일반화 필요.

## 결정

- 단일 SoT: `apps/web/src/features/typing-service/characters/{types.ts,index.ts}` (CharacterDef registry).
- 자산 컨벤션: `apps/web/public/sprites/characters/<id>/run.png` (8프레임 가로 시트, alpha 보존).
- 정규화 도구: `scripts/sprites/extract-character-run.py` — 원본 시트의 RUN_ROW(2번째 줄)을 8프레임 PNG로 추출. flip 옵션 보유. 캐릭터 추가 시 manifest에 한 줄.
- 카드/엔진 모두 메타에서 frameWidth/Height/Count/Cols/fps 읽음. 캐릭터별 다름 허용.
- 엔진의 NPC 레인은 항상 카멜 고정. local 레인만 선택 캐릭터 사용. 트랙 시각 일관성 위해 LANE_DISPLAY_HEIGHT(46px) 기준 자동 scale.
- 라벨 i18n은 TRANSLATIONS 키 추가 대신 registry에 `label: { ko, en }` 직접 두는 방식 채택 (캐릭터 추가 부담 최소화).
